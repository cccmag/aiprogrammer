## 濃縮回顧

### 注意力機制的直覺

注意力機制的核心思想非常直覺：**在處理資訊時，動態地給予不同部分不同的權重**。

```
┌─────────────────────────────────────────────────────┐
│                  注意力機制圖示                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│   Query: "查詢"                                     │
│         │                                           │
│         ▼                                           │
│   ┌─────────────────────────┐                      │
│   │   Attention Score       │                      │
│   │   (計算相關性)           │                      │
│   └───────────┬─────────────┘                      │
│               │                                    │
│               ▼                                    │
│   ┌─────────────────────────┐                      │
│   │   Softmax → Attention   │                      │
│   │   Weights               │                      │
│   └───────────┬─────────────┘                      │
│               │                                    │
│               ▼                                    │
│         加權和 = 輸出                               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 從 seq2seq 注意力到自注意力

**編碼器-解碼器注意力**（2015）：
- 解碼器每步「關注」編碼器的隱藏狀態
- 解決了固定維度向量的瓶頸

**自注意力**（2016）：
- 序列中的每個位置「關注」同一序列的所有位置
- 捕捉長距離依賴
- 完全並行的計算

### Transformer 的創新

Transformer 完全拋棄了 RNN 的循環結構，只使用注意力機制：

```
┌─────────────────────────────────────────────────────┐
│              Transformer 架構                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│   輸入: [x1, x2, x3, x4]                           │
│         │                                           │
│         ▼                                           │
│   ┌─────────────────────────┐                      │
│   │  Embedding + Positional │                      │
│   │  Encoding               │                      │
│   └───────────┬─────────────┘                      │
│               │                                    │
│               ▼                                    │
│   ┌─────────────────────────┐                      │
│   │    Multi-Head Attention │                      │
│   │    (Self-Attention)     │                      │
│   └───────────┬─────────────┘                      │
│               │                                    │
│               ▼                                    │
│   ┌─────────────────────────┐                      │
│   │    Feed-Forward Network │                      │
│   └───────────┬─────────────┘                      │
│               │                                    │
│               │  × N 層                           │

```

### 預訓練革命的開端

2018 年的 BERT 和 GPT 將 Transformer 與預訓練結合：

**BERT（雙向）**：
- 遮罩語言模型（MLM）
- 預測被遮罩的詞
- 適用於理解任務

**GPT（單向）**：
- 語言模型（LM）
- 預測下一個詞
- 適用於生成任務

## 結論與展望

注意力機制的發展經歷了幾個重要階段：

1. **認知靈感**：從人類視覺注意力得到啟發
2. **seq2seq 應用**：解決機器翻譯的瓶頸
3. **自注意力**：實現完全並行的長距離依賴
4. **Transformer**：純注意力架構
5. **預訓練革命**：BERT、GPT 開啟新時代

Transformer 的成功揭示了一個重要原則：**注意力機制是一種強大的通用計算原語，可以替代循環結構**。

未來趨勢：
- 更高效的注意力機制
- 稀疏注意力
- 跨模態注意力
- 理論解釋性的深入
