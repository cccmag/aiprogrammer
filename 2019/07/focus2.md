# 長短期記憶網路 (LSTM)

## 1997 年的重大突破

1997 年，Sepp Hochreiter 和 Jürgen Schmidhuber 在論文《Long Short-Term Memory》中提出了長短期記憶網路（Long Short-Term Memory, LSTM），這是類神經網路領域最具影響力的發明之一。

LSTM 的核心貢獻是：**透過引入閘門機制，有效解決了標準 RNN 的梯度消失問題，使網路能夠學習長達數百甚至數千時間步的長期依賴關係**。

---

## 為什麼需要 LSTM？

### 標準 RNN 的問題

標準 RNN 的隱藏層更新公式：

```
h_t = tanh(W * x_t + U * h_{t-1} + b)
```

反向傳播時，梯度包含 `U` 的連乘：

```
∂E/∂U = Σ_t ∂E/∂h_t * ∂h_t/∂U

其中 ∂h_t/∂h_{t-1} ∝ U^T * tanh'(...) 
    ∂h_t/∂h_{t-k} ∝ (U^T * tanh'(...))^k
```

當序列很長時，`(U^T * tanh'(...))^k` 會：
- 如果譜半徑 > 1：**梯度爆炸**
- 如果譜半徑 < 1：**梯度消失**

### 梯度消失的具體例子

假設我們要訓練一個語言模型，學習以下依賴：

```
輸入：「雖然 天氣 很 冷，但 他 仍然 去 游泳 因為 他 熱愛 [?]"
                                                    ↑
                                            需要記住「他」的性別
```

性別資訊（「他」）出現在第 8 個詞，但預測目標在第 14 個詞。如果梯度消失，網路無法學習這種長期依賴。

## LSTM 的核心設計

### 記憶單元（Memory Cell）

LSTM 引入了「記憶單元」的概念，用於長期儲存資訊：

```
┌─────────────────────────────────────────────────────┐
│                 LSTM 記憶單元                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│   c_{t-1} ──────────────────────────► c_t         │
│       │                                           │
│       │ 遗忘                                       │
│       ▼                                           │
│   ┌─────────┐     ┌─────────┐     ┌─────────┐       │
│   │ forget  │     │  input  │     │ output  │       │
│   │  gate   │     │  gate   │     │  gate   │       │
│   │  f_t    │     │  i_t    │     │  o_t    │       │
│   └────┬────┘     └────┬────┘     └────┬────┘       │
│        │             │             │              │
│        │             │             │              │
│        │      ┌──────┴──────┐      │              │
│        │      │             │      │              │
│        │      ▼             ▼      │              │
│        │   ┌──────────────┐  │      │              │
│        └──►│    新候選值    │◄─┘      │              │
│             │    c_tilde   │         │              │
│             └──────────────┘         │              │
│                    │                   │              │
│                    └────────┬──────────┘              │
│                             ▼                        │
│                         c_t                          │
│                         │                           │
│                         ▼                           │
│                      tanh(c_t) * o_t = h_t           │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 三個閘門的數學定義

**1. 遺忘閘（Forget Gate）**
```
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)
```
決定從記憶單元中丟棄什麼資訊。輸出值接近 0 表示「完全遺忘」，接近 1 表示「完全保留」。

**2. 輸入閘（Input Gate）**
```
i_t = σ(W_i · [h_{t-1}, x_t] + b_i)
c_tilde = tanh(W_c · [h_{t-1}, x_t] + b_c)
```
決定儲存什麼新資訊到記憶單元。

**3. 輸出閘（Output Gate）**
```
o_t = σ(W_o · [h_{t-1}, x_t] + b_o)
h_t = o_t * tanh(c_t)
```
決定從記憶單元輸出什麼資訊。

### 記憶單元更新

```
c_t = f_t * c_{t-1} + i_t * c_tilde
```

這個公式結合了：
- **遺忘**：移除舊資訊的一部分（`f_t * c_{t-1}`）
- **輸入**：添加新資訊的一部分（`i_t * c_tilde`）

這種設計使得：
- 記憶可以在長時間內保持穩定
- 梯度可以在記憶單元中長期流動而不衰減

## LSTM 的變體

### Peephole 連接

原始 LSTM 沒有窺視孔（Peephole）連接。後來的研究者添加了從記憶單元到閘門的信號：

```
f_t = σ(W_f · [c_{t-1}, h_{t-1}, x_t] + b_f)
```

這讓閘門能夠「看到」當前的記憶狀態，進一步提升性能。

### 耦合輸入和遺忘閘

將兩個閘門耦合可以減少參數數量：

```
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)
i_t = 1 - f_t  // 直接耦合
c_t = f_t * c_{t-1} + i_t * c_tilde
```

### Gated Recurrent Unit (GRU)

2014 年，Cho 等人提出了 GRU，這是 LSTM 的簡化版本（詳見 focus3.md）。

## LSTM 為何能解決梯度消失？

### 記憶單元的恆等傳播

關鍵在於：當 `f_t ≈ 1` 且 `i_t ≈ 0` 時：

```
c_t ≈ c_{t-1}
```

這意味著：
- 記憶單元可以在很長的時間內保持幾乎不變
- 梯度可以沿著記憶單元傳播而不衰減

```
∂c_t/∂c_{t-1} = f_t ≈ 1 (梯度近乎完美傳播)

對比標準 RNN：
∂h_t/∂h_{t-1} = U * tanh'(...) (梯度依賴於 U)
```

### 閘門的軟控制

閘門的輸出是 0-1 之間的連續值，由 Sigmoid 函式產生。這允許網路學習「部分遺忘」和「部分記憶」的策略。

```
σ(x) = 1 / (1 + e^{-x}) ∈ (0, 1)

示例：
σ(3) ≈ 0.95  (保留 95% 資訊)
σ(-3) ≈ 0.05 (遺忘 95% 資訊)
σ(0) = 0.50  (隨機)
```

## LSTM 的應用

### 自然語言處理

LSTM 徹底改變了 NLP 領域：

1. **語言模型**：學習預測下一個詞
2. **機器翻譯**：編碼-解碼架構
3. **情感分析**：判斷文字情緒
4. **文字生成**：創意寫作輔助
5. **命名實體識別**：識別人名、地名等

### 語音處理

1. **語音辨識**：將語音轉換為文字
2. **語音合成**：生成自然語音
3. **說話者辨識**：聲紋認證

### 時間序列預測

1. **股票價格預測**：金融模型
2. **氣象預測**：天氣預報
3. **能源消耗預測**：智慧電網

### 視頻分析

1. **動作辨識**：理解視頻中的動作
2. **視頻描述**：生成視頻文字描述
3. **異常檢測**：識別異常行為

## LSTM 的局限性

### 計算複雜度

LSTM 每個時間步需要計算 4 個閘門，每個閘門都需要完整的矩陣運算：

- 參數數量：4 × (hidden_size × (input_size + hidden_size + 1))
- 時間複雜度：O(sequence_length × hidden_size²)

### 難以處理極長序列

即使 LSTM 有記憶機制，當序列非常長時（如 > 1000 步），仍然可能遇到困難：

1. **長期依賴的選擇**：哪些資訊值得記憶？
2. **記憶容量**：固定大小的記憶單元能儲存的資訊有限
3. **順序計算**：無法平行處理序列的不同部分

### 注意力機制的誕生

為了解決這些問題，2015 年研究者提出了注意力機制（Attention Mechanism）。這項創新最終導致了 Transformer 架構的誕生，開啟了深度學習的新時代。
