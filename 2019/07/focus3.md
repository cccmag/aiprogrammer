# GRU 與閘道循環單元

## 2014 年的簡化革命

2014 年，Kyunghyun Cho 等人在論文《Learning Phrase Representations using RNN Encoder-Decoder for Statistical Machine Translation》中提出了 Gated Recurrent Unit（GRU），這是對 LSTM 的一次重要簡化。

GRU 的核心思想是：**在保持 LSTM 強大功能的同時，減少計算複雜度和參數數量**。

---

## GRU vs LSTM：架構比較

### LSTM 的複雜性

LSTM 有三個閘門和一個記憶單元：

```
LSTM 更新公式（每次迭代需計算）：
1. f_t = σ(W_f · [h_{t-1}, x_t] + b_f)     # 遺忘閘
2. i_t = σ(W_i · [h_{t-1}, x_t] + b_i)     # 輸入閘
3. c~_t = tanh(W_c · [h_{t-1}, x_t] + b_c) # 新候選值
4. c_t = f_t * c_{t-1} + i_t * c~_t        # 記憶更新
5. o_t = σ(W_o · [h_{t-1}, x_t] + b_o)     # 輸出閘
6. h_t = o_t * tanh(c_t)                   # 隱藏狀態
```

**參數數量**：`4 × (input_size + hidden_size + 1) × hidden_size`

### GRU 的簡化

GRU 只有兩個閘門，沒有獨立的記憶單元：

```
GRU 更新公式：
1. z_t = σ(W_z · [h_{t-1}, x_t])          # 更新閘
2. r_t = σ(W_r · [h_{t-1}, x_t])          # 重置閘
3. h~_t = tanh(W · [r_t * h_{t-1}, x_t])  # 新候選隱藏狀態
4. h_t = (1 - z_t) * h_{t-1} + z_t * h~_t # 隱藏狀態更新
```

**參數數量**：`2 × (input_size + hidden_size + 1) × hidden_size`

## GRU 架構圖解

```
┌─────────────────────────────────────────────────────┐
│                    GRU 單元                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│   h_{t-1} ──┬──► ┌────────┐                       │
│              │    │  更新  │                       │
│              ├──► │  閘 z_t │                       │
│              │    └────────┘                       │
│              │         │                           │
│              │         ▼                           │
│              │    ┌────────┐     ┌───────────┐    │
│              ├──► │  重置  │────►│ 候選隱藏   │    │
│              │    │  閘 r_t │     │  狀態 h~_t│    │
│              │    └────────┘     └───────────┘    │
│              │         │              │             │
│              └─────────┴──────────────┤             │
│                                        │             │
│                                        ▼             │
│                               h_t = (1-z_t)*h_{t-1} │
│                                     + z_t * h~_t    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 兩個閘門的含義

**1. 更新閘（Update Gate）`z_t`**

更新閘類似於 LSTM 的遺忘閘和輸入閘的組合：
- `z_t` 越大，保留的前一個隱藏狀態越多
- `z_t` 越小，候選隱藏狀態的影響越大

```
h_t = (1 - z_t) * h_{t-1} + z_t * h~_t

當 z_t = 1：h_t ≈ h_{t-1}（完全保留過去）
當 z_t = 0：h_t ≈ h~_t（完全接受新值）
```

**2. 重置閘（Reset Gate）`r_t`**

重置閘控制前一個隱藏狀態在候選計算中的使用程度：
- `r_t` 越大，前一個隱藏狀態的影響越大
- `r_t` 越小，前一個隱藏狀態被忽略

```
h~_t = tanh(W · [r_t * h_{t-1}, x_t])

當 r_t = 1：正常計算，考慮完整歷史
當 r_t = 0：忽略歷史，只看當前輸入
```

## GRU 的優勢

### 參數更少

| 架構 | 閘門數 | 參數數量（相對） |
|------|--------|-----------------|
| LSTM | 3 個閘 + 記憶 | 100% |
| GRU  | 2 個閘 | ~75% |

減少的參數來自：
- 合併了遺忘閘和輸入閘
- 沒有獨立的輸出閘
- 沒有額外的記憶單元（直接用隱藏狀態）

### 訓練更快

較少的參數意味著：
- 更少的記憶體需求
- 更快的梯度計算
- 更快的收斂速度

### 性能相當

在多項任務上，GRU 的性能與 LSTM 相近：

| 任務 | LSTM 準確率 | GRU 準確率 |
|------|-------------|------------|
| 語言建模 | 1.55 | 1.58 |
| 機器翻譯 | 24.2 BLEU | 24.5 BLEU |
| 情感分析 | 88.6% | 88.4% |

（小差異在統計顯著性邊緣）

### 更適合小數據集

由於 GRU 參數較少，過擬合風險較低，在數據集較小時可能表現更好。

## 何時選擇 LSTM vs GRU？

### 選擇 GRU 的情況

1. **計算資源有限**：需要更快的訓練和推理
2. **數據集較小**：減少過擬合風險
3. **需要簡單架構**：便於理解和部署
4. **序列相對較短**：如語音命令辨識

### 選擇 LSTM 的情況

1. **需要更強的記憶能力**：處理非常長的序列
2. **任務複雜度高**：如長文本生成
3. **有多個時間尺度的依賴**：需要精細控制
4. **研究目的**：需要更多彈性和可解釋性

### 實證建議

> 「如果不知道選哪個，先試 GRU。」
> — deep learning 社群常見建議

GRU 通常是更好的起點，如果效果不佳再嘗試 LSTM。

## 延伸：其他 RNN 變體

### MGU（Minimal Gated Unit）

2016 年提出的進一步簡化版本，只有單個閘門：

```python
# MGU 更新
z_t = σ(W_z · [h_{t-1}, x_t])
n_t = tanh(W_n · [h_{t-1}, x_t])
h_t = (1 - z_t) * h_{t-1} + z_t * n_t
```

## 結論

GRU 代表了一種重要的深度學習設計哲學：**簡化和效率**。

在多項任務上，GRU 展現了與 LSTM 相近甚至更好的性能，同時擁有更少的參數和更快的訓練速度。這種簡化並沒有犧牲核心功能，而是透過合併相似的操作，找到了效率與效果的平衡點。

GRU 的成功也啟示我們：有時候，更少的元件反而能帶來更好的結果。在追求模型複雜度之前，值得先思考是否有不必要的複雜性可以去除。
