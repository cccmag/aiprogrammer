# 本期焦點

## 循環神經網路 (RNN/LSTM) 的發展歷程

### 引言

在深度學習的宇宙中，有一類模型特別擅長處理序列資料——這就是循環神經網路（Recurrent Neural Network, RNN）。從語音辨識到自然語言處理，從音樂生成到股票預測，RNN 在處理時間序列和序列資料方面展現了獨特的威力。

本期將帶領讀者回顧 RNN 的發展歷程，從最早的 Hopfield 網路，到改變遊戲規則的長短期記憶網路（LSTM），再到各種變體和應用。讓我們一起探索這段迷人的技術演進之旅。

## 濃縮回顧

### RNN 的基本原理

RNN 的核心思想是什麼？簡單來說，就是讓神經網路擁有「記憶」。傳統的神經網路（例如 MLP）假設輸入和輸出是獨立的，但序列資料的特點是：每一個時間點的輸出，不僅取決於當前輸入，還取決於之前的「記憶」。

```
┌─────────────────────────────────────────────────────┐
│                  RNN 展開圖                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│   h0 ──► h1 ──► h2 ──► h3 ──► ...                  │
│   │     │     │     │                              │
│   x0     x1     x2     x3     ...                  │
│                                                     │
│   h_t = f(W * x_t + U * h_{t-1} + b)              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

其中：
- `x_t` 是時間 t 的輸入
- `h_t` 是時間 t 的隱藏狀態（記憶）
- `W`, `U`, `b` 是可學習的參數
- `f` 是非線性激活函式（通常用 tanh 或 ReLU）

### LSTM 的創新

標準 RNN 面臨一個嚴重的問題：梯度消失（Vanishing Gradient）。當序列很長時，梯度在反向傳播過程中會呈指數衰減，導致早期的資訊幾乎完全丢失。

1997 年，Sepp Hochreiter 和 Jürgen Schmidhuber 提出了 LSTM，透過引入「閘門」機制解決了這個問題：

```
┌─────────────────────────────────────────────────────┐
│                  LSTM 記憶單元                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│   ┌───────┐     ┌───────┐     ┌───────┐           │
│   │ forget │     │ input │     │ output│           │
│   │  gate  │     │  gate │     │  gate │           │
│   └───┬───┘     └───┬───┘     └───┬───┘           │
│       │             │             │                │
│       ▼             ▼             ▼                │
│   ┌─────────────────────────────────┐             │
│   │          記憶單元狀態 c           │             │
│   │                                 │             │
│   │   c_t = f_t * c_{t-1} + i_t * c~│             │
│   │   h_t = o_t * tanh(c_t)         │             │
│   └─────────────────────────────────┘             │
│                                                     │
└─────────────────────────────────────────────────────┘
```

LSTM 的三個閘門：
- **遺忘閘（Forget Gate）**：決定丟棄什麼資訊
- **輸入閘（Input Gate）**：決定儲存什麼新資訊
- **輸出閘（Output Gate）**：決定輸出什麼資訊

### GRU：LSTM 的簡化版本

2014 年，Cho 等人提出了 GRU（Gated Recurrent Unit），這是 LSTM 的簡化版本。GRU 只使用兩個閘門：

- **更新閘（Update Gate）**：結合了 LSTM 的遺忘閘和輸入閘
- **重置閘（Reset Gate）**：控制忽略多少之前的隱藏狀態

GRU 在許多任務上達到了與 LSTM 相當的性能，同時計算更簡單。

### 序列到序列模型

2014 年，Sutskever 等人提出了 Sequence-to-Sequence 模型，這是 RNN 發展的重要里程碑。這個架構由兩個 RNN 組成：

- **Encoder**：將輸入序列編碼為一個固定維度的向量
- **Decoder**：根據編碼向量生成輸出序列

這使得 RNN 能夠處理輸入輸出長度不同的任務，如機器翻譯、摘要生成。

### 注意力機制

2015 年，Bahdanau 等人提出了注意力機制，這是 RNN 架構的重大突破。傳統的 Seq2Seq 模型瓶頸在於：所有資訊必須壓縮到一個固定維度的向量中。

注意力機制允許 Decoder 在生成每個輸出時，「關注」輸入序列的不同部分：

```
┌─────────────────────────────────────────────────────┐
│                    注意力機制                       │
├─────────────────────────────────────────────────────┤
│                                                     │
│   Input:   [x1,   x2,   x3,   x4,   x5]           │
│             │     │     │     │     │              │
│           α1     α2     α3     α4     α5           │
│           ↓      ↓      ↓      ↓      ↓            │
│         注意力加權和                                 │
│              │                                       │
│              ▼                                       │
│         當前輸出 y_t                                 │
│                                                     │
│   α_{ti} = softmax(score(h_{t-1}, h_i))           │
│                                                     │
└─────────────────────────────────────────────────────┘
```

注意力機制後來成為 Transformer 架構的基礎，最終引發了 NLP 領域的革命。

## 結論與展望

RNN 經歷了從理論誕生到工程成熟的完整歷程。Hopfield 網路揭示了循環連接的價值，Elman 網路開啟了語言建模的探索，而 LSTM/GRU 解決了困擾研究者多年的梯度問題。

然而，RNN 也有其局限性：
1. **順序計算**：難以平行化，訓練速度受限於序列長度
2. **長期依賴**：即使 LSTM/GRU，在非常長的序列上仍有挑戰
3. **上下文限制**：固定大小的隱藏狀態難以儲存無限的資訊

這些局限性最終促成了 Transformer 架構的誕生——它完全拋棄了循環結構，採用純注意力機制，取得了更好的效果。

但 RNN 的影響力並未消失。在處理即時語音、控制系統、時間序列預測等需要「在線」處理的場景下，RNN 仍然是首選。而 RNN 的思想——處理序列資料、維護隱藏狀態——也深深影響了後來的模型設計。
