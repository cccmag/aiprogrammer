# RNN 的誕生與早期發展（1980s-1990s）

## 從感知機到循環連接

在類神經網路的早期發展中，研究者面臨一個核心問題：如何讓網路處理序列資料？傳統的感知機和 MLP 假設輸入和輸出是獨立的，但現實世界中的很多問題——語音辨識、語言處理、時間序列預測——都具有時間依賴性。

這個問題的解決思路很自然：如果讓網路的輸出連接到自身，形成「循環」，那麼網路就拥有了「記憶」之前輸入的能力。

---

## Hopfield 網路：循環網路的先驅

### 物理學家的跨界貢獻

1982 年，John Hopfield（一位主要研究統計力學的物理學家）提出了以他名字命名的 Hopfield 網路。這是最早的循環神經網路之一，雖然結構簡單，卻開創了重要的研究方向。

Hopfield 網路是一種完全連接的循環網路，每個神經元都與其他所有神經元相連（圖像中的虛線表示連接）：

```
┌─────────────────────────────────────────────────────┐
│                  Hopfield 網路                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│        ○ ←──→ ○                                    │
│       ↙↖     ↙↖                                   │
│      ○   ←→   ○                                   │
│       ↘↙     ↘↙                                   │
│        ○ ←──→ ○                                    │
│                                                     │
│   每個神經元與所有其他神經元雙向連接                 │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### Hopfield 網路的特性

Hopfield 網路的核心特性：

1. **二元狀態**：每個神經元的狀態是 0 或 1
2. **對稱連接**：連接權重 `w_ij = w_ji`
3. **能量函式**：網路狀態對應一個能量值，系統趨向能量極小值

網路的運作方式：
- **儲存階段**：將記憶模式以權重形式儲存
- **回想階段**：輸入部分模式，網路會收斂到最接近的儲存模式

### 聯想記憶

Hopfield 網路最著名的應用是「聯想記憶」。假設我們儲存了幾個圖案：

```
記憶1：        記憶2：        輸入：
■ □ ■         □ ■ □         ■ □ ■
□ ■ □         ■ □ ■         □ □ □
■ □ ■         □ ■ □         ■ □ ■

回想結果：→ ■ □ ■ （網路收斂到記憶1）
```

這種「根據部分資訊恢復完整記憶」的能力，與人腦的聯想記憶有相似之處。

### 局限性

Hopfield 網路雖然有理論價值，但存在嚴重的局限性：

1. **儲存容量有限**：n 個神經元最多只能儲存約 0.14n 個模式
2. **虛假記憶**：兩個相似的記憶可能會互相干扰
3. **收斂緩慢**：訓練和回想都需要多次迭代

這些問題限制了 Hopfield 網路的實際應用，但它的循環結構概念為後來的 RNN 奠定了基礎。

## Jordan 網路：狀態輸出回饋

### 1986 年的重要貢獻

1986 年，Michael Jordan 提出了 Jordan 網路，這是另一個重要的早期循環網路架構。

Jordan 網路的特點是：將隱藏層的輸出作為額外輸入，重新饋入網路：

```
┌─────────────────────────────────────────────────────┐
│                  Jordan 網路                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│   輸入 ──► 隱藏層 ──► 輸出層                        │
│              │                                     │
│              │ 狀態單元                             │
│              ↓                                     │
│         ┌─────────┐                               │
│         │  延遲   │ (delay)                       │
│         └─────────┘                               │
│              │                                     │
│              └──────► 饋入隱藏層                   │
│                                                     │
│   狀態單元 = 輸出層（或隱藏層）的延遲版本           │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 狀態單元的概念

Jordan 網路引入了「狀態單元」（State Unit）的概念：

- 狀態單元記錄網路之前的「狀態」
- 這個狀態被饋入網路，影響後續輸出
- 實現了簡單的「記憶」功能

這種結構特別適合處理需要「記憶一段時間」的任務，如：

- **簡單的序列預測**：根據前 n 個值預測下一個值
- **運動控制**：根據當前狀態和歷史軌跡生成控制訊號
- **簡單的模式識別**：識別具有時間結構的圖案

## Elman 網路：隱藏層狀態回饋

### 1990 年的突破

1990 年，Jeffrey Elman 提出了 Elman 網路，這是另一個經典的早期 RNN 架構。Elman 網路與 Jordan 網路的主要區別在於：回饋的信號來源。

```
┌─────────────────────────────────────────────────────┐
│                  Elman 網路                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│   輸入 ──► 隱藏層 ──► 輸出層                        │
│              │                                     │
│              │ 隱藏層狀態                           │
│              ↓                                     │
│         ┌─────────┐                               │
│         │  延遲   │ (delay)                       │
│         └─────────┘                               │
│              │                                     │
│              └──────► 饋入隱藏層                   │
│                                                     │
│   狀態單元 = 隱藏層的延遲版本                        │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### Elman 網路的創新

Elman 網路的關鍵創新：

1. **隱藏層回饋**：使用隱藏層的輸出而非輸出層的輸出
2. **更豐富的表示**：隱藏層通常能學習到更高層次的特徵
3. **靈活性**：適用於更複雜的序列建模任務

### 語言處理的早期應用

Elman 網路在語言處理領域有早期應用。研究者用它來學習簡單的語法結構，例如：

- **主語-動詞一致性**：學習 "The dog runs" vs "The dogs run"
- **時態辨識**：學習過去式和現在式的區別
- **簡單的模式**：如括號匹配

這些研究展示了 RNN 在處理序列資料上的潛力，但限於當時的計算能力和訓練演算法，效果有限。

## BPTT：訓練循環網路

### 反向傳播的擴展

1990 年，Paul Werbos 提出了 BPTT（Backpropagation Through Time）演算法，這是訓練循環網路的關鍵方法。

BPTT 的核心思想：**將循環網路展開為多層前饋網路，然後使用標準的反向傳播**。

```
時間步 1      時間步 2      時間步 3
   │            │            │
   ▼            ▼            ▼
┌──────┐     ┌──────┐     ┌──────┐
│  x1  │────►│  x2  │────►│  x3  │
└──┬───┘     └──┬───┘     └──┬───┘
   │            │            │
   ▼            ▼            ▼
┌──────┐     ┌──────┐     ┌──────┐
│  h1  │────►│  h2  │────►│  h3  │──► 輸出
└──┬───┘     └──┬───┘     └──┬───┘
   │            │            │
   └────────────┴────────────┘
      (連接權重相同)
```

### BPTT 的步驟

1. **展開**：將 RNN 在時間維度上展開
2. **前向傳播**：計算每個時間步的輸出
3. **計算誤差**：比較預測輸出與真實標籤
4. **反向傳播**：將誤差沿時間維度反向傳播
5. **梯度裁剪**：防止梯度爆炸

### 梯度消失：RNN 的阿基里斯之腱

BPTT 揭示了標準 RNN 的一個嚴重問題：**梯度消失（Vanishing Gradient）**。

當序列很長時，誤差信號需要通過多個時間步傳播回早期：

```
時間步 t ──► ... ──► 時間步 2 ──► 時間步 1
     │                │              │
     └────────────────┴──────────────┘
              反向傳播路徑

梯度在傳播過程中：|∂E/∂W| → 0（指數衰減）
```

這意味著：
- **早期時間步的梯度趨近於零**
- **網路無法學習長期依賴**
- **只能捕捉近期的依賴關係**

這個問題長期困擾著 RNN 的發展，直到 LSTM 的出現才得到有效解決。

## 早期 RNN 的應用與局限性

### 成功應用

在 LSTM 出現之前，早期 RNN 在某些領域取得了成功：

1. **手寫辨識**：利用 RNN 的序列建模能力
2. **簡單的語音辨識**：封閉詞彙的語音命令辨識
3. **時間序列預測**：金融市場預測、氣象預報
4. **音樂生成**：學習音樂的時間結構

### 局限性

但這些應用都有嚴格的限制：

1. **序列長度**：只能處理短序列（通常 < 20 步）
2. **計算代價**：訓練困難，收斂緩慢
3. **記憶能力**：無法有效記憶長期資訊
4. **理論基礎**：缺乏對學習演算法的深入理解

這些局限性使得 RNN 在 1990 年代末至 2010 年代初逐渐被其他方法超越，直到深度學習的興起和 LSTM 的發明才重新焕發活力。
