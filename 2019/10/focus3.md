# GPT 與 GPT-2 的發展

## 前言

OpenAI 的 GPT 系列是預訓練語言模型發展的重要推動者。從 2018 年的 GPT 到 2019 年的 GPT-2，這些模型展示了生成式 AI 的巨大潛力。本篇文章將詳細介紹 GPT 系列的發展歷程。

## GPT：生成式預訓練的開端

### OpenAI 的願景

2018 年 6 月，OpenAI 發布了 GPT（Generative Pre-training），這是第一個基於 Transformer 的大規模生成式預訓練模型。

### GPT 的架構

GPT 使用了 12 層 Transformer 解碼器：

```
GPT 配置：
- 層數：12
- 隱藏維度：768
- 注意力頭數：12
- 總參數量：117M
```

### GPT 的訓練

GPT 的訓練分為兩個階段：

**無監督預訓練**：
```python
# 最大化語言模型的似然
L = Σ log P(w_t | w_{<t})
```

**監督微調**：
```python
# 在下游任務上微調
L = Σ log P(y | x_1, ..., x_n)
```

## GPT-2：更大的願景

### 發布歷程

GPT-2 的發布經歷了一個備受爭議的過程：

| 時間 | 發布版本 | 參數量 |
|------|----------|--------|
| 2019年2月 | Small（完整版） | 117M |
| 2019年4月 | Medium | 345M |
| 2019年5月 | Large | 774M |
| 2019年8月 | XL | 1.5B |
| 2019年11月 | 完整版 | 1.5B |

### GPT-2 的爭議

2019 年 2 月，OpenAI 宣布 GPT-2 可能「太危險而不宜公開」，引發了業界廣泛討論。這是因為 GPT-2 在文字生成方面展現了驚人的能力：

```
輸入：在一個寒冷的冬夜，
輸出：一個老人走進了酒吧，點了一杯熱可可...
```

### GPT-2 的能力

GPT-2 1.5B 模型在多個方面展現了卓越能力：

1. **文字生成**：可以生成流暢、連貫的文章
2. **問答**：能夠回答簡單的常識問題
3. **摘要**：可以對文件進行摘要
4. **翻譯**：在多語言場景中表現不錯

### GPT-2 的架構改進

GPT-2 在 GPT 的基礎上進行了幾項改進：

1. **更大的規模**：從 117M 增加到 1.5B 參數
2. **更好的資料過濾**：使用更高品質的網頁文字
3. **更長的上下文**：從 512 增加到 1024 tokens
4. **改進的正規化**：使用更好的權重初始化

### GPT-2 的訓練資料

GPT-2 使用了 WebText 資料集，這是一個從 Reddit 高贊連結中抓取的網頁文字集合：

- 約 800 萬文檔
- 約 40GB 文字
- 時間跨度：2008-2019

## 生成式預訓練的意義

### 語言建模的傳統

語言建模是 NLP 中最基礎的任務之一。GPT 系列展示了：

> 「語言建模是 NLP 任務的一個良好晴雨表，也是大多數其他 NLP 任務的壓縮表示。」

### GPT 系列的貢獻

1. **展示了大規模預訓練的威力**
2. **推動了生成式 AI 的發展**
3. **引發了對 AI 安全的討論**

## GPT-2 的爭議與討論

### 安全考量

OpenAI 當時選擇分階段發布 GPT-2，引發了關於 AI 安全的討論：

- **擔憂**：可能被用於生成假新聞、垃圾郵件
- **支持**：透明度有助於研究社群應對風險
- **結果**：完整版最終於 2019 年 11 月發布

### 開源與限制的平衡

GPT-2 的爭議反映了 AI 領域的一個核心問題：

> 如何在開放研究和安全考量之間取得平衡？

## 結論

GPT 和 GPT-2 開創了生成式預訓練的先河，展示了大規模語言模型的驚人能力。雖然 GPT-2 的發布充滿爭議，但它無疑推動了 NLP 領域的發展，並引發了對 AI 安全的深入討論。
