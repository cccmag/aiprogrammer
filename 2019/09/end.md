# 回顧與結語

## 2019 年 9 月號結語

### 本月主題回顧

九月是 TensorFlow 2.0 正式發布的月份，也是 Keras 成為官方高階 API 的里程碑。經過四年的發展，TensorFlow 從一個強調計算圖和 Session 的框架，轉變為一個更加 Pythonic 和易用的深度學習平台。

#### TensorFlow 2.0 的革命

**Eager Execution 默認**：TensorFlow 2.0 將動態計算圖設為默認，告別了繁瑣的 `tf.Session`。這極大地降低了 TensorFlow 的學習曲線，讓調試變得如同普通 Python 代碼一樣直觀。

**tf.keras 統一**：`tf.keras` 成為官方的高階 API，統一了模型構建的方式。現在，無論是快速原型開發還是生產環境部署，都可以通過同一套 API 完成。

**API 清理**：移除了大量過時的 API，簡化了整個框架的結構。

#### Keras 的崛起

Keras 從一個社群人氣庫成長為 TensorFlow 的官方 API，這個歷程值得深思。Keras 的成功證明了「易用性」在深度學習中的重要性。一個好的 API 設計不僅能吸引更多用戶，還能推動整個生態系統的發展。

### 深度學習框架的競爭態勢

2019 年秋季的深度學習框架競爭：

| 框架 | 優點 | 劣勢 |
|------|------|------|
| TensorFlow 2.0 | 生態完整、部署方便 | 學習曲線仍較陡 |
| PyTorch | 動態圖、學術界受歡迎 | 部署相對麻煩 |
| JAX | 函式式、靈活 | 生態較小 |
| Keras | 易用性高 | 靈活性較低 |

### 未來展望

**框架融合**：TensorFlow 2.0 和 PyTorch 正在互相學習，差異越來越小。

**自動機器學習**：AutoML 和模型搜尋將更加普及。

**邊緣部署**：TensorFlow Lite、ONNX 等工具讓邊緣部署更加便捷。

**更大的模型**：預訓練模型的規模將繼續增長。
