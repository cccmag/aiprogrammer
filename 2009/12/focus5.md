# 分散式版本控制：Git 與 GitHub

## Git 的成熟

### 版本發布

```bash
# Git 1.6.x（2009年）

# 主要改進
git clone       # 更快
git bisect      # 自動二分搜尋
git stash       # 暫存更改
git submodule    # 子模組支援

# 更好的補丁
git format-patch
git send-email
```

## GitHub 的崛起

### 成長數據

```markdown
# GitHub 2009 年統計

用戶數：突破 100 萬

組織：

```

## 社交程式設計

### 功能

```markdown
# GitHub 社交功能

1. Fork & Pull Request
   - 貢獻開源專案
   - 程式碼審查

2. 關注（Watch）
   - 追蹤有興趣的專案

3. Gist
   - 程式碼片段分享

4. 組織（Organizations）
   - 團隊協作
```

## Git vs 其他 VCS

```markdown
| 特性       | Git      | SVN      | Mercurial |
|-----------|----------|----------|-----------|
| 速度       | 非常快    | 慢        | 快        |
| 分支       | 簡單     | 複雜      | 簡單       |
| 離線支援    | 完整     | 有限      | 完整       |
| 學習曲線    | 陡峭     | 簡單      | 中等       |
```

## 結語

Git 和 GitHub 改變了開源開發的方式，2009 年是分散式版本控制成為主流的起點。
