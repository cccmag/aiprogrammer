# TCP/IP 協議詳解

TCP/IP 是網路通訊的基礎，是一組協議的集合，定義了資料如何在網路中傳輸。

---

## OSI 模型

網路通訊的分層模型：

| 層級 | 名稱 | 協議範例 |
|------|------|----------|
| 7 | 應用層 | HTTP, FTP, SSH |
| 6 | 表達層 | TLS, SSL |
| 5 | 會議層 | NetBIOS |
| 4 | 傳輸層 | TCP, UDP |
| 3 | 網路層 | IP, ICMP |
| 2 | 資料連結層 | Ethernet |
| 1 | 實體層 | 光纖、銅線 |

---

## TCP 協定

### 特性

- **連接導向**：需要建立連接
- **可靠傳輸**：確認、重傳機制
- **順序保證**：資料按序到達
- **流量控制**：避免發送過快
- **擁塞控制**：避免網路壅塞

### 三向交握 (Three-Way Handshake)

```
用戶端                      伺服器
  │                          │
  │────── SYN ──────────────>│  1. 同步請求
  │                          │
  │<──── SYN-ACK ───────────│  2. 同步確認
  │                          │
  │────── ACK ──────────────>│  3. 確認
  │                          │
  │       資料傳輸           │
```

## UDP 協定

### 特性

- **無連接**：不需要建立連接
- **不可靠**：不保證傳輸
- **高效能**：開銷小、延遲低
- **無流量/擁塞控制**

### 使用場景

- DNS 查詢
- 線上遊戲
- 串流媒體
- VoIP

### UDP vs TCP 比較

| 特性 | TCP | UDP |
|------|-----|-----|
| 連接性 | 連接導向 | 無連接 |
| 可靠性 | 可靠 | 不可靠 |
| 順序 | 有序 | 無序 |
| 速度 | 較慢 | 較快 |
| 開銷 | 較大 | 較小 |
| 應用 | 網頁、郵件 | 影片、遊戲 |

## IP 協定

### IPv4

- 32 位元位址
- 約 42 億個地址
- 範例：192.168.1.1

### IPv6

- 128 位元位址
- 幾乎無限的地址空間
- 範例：2001:0db8:85a3:0000:0000:8a2e:0370:7334

### IP 標頭
