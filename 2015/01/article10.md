# HTTP/2 標準正式通過

## 前言

2015 年，HTTP/2 標準正式通過，這是 HTTP 协议自 1999 年 HTTP/1.1 以來的最大更新。

## 與 HTTP/1.1 的比較

```
HTTP/1.1 問題：
────────────────
- 連線阻塞：同一時間只能一個請求
- 龐大的 Header：每次請求都帶完整 Header
- 無意義的等待：必須等待回應才能發送下一個請求

HTTP/2 解決方案：
────────────────
- 多工傳輸：單一連線並行多個請求
- Header 壓縮：HPACK 演算法
- 伺服器推送：主動推送資源
- 流量控制：更好的資源管理
```

## 多工傳輸

```
HTTP/1.1:                    HTTP/2:
─────────                    ─────────
請求 1 ────────             ════════════════
回應 1 ────────             請求 1 ═══════
                             回應 1 ═══════
請求 2 ────────             請求 2 ═══════
回應 2 ────────             回應 2 ═══════
                             請求 3 ═══════
請求 3 ────────             回應 3 ═══════
```

## 伺服器推送

```apache
# Apache 設定
<IfModule http2_module>
    Protocols h2 h2c
</IfModule>

# Nginx 設定
http {
    http2 on;
}
```

## 結論

HTTP/2 大幅提升了網頁效能，為未來的 Web 應用奠定基礎。
