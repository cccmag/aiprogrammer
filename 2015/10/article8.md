# 軟體架構圖繪製技巧

## 前言

好的架構圖可以幫助團隊理解系統設計、促進溝通、記錄決策。

## 常用圖表類型

### 1. 系統架構圖

展示系統的主要元件和它們之間的關係。

```
┌─────────────┐     ┌─────────────┐
│   Client    │────>│   API GW    │
└─────────────┘     └──────┬──────┘
                            │
            ┌───────────────┼───────────────┐
            │               │               │
            ▼               ▼               ▼
     ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
     │   Users     │ │   Orders    │ │   Products  │
     │   Service   │ │   Service   │ │   Service   │
     └─────────────┘ └─────────────┘ └─────────────┘
```

### 2. 元件圖

展示元件的結構和介面。

### 3. 時序圖

展示物件之間的交互順序。

### 4. 部署圖

展示軟體元件如何部署到硬體。

## UML 基礎

### 類別圖

```
┌────────────────────────┐
│        User            │
├────────────────────────┤
│ - name: str            │
│ - email: str           │
├────────────────────────┤
│ + create(): void       │
│ + authenticate(): bool │
└────────────────────────┘
         △
         │
         │ extends
         │
┌────────────────────────┐
│     AdminUser          │
├────────────────────────┤
│ + permissions: list    │
├────────────────────────┤
│ + manage_users(): void │
└────────────────────────┘
```

## 工具推薦

- **draw.io**：免費、線上、易用
- **Lucidchart**：功能豐富、協作
- **Miro**：白板工具、適合頭腦風暴
- **PlantUML**：文字生成圖表

## 小結

架構圖是重要的溝通工具。選擇適合的圖表類型和工具，讓團隊更容易理解系統設計。
