# 系統程式設計的學習路徑

## 學習 C 語言

### 基礎語法
- 變數、運算子、控制流程
- 函式
- 陣列和指標
- 結構和聯合

### 進階主題
- 指標和記憶體管理
- 指標運算
- 函式指標
- 動態資料結構

## Unix 系統程式設計

### 必須掌握的概念

1. **檔案 I/O**：open、read、write、close
2. **程序管理**：fork、exec、wait
3. **程序間通訊**：pipe、socket、shm
4. **信號**：signal、kill
5. **執行緒**：pthread

### 推薦的學習步驟

1. 學習基本的 Unix API
2. 實作簡單的程式（如 cat、ls）
3. 理解程序模型
4. 實現客戶端/伺服器
5. 學習並發程式設計

## 結論

系統程式設計需要時間和耐心。從 C 語言的基礎開始，逐步學習 Unix API，最終能夠編寫複雜的系統軟體。這個過程雖然艱辛，但會讓你對電腦系統有深入的理解，這是任何應用程式設計師都需要的基礎。
