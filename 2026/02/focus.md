## 從程式到 AI — 機器學習開發者入門

### 引言

傳統程式設計與機器學習之間有本質的差異。傳統程式中，開發者必須明確編寫每一條規則；在機器學習中，開發者提供資料和演算法，讓模型自己從資料中學習規則。本期雜誌將帶領讀者從熟悉程式設計的世界出發，逐步進入機器學習的殿堂。

我們將從最基本的機器學習概念開始，逐步深入監督式學習、分類、決策樹、神經網路等主題。每一篇文章都配有 Python 程式碼範例，讓您能親手實作、親眼見證模型訓練的過程。

## 核心概念：機器學習流程

```
┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐
│  收集資料   │ →  │  資料預處理  │ →  │  訓練模型   │ →  │  評估模型   │
└────────────┘    └────────────┘    └────────────┘    └────────────┘
                          ↑                                 │
                          │                                 ↓
                          └──────── 調整參數 ──────────────┘
```

機器學習的典型流程包含四個步驟：

1. **收集資料**：取得有代表性的訓練資料
2. **資料預處理**：清理缺失值、標準化特徵
3. **訓練模型**：選擇演算法並用資料學習
4. **評估模型**：用未見過的資料測試效能

## 結論與展望

機器學習正在改變我們撰寫程式的方式。從傳統的「寫規則」到現代的「餵資料」，這種思維轉變是每個程式開發者在 AI 時代必須經歷的過程。

本期的主題涵蓋了機器學習的核心概念，從簡單的線性迴歸到神經網路，從模型評估到實戰應用。我們希望這能為讀者打下堅實的基礎，讓您在深度學習的學習路上走得更穩。
