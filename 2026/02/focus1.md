# 什麼是機器學習？

## 程式設計 vs 機器學習

傳統程式設計的運作方式如下：開發者撰寫明確的規則，電腦根據這些規則處理輸入資料，產生輸出結果。這種模式在規則明確的問題上表現極佳，例如計算稅金、排序資料或渲染圖形。

```
傳統程式：輸入 + 規則 = 輸出
機器學習：輸入 + 輸出 = 規則
```

機器學習翻轉了這個流程。開發者提供大量的輸入和對應的輸出（標籤），讓演算法自動從中學習規則。學習完成後，模型就能對新的輸入進行預測。

## 機器學習的三大類型

### 監督式學習

監督式學習使用已標記的資料進行訓練。每個訓練樣本都包含特徵（輸入）和標籤（正確答案）。模型學習從特徵到標籤的映射關係。

```
訓練資料：[特徵1, 特徵2, ...] → 標籤
新資料：[特徵1, 特徵2, ...] → 模型預測
```

常見任務：
- **迴歸**：預測連續值，如房價、氣溫
- **分類**：預測離散類別，如垃圾郵件偵測、疾病診斷

### 非監督式學習

非監督式學習使用未標記的資料。模型自行發現資料中的結構、模式和群集。

常見任務：
- **聚類**：將相似樣本分組，如客戶分群
- **降維**：減少特徵數量，保留重要資訊

### 強化學習

強化學習中，代理（agent）透過與環境互動來學習。代理執行動作，環境回饋獎勵或懲罰，代理從中學習最大化累積獎勵的策略。

## 核心術語

- **特徵（Feature）**：用來描述樣本的屬性，如房屋的坪數、臥室數量
- **標籤（Label）**：要預測的目標值，如房屋的實際售價
- **模型（Model）**：學習到的映射函式，能將新特徵映射到預測值
- **訓練（Training）**：使用資料讓模型調整參數的過程
- **推論（Inference）**：使用訓練好的模型進行預測

## 機器學習的應用場景

- **電腦視覺**：人臉辨識、物體檢測、醫學影像分析
- **自然語言處理**：機器翻譯、情感分析、聊天機器人
- **推薦系統**：電商商品推薦、影音內容推薦
- **異常檢測**：信用卡詐騙偵測、工業設備故障預警
- **預測分析**：股市預測、天氣預報、銷售預測
