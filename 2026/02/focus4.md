# 決策樹與隨機森林

## 直觀的決策過程

想像你要判斷是否出門運動。你會根據一系列問題做決定：

```
天氣好嗎？
├── 是 → 我有時間嗎？
│       ├── 是 → 出門運動
│       └── 否 → 待在家
└── 否 → 下雨嗎？
        ├── 是 → 去健身房
        └── 否 → 出門運動
```

這就是決策樹的直觀理解——透過一系列 if-else 問題來做決策。

## 決策樹模型

決策樹是一種樹狀結構模型，由以下部分組成：
- **根節點**：第一個判斷條件
- **內部節點**：特徵判斷條件
- **葉節點**：最終預測結果
- **分支**：判斷結果的路徑

### 分類與迴歸

決策樹可以處理分類和迴歸任務：
- **分類樹**：葉節點是類別標籤（取眾數）
- **迴歸樹**：葉節點是連續值（取平均值）

## 如何建立決策樹

建立決策樹的關鍵是決定「在哪個特徵、用什麼閾值進行分割」。

### 資訊增益

對於分類任務，常用的分割標準是資訊增益，基於熵（Entropy）的概念：

```
熵 H = -Σ pᵢ · log₂(pᵢ)
```

熵衡量資料的混亂程度。當所有樣本屬於同一類別時，熵為 0；當各類別均勻分布時，熵最大。

分割後的加權熵與原熵的差值就是資訊增益：

```
資訊增益 = 原熵 - 加權子節點熵
```

演算法選擇使資訊增益最大的特徵和閾值進行分割。

### 基尼不純度

另一個常用的指標是基尼不純度（Gini Impurity）：

```
Gini = 1 - Σ pᵢ²
```

基尼不純度計算簡單，是 scikit-learn 的預設指標。

## 決策樹的剪枝

決策樹容易過擬合——樹越深，越可能記住訓練資料中的雜訊而非真實模式。

防止過擬合的方法：
- **預剪枝**：限制樹的最大深度、最小葉節點樣本數等
- **後剪枝**：先建立完整樹，再移除貢獻不大的分支

## 隨機森林

隨機森林（Random Forest）是決策樹的整合版本，透過 Bagging（Bootstrap Aggregating）技術提升效能：

1. 從原始資料中有放回地隨機抽樣，建立多個子資料集
2. 對每個子資料集訓練一棵決策樹
3. 預測時，分類任務取眾數，迴歸任務取平均

隨機森林還引入「隨機特徵選擇」：每次分割時只考慮隨機選取的部分特徵，增加了樹之間的多樣性。

### 隨機森林的優點

- **高準確率**：多棵樹的投票通常優於單棵樹
- **抗過擬合**：平均化減少變異數
- **特徵重要性**：可計算每個特徵的貢獻度
- **處理高維資料**：對大量特徵表現良好
- **不需標準化**：樹模型不受特徵尺度的影響

```python
from sklearn.ensemble import RandomForestClassifier

clf = RandomForestClassifier(n_estimators=100, max_depth=5, random_state=42)
clf.fit(X_train, y_train)
print(clf.feature_importances_)  # 特徵重要性
```
