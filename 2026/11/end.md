# 回顧與結語

## 2026 年 11 月號結語

### 本月主題回顧

本期雜誌深入探討了 **Rust 在嵌入式系統中的應用**——從最底層的暫存器操作，到 embedded-hal 抽象層，再到 RTIC 即時框架，最後觸及 AI 輔助開發與邊緣 ML 推論。

#### mini-embedded 專案的意義

本期附帶的 [mini-embedded](focus_code.md) 專案以模擬的方式展示了嵌入式系統的核心抽象。雖然無法代替真實的硬體除錯體驗，但它清楚地說明了 Rust 如何以型別安全的方式管理 GPIO、UART、定時器和中斷——這些是嵌入式開發的四大基石。

專案的三個關鍵設計決策值得記錄：

1. **`RefCell` 模擬周邊共享**：真實硬體中，周邊是中斷和主程式共享的資源。`RefCell` 提供了執行期借用檢查，近似於真實 MCU 的硬體存取限制。

2. **型別狀態區分 GPIO 模式**：雖然模擬中用了 `assert!` 在執行期檢查，真實的 embedded-hal 透過型別系統在編譯期防止錯誤——這才是 Rust 的真正優勢。

3. **NVIC 模擬的中斷佇列**：真實的中斷控制器遠比 FIFO 佇列複雜，但這個簡化模型抓住了本質：中斷啟用、掛起、服務的完整生命週期。

#### Rust 嵌入式生態的現狀

2026 年的今天，Rust 在嵌入式領域已經從「實驗性專案」成長為「生產就緒的選擇」。

**已成熟的領域：**
- Cortex-M 系列的完整支援（M0–M85）
- 主流 MCU 的 HAL crate（STM32、ESP32、nRF、RP2040）
- RTIC 和 embassy 兩個優秀的即時/非同步框架
- probe-rs 等 Rust 原生除錯工具

**仍在發展中的領域：**
- RISC-V 嵌入式的工具鏈成熟度
- 汽車級（ISO 26262）和安全關鍵系統的認證
- 超低功耗場景的生態整合
- AI/ML 推論框架在 MCU 上的最佳化

### AI + 嵌入式：新時代的開始

本月最令人振奮的趨勢是 **AI 正在改變嵌入式開發的工作流程**：

- **從 Datasheet 到程式碼**：LLM 可以直接從暫存器描述生成 PAC 層綁定，將數小時的手動工作縮短到幾分鐘
- **自動化驅動開發**：對於常見感測器（BME280、MPU9250、SSD1306），AI 可以生成完整的驅動程式碼框架
- **除錯輔助**：AI 能綜合 datasheet 知識和錯誤現象，快速定位問題根源

但需要清醒認識的是：AI 生成的嵌入式程式碼仍需人工審查和測試。硬體容錯空間遠比軟體小——一個錯誤的暫存器位元設定可能燒壞硬體。

### 未來展望

**Rust 將成為嵌入式開發的主流語言**：雖然 C 語言仍會存在很長一段時間，但新專案選擇 Rust 的比例正在快速增長。所有權模型對嵌入式開發的記憶體安全問題是根本性的解決方案。

**AI 輔助開發將標準化**：未來五年，AI 將成為嵌入式開發工具箱中的標準配備——從自動生成暫存器定義到智慧除錯，AI 將大幅降低嵌入式開發的門檻。

**邊緣 AI 將持續成長**：TFLite Micro 和 Rust 原生方案的競爭將推動更好的 MCU ML 推論引擎出現。語音辨識、異常檢測、預測性維護將成為邊緣 AI 的主要應用場景。

### 給讀者的建議

如果您是嵌入式開發的新手：
- 從 [mini-embedded](focus_code.md) 專案開始，理解基礎概念
- 然後買一塊 STM32 或 ESP32 開發板，實際動手
- 參考本期文章，逐步建立完整的嵌入式 Rust 知識體系

如果您已經是嵌入式工程師：
- 嘗試將一個小型專案用 Rust 重寫
- 比較開發效率和最終產品的可靠性
- 你會驚訝於編譯器幫你抓到了多少 C 語言中難以發現的錯誤
