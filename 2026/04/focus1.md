# 機器語言的起源：從打孔卡到組合語言（1940s-1950s）

## 計算的遠古時代

在編譯器誕生之前，程式設計是一項極為原始的活動。程式設計師需要直接與機器的硬體對話——使用二進位機器碼。

### 打孔卡時代

19 世紀末，Herman Hollerith 發明了打孔卡（Punched Card）用於美國人口普查。到了 1940 年代，打孔卡成為電腦程式輸入的主要媒介。

```
┌─────────────────────────────────────────────────────┐
│                   打孔卡示意                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌───┬───┬───┬───┬───┬───┬───┬───┐              │
│  │ ■ │   │ ■ │   │ ■ │   │ ■ │   │              │
│  ├───┼───┼───┼───┼───┼───┼───┼───┤              │
│  │ 0 │ 1 │ 0 │ 1 │ 0 │ 1 │ 0 │ 1 │              │
│  └───┴───┴───┴───┴───┴───┴───┴───┘              │
│                                                     │
│  打孔代表 1，不打孔代表 0                            │
│  這種編碼方式一直影響到今天                           │
│                                                     │
└─────────────────────────────────────────────────────┘
```

程式設計師（當時通常是物理學家或數學家）需要：

1. 在紙上寫出要執行的機器指令
2. 將指令手動轉換為二進位碼
3. 在打孔卡上打出對應的孔洞
4. 將打孔卡送入電腦執行

一個簡單的加法可能需要數十張打孔卡，而一旦卡序弄亂，除錯就成了一場惡夢。

ENIAC（1945 年）是世界上第一台通用電子計算機。它的程式設計方式是：

```

│                 ENIAC 程式設計                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ENIAC 沒有儲存程式能力！                            │
│                                                     │
│  程式是透過以下方式「設定」的：                       │
│                                                     │
│  1. 插拔電纜（像老式電話接線生）                    │
│  2. 設定數百個旋鈕和開關                            │
│  3. 物理上重新配置機櫃                              │
│                                                     │
│  換一個程式需要數小時甚至數天的重新配線！            │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 儲存程式概念與組合語言的誕生

### 馮紐曼架構

1945 年，John von Neumann 提出了儲存程式概念（Stored-Program Concept）：程式和資料共用同一個記憶體空間。這使得程式可以像資料一樣被讀寫，為編譯器的誕生鋪平了道路。

```
┌─────────────────────────────────────────────────────┐
│                  馮紐曼架構                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌──────────────┐    ┌──────────────┐             │
│  │    CPU       │    │    記憶體    │             │
│  │              │◄──►│              │             │
│  │  ┌────────┐  │    │  ┌────────┐  │             │
│  │  │ 控制單元│  │    │  │ 程式   │  │             │
│  │  ├────────┤  │    │  ├────────┤  │             │
│  │  │ 算術邏輯│  │    │  │ 資料   │  │             │
│  │  │ 單元   │  │    │  └────────┘  │             │
│  │  └────────┘  │    │              │             │
│  └──────┬───────┘    └──────────────┘             │
│         │                                           │
│         ▼                                           │
│  ┌──────────────┐                                   │
│  │   I/O 設備   │                                   │
│  └──────────────┘                                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 組合語言的誕生

1950 年代初期，程式設計師開始使用助憶碼（Mnemonics）來代替二進位指令：

```
// 二進位機器碼（人難以閱讀）
10110000 01100001

// 組合語言助憶碼（人可閱讀）
MOV AL, 61h

// 組合語言範例
MOV AX, 5     ; 將 5 送入 AX 暫存器
ADD AX, 3     ; AX = AX + 3
MOV RESULT, AX ; 將結果存入記憶體
```

組合語言（Assembly Language）的優點是顯而易見的：

1. **可讀性**：`MOV`、`ADD` 比 `10110000` 好記得多
2. **生產力**：不需要記住每個指令的二進位編碼
3. **錯誤減少**：助憶碼比二進位更容易檢查

### 組合器的發明

組合器（Assembler）是第一個將原始碼轉換為機器碼的程式。它的工作非常簡單——建立一個「助憶碼 → 機器碼」的對照表：

```
原始碼 (source.asm)        組合器              機器碼
──────────────────────────────────────────────────────
  MOV AX, 5       ───►    組合器     ───►   B8 05 00
  ADD AX, 3       ───►    組合器     ───►   05 03 00
  MOV [RESULT], AX──►      組合器     ───►   A3 00 02
```

組合器引入了幾個重要概念：

1. **符號標籤（Labels）**：使用名稱代替記憶體位址
2. **偽指令（Pseudo-ops）**：給組合器的指令，如 `DATA`、`CODE` 段定義
3. **巨集（Macros）**：重複使用的程式碼片段

## Grace Hopper 與第一個編譯器

### Admiral Grace Hopper

Grace Hopper（1906-1992）是美國海軍少將，也是計算機科學的先驅。她在 1952 年提出了編譯器（Compiler）的概念，並開發了第一個編譯器——A-0 System。

> 「我被人們問過很多次：為什麼你要發明編譯器？答案是：我太懶了。我不想每次都要記住二進位指令，於是我就寫了一個程式來幫我做這件事。」—— Grace Hopper

### A-0 System

A-0 System 是世界上第一個編譯器。它的工作方式是：

```
┌─────────────────────────────────────────────────────┐
│                 A-0 System 編譯流程                   │
├─────────────────────────────────────────────────────┤
│                                                     │
│  輸入：符號表示的子程式集合                          │
│    ┌──────────────────┐                            │
│    │ SUBROUTINE ADD   │                            │
│    │ LOAD A           │                            │
│    │ ADD B            │                            │
│    │ STORE C          │                            │
│    └──────────────────┘                            │
│            │                                        │
│            ▼                                        │
│    ┌──────────────────┐                            │
│    │  A-0 Compiler   │                            │
│    └──────────────────┘                            │
│            │                                        │
│            ▼                                        │
│  輸出：機器碼                                      │
│    ┌──────────────────┐                            │
│    │ 00 01 02 03 04  │                            │
│    └──────────────────┘                            │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 對電腦界的巨大影響

Hopper 對電腦界的貢獻巨大，以下是一些關於她的故事：

1. **編譯器（Compiler）一詞的由來**：Hopper 從「彙編」的概念中獲得了靈感。她說：「我發明這個詞的時候，人們覺得我瘋了——電腦怎麼可能幫你寫程式？」

2. **除錯（Debug）一詞的來由**：1947 年，哈佛 Mark II 電腦出現故障。Hopper 的團隊打開機器，發現一隻飛蛾卡在繼電器中。她用膠帶將飛蛾貼在日誌上，並寫下「First actual case of bug being found.」——這就是「bug」和「debug」的來由。

3. **COBOL 的誕生**：1959 年，Hopper 參與了 COBOL（Common Business-Oriented Language）的設計。COBOL 是世界上第一個廣泛使用的商業程式語言，其編譯器在當年是一項重大成就。

## 第一個編譯器面臨的挑戰

### 自我編譯的悖論

早期編譯器開發者面臨一個哲學問題：編譯器本身就是一個程式，那麼第一個編譯器是怎麼寫出來的？

```
┌─────────────────────────────────────────────────────┐
│                 Bootstrapping 問題                   │
├─────────────────────────────────────────────────────┤
│                                                     │
│  有人說：「在 C 編譯器存在之前，人們是用什麼來      │
│  編譯 C 程式的？」                                  │
│                                                     │
│  答案：先用另一種語言寫第一個版本，然後用那個       │
│  版本來編譯自己。這個過程叫 Bootstrapping！         │
│                                                     │
│  步驟：                                             │
│  1. 用組合語言寫一個簡單的 C 編譯器（C0）         │
│  2. 用 C0 編譯一個更好的 C 編譯器（C1）            │
│  3. 用 C1 編譯 C 編譯器的正式版本                  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 硬體限制

1950 年代的電腦記憶體非常有限：

- IBM 701（1952）：最多 32K 字（36 位元/字）
- UNIVAC I（1951）：最多 12K 字（12 位元/字元）
- 編譯器本身必須很小，還要留空間給編譯結果

## 結語

從打孔卡到組合語言，再到第一個編譯器——1940 年代到 1950 年代是程式設計從機械走向自動化的關鍵時期。Grace Hopper 的遠見和 John Backus 的實作，為後來的 FORTRAN 編譯器鋪平了道路。
