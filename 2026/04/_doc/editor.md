# AI 輔助雜誌編輯實戰手冊

本文記錄使用 AI（OpenCode + Big Pickle）編輯《AI 程式人雜誌》的完整流程與技巧。這些經驗來自於 2026 年 4 月號的實際編輯過程，主題是「編譯器的奧秘：從原始碼到機器碼」。

## 二、AI 協作流程

### 2.1 任務分解策略

不要一次要求 AI 完成整本雜誌，而是分層次逐步完成：

1. **先建立骨架**：先完成目錄結構、主要連結
2. **再填入內容**：針對每個檔案逐步填充內容
3. **最後除錯修正**：檢查連結、程式碼正確性

### 2.2 上下文傳遞

在對話開始時，明確告知 AI：

- 當前工作目錄
- 專案目的（編輯雜誌）
- 已完成的檔案列表
- 當前任務目標

### 2.3 進度追蹤

使用摘要開頭，讓 AI 知道：

- 已完成的工作
- 當前任務
- 遇到的問題
- 下一個步驟

## 三、程式碼處理技巧

### 3.1 大型程式碼檔案處理

ToyLang 直譯器包含 Lexer、Parser、AST、Interpreter 等多個元件，總計超過 400 行。建議：

1. 先在 focus_code.md 中設計好程式碼架構
2. 直接寫入 `_code/toylang.py` 檔案
3. 執行測試驗證正確性

### 3.2 多檔案連結維護

**策略**：當重新命名檔案時

1. 使用 `mv` 批次重新命名
2. 用 `grep` 找出所有引用舊檔名的地方
3. 使用 `edit` 工具的 `replaceAll` 參數一次性替換

### 3.3 程式碼測試

在 Python 檔案中實作 `test()` 函式：

```python
def test():
    # 測試範例
    code = """
    func fib(n) {
        if n <= 1 { return n; }
        return fib(n-1) + fib(n-2);
    }
    print(fib(10));
    """

if __name__ == "__main__": test()
```

## 四、文件一致性維護

### 4.1 連結檢查清單

每次修改檔案結構後，檢查：

- [ ] README.md 的目錄連結
- [ ] focus.md 的大綱連結
- [ ] articles.md 的文章連結
- [ ] 所有交叉引用是否正確

### 4.2 批次取代技巧

使用 `replaceAll=true` 參數時要小心，確保舊字串具有唯一性。

### 4.3 內文同步

修改標題時，要一併更新：

- 檔案開頭的標題 (`# Title`)
- 結語段落中的標題提及
- 所有導航連結中的標題文字

## 五、編譯器主題的寫作經驗

### 5.1 歷史脈絡的把握

編譯器主題長達 80 年，需要把握關鍵節點：

- 1940s：從機器碼到組合語言（Grace Hopper）
- 1957：FORTRAN 編譯器的誕生（John Backus）
- 1960s：形式語言理論與 Dragon Book
- 1970s-1990s：最佳化技術的成熟
- 1990s-2000s：JIT 與虛擬機器
- 2000s-2020s：LLVM 與現代基礎設施
- 2020s：AI 與編譯器的融合

### 5.2 程式碼範例的設計

ToyLang 直譯器的設計考量：

1. **簡單易懂**：使用 Python，不需要額外依賴
2. **功能完整**：支援變數、函式、條件、迴圈、遞迴
3. **可擴展**：可加入型別檢查、靜態編譯等進階功能
4. **可執行**：有 test() 函式可直接驗證

### 5.3 技術深度與可讀性的平衡

編譯器是複雜的主題，需要：

- 用圖表簡化概念（如架構圖、流程圖）
- 提供大量程式碼範例
- 從歷史故事入手增加趣味性
- 將理論與實際應用連結

## 七、常見問題與解決

### 7.1 AI 忘記上下文

**現象**：AI 在長對話中忘記已完成的工作

**解決**：
1. 定期提供進度摘要
2. 明確指出下一步
3. 重新讀取關鍵檔案

### 7.2 編輯衝突

**現象**：檔案被多次修改後出現不一致

**解決**：
1. 使用 `read` 工具重新讀取檔案
2. 使用 `replaceAll` 確保一致性
3. 完成後立即驗證

### 7.3 程式碼複雜度

**現象**：直譯器實作涉及多個元件，容易出錯

**解決**：
1. 先設計好 AST 節點結構
2. 逐步實作 Lexer → Parser → Interpreter
3. 每個階段測試一次
4. 用 test() 函式驗證完整流程

## 八、最佳實踐總結

1. **分層建構**：先結構、後內容、最後除錯
2. **明確命名**：一致的命名慣例便於維護
3. **自動化測試**：每個程式檔案都要有 test()
4. **連結檢查**：修改後立即驗證所有引用
5. **進度摘要**：讓 AI 隨時知道專案狀態
6. **歷史脈絡**：複雜主題從故事開始，逐步深入
