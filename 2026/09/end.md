# 回顧與結語

## 2026 年 9 月號結語

### 本月主題回顧

九月是 Rust 系統程式設計的深度探索。本期雜誌從三個層次展現了 Rust 在系統領域的應用：

**底層：裸機與嵌入式**——Bump Allocator、Ring Buffer、MMIO 暫存器、no_std、嵌入式 Linux。這些是 Rust 誕生的初衷——取代 C/C++ 成為系統程式設計的首選語言。

**中層：作業系統與核心**——Tock OS、Linux 核心模組、執行緒同步、Atomics。Rust 正在從應用層向核心層滲透，這是其他語言從未達到的成就。

**上層：驗證與工具**——Miri、Kani、FFI、proc-macro。這些工具讓系統程式碼可以像高階語言一樣進行測試和驗證。

#### Rust 系統程式設計的本質

unsafe 是系統程式設計的核心。正如本期 clear_theme.md 中所說：**unsafe 不是關閉安全檢查，而是開啟低階操作**。Rust 的安全保證來自於型別系統和借用檢查器，而 unsafe 則是在這個安全框架內開了一個可控的後門。

本期的程式專案 mini-rt 展示了這個理念的具體實作：

```
BumpAllocator → GlobalAlloc trait → 全域配置器
FFI 函式     → 安全封裝       → Rust API
裸指標運算   → RingBuffer     → 安全抽象
volatile 存取 → MappedRegister → MMIO 抽象
```

每一層都是在不安全基礎上建立安全抽象。

#### 教育意義

本期特別強調「系統程式設計教育」的重要性。在 AI 時代，越來越多的開發者依賴高階框架和自動化工具，但系統程式設計的知識仍然不可取代：

1. **記憶體管理**是所有程式設計的基礎——了解 Bump Allocator 如何工作，有助於理解任何語言的記憶體模型
2. **並行與同步**是現代多核心 CPU 的核心挑戰——Atomics 和記憶體順序的知識在任何語言中都適用
3. **FFI 與跨語言操作**是整合利基系統的必備技能——現實世界中很少有用一種語言從頭到尾建置的系統
4. **驗證與測試**在系統程式設計中尤為重要——Miri 和 Kani 代表的工具化驗證是系統程式碼可靠性的關鍵

#### 本月技術亮點

**mini-rt** 是本月最具教育意義的程式專案。它用不到 200 行 Rust 程式碼展示了系統程式設計的四個核心概念。特別值得注意的是 bump allocator 只用了約 30 行程式碼就實作了一個「夠用」的記憶體配置器——這在 C 語言中同樣可以做到，但 Rust 的 `GlobalAlloc` trait 讓整合變得更加優雅。

**Rust 在核心中的進展**仍然是 2026 年最重要的技術趨勢。Linux 核心的 Rust 支援持續改進，更多驅動程式被重寫為 Rust。Tock OS 3.0 的生態系統也在成長，特別是在物聯網和感測器網路領域。

**unsafe Rust 的正確使用**是本期反覆強調的主題。10 條規則、Miri 驗證、Kani 形式化驗證——這些工具和規範讓 unsafe 程式碼的安全性和可靠性達到了前所未有的水準。

### 未來展望

**系統程式設計將向更多領域擴展**：從嵌入式到邊緣 AI，從核心模組到安全關鍵系統——Rust 系統程式設計的應用場景在不斷擴大。

**驗證工具將更加普及**：Miri 和 Kani 只是開始。隨著 Rust 生態的成熟，更多的驗證工具將被整合進 CI/CD 流程中。

**unsafe 最佳實踐將持續演化**：隨著 Rust 社群的成長，unsafe 的使用模式會更加規範化和工具化。

**系統程式設計教育將更加重要**：在 AI 自動化程式碼生成的時代，理解程式底層運作原理的開發者將更加稀缺和有價值。

### 給讀者的建議

如果您是 Rust 初學者，建議從本月的新知 part1 開始——它梳理了學習 Rust 系統程式設計的路徑。

如果您是進階使用者，以下資源值得深入研究：

1. **The Rustonomicon**——Rust 的 unsafe 程式碼聖經
2. **Miri**——Rust 的未定義行為偵測器
3. **Kani**——Rust 的形式化驗證工具
4. **Tock OS**——Rust 嵌入式作業系統的典範
5. **embedded-hal**——Rust 嵌入式硬體抽象層
