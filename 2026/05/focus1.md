# AI Agent 的起源：從專家系統到智慧體（1950s-2010s）

## 什麼是 AI Agent？

在深入歷史之前，我們需要先定義什麼是 AI Agent。一個 AI Agent 通常具備以下特徵：

1. **感知（Perception）**：能夠從環境中獲取資訊
2. **推理（Reasoning）**：能夠根據目標制定計畫
3. **行動（Action）**：能夠執行動作影響環境
4. **自主性（Autonomy）**：無需人類持續干預

這些概念早在 1950 年代 AI 誕生之初就已萌芽。

## 專家系統：第一個實用的 AI Agent

### 知識工程的誕生

1970 年代，AI 研究者發現：要讓電腦具備專業能力，最有效的方法不是模擬通用智慧，而是注入特定領域的專家知識。於是專家系統（Expert Systems）誕生了。

```
┌───────────────────────────────────────────────────┐
│                專家系統架構                          │
├───────────────────────────────────────────────────┤
│                                                     │
│  ┌──────────────┐    ┌──────────────┐            │
│  │  知識庫      │    │  推理引擎    │            │
│  │  (Rule Base) │◄──►│  (Inference  │            │
│  │              │    │   Engine)    │            │
│  └──────────────┘    └──────┬───────┘            │
│                              │                      │
│                     ┌───────┴───────┐            │
│                     │  使用者介面    │            │
│                     │   (UI)       │            │
│                     └───────────────┘            │
│                                                     │
│  IF 症狀 = 發燒 AND 咳嗽 THEN 診斷 = 感冒          │
│  IF 症狀 = 發燒 AND 頭痛 THEN 診斷 = 流感          │
│                                                     │
└───────────────────────────────────────────────────┘
```

### MYCIN：醫療診斷專家系統

MYCIN（1976 年，史丹佛大學）是史上最具影響力的專家系統之一。它能夠診斷血液感染並建議抗生素治療。

```
範例對話：
┌───────────────────────────────────────────┐
│ 系統：病人的症狀是什麼？                    │
│ 醫生：頭痛、發燒、頸部僵硬                  │
│ 系統：是否有細菌感染的證據？                │
│ 醫生：是的，腦脊液培養陽性                  │
│ 系統：可能的診斷是細菌性腦膜炎              │
│       建議使用抗生素 X（信心水準 0.85）     │
└───────────────────────────────────────────┘
```

MYCIN 的診斷準確率達到了 69%，超過了當時許多初級醫師的水準。更重要的是，MYCIN 引入了「解釋能力」——它能夠告訴使用者為什麼做出某個診斷，這是現代 AI Agent 仍在追求的目標。

### 專家系統的局限性

儘管專家系統在 1980 年代取得了商業成功（如 XCON 系統為 DEC 公司每年節省數千萬美元），但它們有以下根本限制：

1. **知識瓶頸**：規則需要人工編寫，費時費力
2. **缺乏常識**：只能處理預定義的場景
3. **無法學習**：不能從經驗中改進
4. **脆弱性**：遇到未覆蓋的情況就會完全失效

這些局限性導致了 1980 年代末的「AI 冬天」。

## 包容架構：行為主義的革命

### Rodney Brooks 的挑戰

1990 年代，MIT 的 Rodney Brooks 對傳統 AI 的「表徵－推理」範式發起了挑戰。他認為：

> 「智慧不需要明確的表徵和推理——智慧是透過與環境的互動湧現出來的。」

Brooks 提出了包容架構（Subsumption Architecture），這是一種將行為分層的 Agent 設計方法：

```
┌───────────────────────────────────────────────┐
│            包容架構（Subsumption）              │
├───────────────────────────────────────────────┤
│                                                 │
│  層級 3：探索行為                               │
│    │                                            │
│  層級 2：避障行為                               │
│    │                                            │
│  層級 1：隨機行走                               │
│    │                                            │
│  感知輸入 ──► 行為層 ──► 馬達輸出               │
│                                                 │
│  每層都是完整的「感知→行動」迴路                 │
│  高層可以抑制（Subsume）低層的行為              │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 六足機器人 Herbert

Brooks 的團隊製作了 Herbert——一個沒有中央控制系統、完全由包容架構驅動的六足機器人。Herbert 可以在辦公室中自主行走、避開障礙、尋找並抓取空罐頭。

Herbert 沒有任何「地圖」或「規劃器」——它的行為完全由簡單的感知－行動規則湧現出來。這與後來 LLM Agent 的「ReAct」模式有著驚人的相似之處：**不做長期規劃，而是即時根據感知選擇行動**。

## SOAR 與認知架構

### 統一的 Agent 理論

1987 年，Allen Newell 提出了「統一的認知理論」（Unified Theories of Cognition），並實作了 SOAR（State, Operator And Result）系統。SOAR 試圖建立一個通用的智慧體架構：

```
┌───────────────────────────────────────────────┐
│              SOAR 認知架構                      │
├───────────────────────────────────────────────┤
│                                                 │
│  ┌──────────┐    ┌──────────┐                  │
│  │ 長期記憶 │    │ 工作記憶 │                  │
│  │ (規則)   │◄──►│ (狀態)   │                  │
│  └──────────┘    └────┬─────┘                  │
│                        │                         │
│                  ┌─────┴─────┐                   │
│                  │  決策循環  │                   │
│                  │            │                   │
│                  │  1. 提議   │                   │
│                  │  2. 選擇   │                   │
│                  │  3. 執行   │                   │
│                  │  4. 評估   │                   │
│                  └───────────┘                   │
│                                                 │
└─────────────────────────────────────────────────┘
```

SOAR 的決策循環（提出 → 選擇 → 執行 → 評估）與現代 LLM Agent 的 ReAct 循環（思考 → 行動 → 觀察）在本質上是相同的架構模式。SOAR 還引入了「回溯」（Chunking）機制——將解決問題的經驗轉化為新的知識規則，這與 Agent 的「記憶」機制遙相呼應。

## 強化學習：從環境中學習

### Q-Learning 與 Deep Q-Network

2013 年，DeepMind 發表了 Deep Q-Network（DQN），將深度學習與強化學習結合，讓 Agent 能夠直接從原始像素中學習玩 Atari 遊戲。這是 AI Agent 史上的里程碑：

```python
# Deep Q-Learning 的核心概念（簡化）
class DQNAgent:
    def __init__(self, state_dim, action_dim):
        self.q_network = NeuralNetwork(state_dim, action_dim)
        self.memory = []  # 經驗回放緩衝區
    
    def act(self, state):
        # 根據 Q 值選擇行動
        q_values = self.q_network.predict(state)
        return argmax(q_values)
    
    def learn(self, state, action, reward, next_state):
        # 更新 Q 值
        target = reward + gamma * max(self.q_network.predict(next_state))
        self.q_network.update(state, action, target)
```

### AlphaGo：Agent 的巔峰之作

2016 年，AlphaGo 擊敗了圍棋世界冠軍李世石。AlphaGo 結合了：

1. **深度學習**：從人類對局中學習模式
2. **強化學習**：透過自我對弈持續改進
3. **蒙地卡羅樹搜索**：在決策時進行前瞻性規劃

AlphaGo 的架構——感知（棋盤狀態）→ 規劃（MCTS）→ 行動（落子）——正是現代 AI Agent 的完美示範。不同的是，AlphaGo 的感知和行動空間是高度結構化的，而 LLM Agent 需要處理的是無限開放的自然語言世界。

## 從歷史到現代 LLM Agent

### 歷史的螺旋上升

回顧 AI Agent 的歷史，我們可以看到一條清晰的螺旋上升路徑：

```
時代          Agent 類型        代表性系統      核心技術
──────        ──────────        ──────────      ────────
1970s         專家系統           MYCIN           規則推理
1980s         認知架構           SOAR            生產系統
1990s         行為式 Agent       Herbert         包容架構
2000s         強化學習 Agent     DQN             深度強化學習
2010s         規劃式 Agent       AlphaGo         MCTS + RL
2020s         LLM Agent          GPT-4/GPT-6     大語言模型
```

### 現代 LLM Agent 的獨特優勢

相比傳統 Agent，LLM Agent 帶來了三個關鍵突破：

1. **無需領域編程**：不需要手工編寫規則或設計獎勵函數
2. **自然語言介面**：人與 Agent 的溝通變得像對話一樣自然
3. **常識與推理**：預訓練模型內建了大量的世界知識和推理能力

### 傳統 Agent 對現代的啟示

儘管 LLM Agent 看起來與傳統 Agent 截然不同，但許多歷史經驗仍然適用：

- **專家系統的「解釋能力」**：啟發了現代 Agent 的思考鏈（CoT）
- **SOAR 的「回溯機制」**：對應於 Agent 的記憶壓縮與經驗學習
- **包容架構的「感知-行動直連」**：啟發了 ReAct 模式的設計
- **強化學習的「獎勵機制」**：對應於 RLHF 和偏好最佳化

## 結語

從 1970 年代的 MYCIN 到 2016 年的 AlphaGo，AI Agent 經歷了近半個世紀的演進。每一個時代的 Agent 都反映了當時 AI 技術的最高水準——從符號推理到神經網絡，從人工編碼規則到從資料中學習。

這些歷史經驗告訴我們：**真正的智慧體不僅需要知識（LLM），還需要結構（框架）、記憶（RAG）和行動能力（工具）**。這正是我們在後續文章中將深入探討的主題。
