# 主定理與遞迴樹

## 遞迴演算法的分析挑戰

分析遞迴演算法的時間複雜度不像分析迭代演算法那麼直覺。對於 `T(n) = 2T(n/2) + O(n)` 這樣的遞迴關係，我們需要系統性的工具來求解。

本文將深入介紹兩種最實用的工具：**主定理（Master Theorem）** 和 **遞迴樹法（Recursion Tree Method）**。

## 主定理

### 標準形式

對於形如 T(n) = a·T(n/b) + f(n) 的遞迴關係（其中 a ≥ 1，b > 1），主定理將解分為三種情況。

### 三種情況詳解

**情況 1：f(n) = O(n^(log_b(a) - ε))**

這意味著 f(n) 的增長慢於 n^(log_b(a))。解為 T(n) = Θ(n^(log_b(a)))。

範例：T(n) = 9T(n/3) + n
- a=9, b=3, log_b(a)=2, f(n)=n=O(n^(2-1))
- T(n) = Θ(n²)

**情況 2：f(n) = Θ(n^(log_b(a)))**

f(n) 與 n^(log_b(a)) 增長相同。解為 T(n) = Θ(n^(log_b(a)) · log n)。

範例：T(n) = 2T(n/2) + n
- a=2, b=2, log_b(a)=1, f(n)=n=Θ(n¹)
- T(n) = Θ(n log n)

**情況 3：f(n) = Ω(n^(log_b(a) + ε)) 且 a·f(n/b) ≤ c·f(n)**

f(n) 的增長快於 n^(log_b(a))。解為 T(n) = Θ(f(n))。

範例：T(n) = 2T(n/2) + n²
- a=2, b=2, log_b(a)=1, f(n)=n²=Ω(n^(1+1))
- 2·(n/2)² = n²/2 ≤ c·n²（取 c=1/2）
- T(n) = Θ(n²)

### 主定理不適用的情況

當遞迴關係不是標準形式，或三種情況的邊界條件不滿足時，需要使用遞迴樹法。

## 遞迴樹法

### 基本思路

將遞迴關係展開為樹狀結構，每一層對應一次遞迴調用。樹的根節點開銷為 f(n)，下一層有 a 個節點，每個開銷為 f(n/b)，以此類推。

### 範例：T(n) = 3T(n/4) + cn²

```
第 0 層：cn²                     (1 個節點)
第 1 層：c(n/4)² × 3             (3 個節點)
第 2 層：c(n/16)² × 9            (9 個節點)
第 k 層：c(n/4ᵏ)² × 3ᵏ          (3ᵏ 個節點)
```

各層開銷：第 k 層有 3ᵏ 個節點，每個大小為 n/4ᵏ，因此第 k 層總開銷為 3ᵏ × c(n/4ᵏ)² = cn²(3/16)ᵏ。

總開銷：T(n) = cn² × Σ_{k=0}^{log₄n} (3/16)ᵏ。

因為 3/16 < 1，這個幾何級數收斂到常數，所以 T(n) = O(n²)。

### 遞迴樹的步驟

1. 繪製遞迴樹：計算每層的節點數和子問題大小
2. 計算每層開銷：節點數 × 每個節點的 f(n)
3. 計算總開銷：加總所有層的開銷
4. 化簡：得到最終的時間複雜度

## 實際應用

### 合併排序的深入分析

T(n) = 2T(n/2) + cn

```
第 0 層：cn
第 1 層：c(n/2) × 2 = cn
第 2 層：c(n/4) × 4 = cn
...
第 k 層：c(n/2ᵏ) × 2ᵏ = cn
```

共有 log₂n 層，每層開銷 cn，總開銷 cn·log₂n = Θ(n log n)。

### 遞迴樹 vs 主定理

| 面向 | 主定理 | 遞迴樹 |
|------|-------|-------|
| 使用難度 | 容易（套公式） | 需要計算 |
| 適用範圍 | 僅限標準形式 | 更靈活 |
| 直覺性 | 較低 | 較高（可視化） |
| 精確度 | 精確的 Θ | 需要化簡 |

## 其他分析技巧

### 代入法（Substitution Method）

先猜出答案，再用數學歸納法證明。這需要經驗和直覺。

### 變數變換法

對某些非標準形式的遞迴關係，可先進行變數變換，轉為標準形式後再求解。

## 總結

主定理和遞迴樹法是分析遞迴演算法時間複雜度的兩個核心工具。主定理快速但有限制，遞迴樹靈活但需要更多計算。掌握這兩種方法，你就能夠分析絕大多數常見遞迴演算法的時間複雜度。
