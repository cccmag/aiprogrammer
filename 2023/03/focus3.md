# 平衡樹：AVL 與紅黑樹

## 為什麼需要平衡樹？

二元搜尋樹在插入有序資料時會退化為鏈結串列，操作時間從 O(log n) 惡化為 O(n)。平衡樹透過旋轉操作維持樹的高度在 O(log n)，確保所有操作都能維持對數級別的高效率。平衡樹是資料庫索引、作業系統排程、編譯器等高效能系統中不可或缺的元件。

## AVL 樹

AVL 樹由 Georgy Adelson-Velsky 與 Evgenii Landis 在 1962 年提出，是最早被發明的自平衡二元搜尋樹。它要求每個節點的左右子樹高度差（平衡因子，Balance Factor）不超過 1，即 |BF| ≤ 1。

當插入或刪除破壞平衡時，執行以下四種旋轉之一來恢復平衡：

- **右旋（LL）**：左子樹高 2，新節點插在左子樹左側。向右旋轉修正。
- **左旋（RR）**：右子樹高 2，新節點插在右子樹右側。向左旋轉修正。
- **左右雙旋（LR）**：左子樹高 2，新節點插在左子樹右側。先左旋再右旋。
- **右左雙旋（RL）**：右子樹高 2，新節點插在右子樹左側。先右旋再左旋。

## 紅黑樹

紅黑樹是一種近似平衡的二元搜尋樹，每個節點有紅色或黑色標記，滿足五個性質：每個節點是紅色或黑色、根節點黑色、葉節點 NIL 黑色、紅色節點的子節點必須黑色、從根到任一葉節點路徑上黑色節點數量相同。

紅黑樹的平衡條件比 AVL 寬鬆，插入刪除時的旋轉次數較少，適用於寫入頻繁的場景。C++ STL 的 map 與 Java 的 TreeMap 均使用紅黑樹實作。

## 比較總結

| 特性 | AVL 樹 | 紅黑樹 |
|------|--------|--------|
| 平衡程度 | 嚴格（高度差 ≤ 1） | 近似（最長 ≤ 2 倍最短） |
| 搜尋效率 | 較高 | 稍低 |
| 插入/刪除效率 | 較低（旋轉較多） | 較高（旋轉較少） |
| 常見應用 | 資料庫索引 | C++ STL map, Java TreeMap |
