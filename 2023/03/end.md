# 結語

## 本期回顧

本月的 AI 程式人雜誌以「資料結構理論」為主題，從最基本的 ADT 概念出發，一路涵蓋樹、平衡樹、堆積、圖論、圖走訪與雜湊等經典主題。搭配七篇焦點文章、十篇深入的技術文章與完整的 Python 程式碼實作，希望能幫助讀者建立紮實的資料結構基礎。

資料結構是電腦科學中少數「學會了永遠用得上」的領域。無論你寫的是網頁應用、系統軟體、嵌入式程式或機器學習模型，理解背後使用的資料結構都能讓你寫出更有效率的程式碼。例如，在處理大量資料時選擇恰當的索引結構（B-Tree 或雜湊表），可能會對系統效能產生數量級的影響。在設計 API 時選擇正確的資料結構（list、set 或 dict），可以大幅提升查詢效率與程式碼可讀性。

我們特別強調理論與實作的結合。每一篇焦點文章都對應到 `_code/ds_theory.py` 中的程式碼範例，讓讀者可以在閱讀理論的同時，實際執行並觀察資料結構的行為。這種「學中做、做中學」的方式，往往能帶來更深的理解。建議讀者在閱讀完理論後，實際執行 `python3 ds_theory.py` 來觀察 BST 中序走訪的排序結果、AVL 的自動平衡機制、BFS 與 DFS 的走訪順序差異，以及雜湊表的鏈結法碰撞處理。
