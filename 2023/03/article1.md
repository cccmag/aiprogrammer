# 鏈結串列變體

## 單向鏈結串列

單向鏈結串列（Singly Linked List）是最基本的鏈結串列形式，每個節點包含資料與指向下一個節點的 next 指標。頭節點（head）指向第一個元素，尾節點的 next 為 None。這種結構簡單且記憶體開銷低，但只能從頭到尾單向走訪。

**優點**：插入與刪除操作在已知位置時為 O(1)，不需要像陣列那樣搬移元素。記憶體動態配置，不需要預先配置連續空間。當資料量不確定時非常適合使用。

**缺點**：無法反向走訪。要存取第 i 個元素需要從頭開始遍歷，時間為 O(n)。每個節點需要額外的指標記憶體開銷。

## 雙向鏈結串列

雙向鏈結串列（Doubly Linked List）的每個節點包含 next 與 prev 兩個指標，分別指向下一個與前一個節點。優點是可以雙向走訪，刪除操作更靈活（不需要知道前一個節點）。缺點是佔用更多記憶體（每個節點多一個指標），且插入刪除時需要更新更多指標。雙向鏈結串列常用於實作 LRU Cache。

## 環狀鏈結串列

環狀鏈結串列（Circular Linked List）將尾節點的 next 指向頭節點，形成一個環。可以進一步分為單向環狀與雙向環狀。適合需要循環走訪的場景，例如作業系統的輪詢排程（Round Robin）以及 Josephus 問題的求解。

## 實作比較

| 特性 | 單向 | 雙向 | 環狀 |
|------|------|------|------|
| 記憶體開銷 | 低（1 指標/節點） | 中（2 指標/節點） | 低 |
| 反向走訪 | 不支援 | 支援 | 有限（繞一圈） |
| 頭部插入 | O(1) | O(1) | O(1) |
| 刪除已知節點 | O(n) | O(1) | O(n) |
| 典型應用 | 堆疊、佇列 | LRU Cache | 輪詢排程 |

## 跳躍串列

跳躍串列（Skip List）是一種基於多層級鏈結串列的機率性資料結構，由 William Pugh 在 1989 年提出。透過在更高層級建立「捷徑」，達到平均 O(log n) 的搜尋效率，最差情況 O(n)。實作比平衡樹簡單，常用於 Redis 的有序集合（Sorted Set）。跳躍串列的核心思想是空間換取時間，透過多層索引來加速搜尋。
