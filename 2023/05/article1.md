# DFA 設計與最小化

## 從問題到 DFA

設計 DFA 的第一步是理解問題的語言 L。我們需要明確哪些字串屬於 L，哪些不屬於。DFA 的狀態應該捕捉「到目前為止所需的記憶」。

### 通用設計策略

1. **確定記憶需求**：問自己，為了做出正確的接受/拒絕決定，必須記住什麼？
2. **定義狀態**：每個狀態代表一種記憶狀態
3. **填寫轉移**：對每個狀態和每個輸入符號，確定下一個狀態
4. **標記接受狀態**：哪些記憶狀態對應於「目前為止的字串是可接受的」

### 範例：辨識包含 "101" 的二進位字串

這個語言需要記住目前匹配到了 "101" 的哪個前綴。我們可以用五個狀態：

- q0：尚未開始匹配，或匹配失敗
- q1：剛剛匹配到 "1"
- q2：剛剛匹配到 "10"
- q3：剛剛匹配到 "101"（接受狀態）

轉移函數：
```
q0 --0--> q0,  q0 --1--> q1
q1 --0--> q2,  q1 --1--> q1
q2 --0--> q0,  q2 --1--> q3
q3 --0--> q3,  q3 --1--> q3
```

## DFA 最小化

同一個語言可以用不同的 DFA 來辨識。最小化 DFA 的目的是找到具有最少狀態數的等價 DFA。

### Myhill-Nerode 定理

這個定理為 DFA 最小化提供了理論基礎：兩個狀態是等價的，若且唯若從它們出發，對於所有後續字串，接受/拒絕行為完全相同。

### 演算法：填表法 (Table-Filling Algorithm)

1. 標記所有可區分的狀態對（一個接受、一個非接受）
2. 重複：如果存在一對狀態 (p, q) 尚未標記，但對於某個輸入符號 a，狀態對 (δ(p,a), δ(q,a)) 已被標記，則標記 (p, q)
3. 重複直到沒有新的標記
4. 合併所有未標記的狀態對

### 範例

考慮辨識以 0 結尾的字串的 DFA，有 q0, q1, q2 三個狀態。透過最小化，我們發現 q1 和 q2 的行為相同，可以合併為一個狀態，最終得到 2-狀態的最簡 DFA。

### 為什麼需要最小化？

- **硬體實作成本**：每個狀態需要對應的邏輯電路
- **理解性**：最小化 DFA 更容易被人理解
- **正則表達式推導**：從最小化 DFA 推導正則表達式更簡單

## 實際應用

DFA 設計在以下場景中有直接應用：
- **通訊協定狀態機**：TCP、HTTP 等協定的狀態轉移
- **控制器設計**：嵌入式系統中的有限狀態機
- **文字處理**：字串匹配演算法 (KMP 演算法等價於 DFA)
