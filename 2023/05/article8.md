# Turing Machine 程式設計

## 從高階語言到 Turing Machine

Turing Machine (TM) 是最簡單的通用計算模型，但用 TM 程式設計充滿挑戰。TM 沒有變數、沒有副程式、沒有高階控制流程——只有磁帶、讀寫頭和狀態轉移。

然而，任何可以被直觀計算的函數都可以用 TM 實現。理解 TM 程式設計，有助於深刻理解計算的本質。

## TM 程式設計的基本模式

### 模式 1：磁帶掃描與標記

TM 經常需要在磁帶上來回移動以找到特定位置。一個常見技巧是用特殊符號來標記已處理的磁帶位置。

```
# 標記第一個 a 為 X
狀態 q0：讀到 a → 寫入 X，向右移動，轉到 q1
         讀到 b 或空白 → 拒絕或繼續
```

### 模式 2：複製與比對

對於需要對比的任務（如檢查 {a^n b^n c^n}），TM 可以來回掃描磁帶，逐步標記和比對。

## 範例 1：二進位加法

輸入格式：二進位數字以 $ 分隔，如 "1011+110="。

```python
tm_transition = {
    # 將讀寫頭移到最右邊
    ('q0', '0'): ('q0', '0', 'R'),
    ('q0', '1'): ('q0', '1', 'R'),
    ('q0', '+'): ('q0', '+', 'R'),
    ('q0', '='): ('q1', '=', 'L'),
    # 開始加法
    ('q1', '0'): ('q2', '1', 'L'),  # 0 + 1 = 1，不進位
    ('q1', '1'): ('q3', '0', 'L'),  # 1 + 1 = 0，進位
    ('q2', '0'): ('q2', '0', 'L'),   # 不進位，繼續左移
    ('q2', '1'): ('q2', '1', 'L'),
    ('q2', '+'): ('q4', '+', 'R'),   # 遇到 +，加法完成
    # 進位傳播
    ('q3', '0'): ('q2', '1', 'L'),  # 0 + 進位 = 1，不進位
    ('q3', '1'): ('q3', '0', 'L'),  # 1 + 進位 = 0，繼續進位
    ('q3', '+'): ('q5', '+', 'L'),  # 進位超出位數
    ('q5', '_'): ('q4', '1', 'R'),  # 在最左邊加 1
}
```

## 範例 2：字串反轉

反轉輸入字串 "abc" 為 "cba"：

1. 找到最後一個字元，將其複製到輸出區域
2. 標記已複製的字元
3. 重複直到所有字元都被處理
4. 清理標記符號

這個任務展示了 TM 如何在磁帶上管理多個資料區域。

## 範例 3：質數判定

輸入：連續的 a（如 "aaaaaaa" 表示數字 7）
任務：判斷 a 的數量是否為質數

策略：嘗試將 a 的數量從 2 到 n-1 進行整除檢查。TM 需要：
1. 製造一個除數 d（從 2 開始）
2. 檢查 a 的數量是否能被 d 整除
3. 如果能整除且結果不為 1，則拒絕（非質數）
4. 遞增 d，重複直到 d = n

## TM 的侷限性

儘管 TM 可以計算任何可計算函數，但效率可能極低。質數判定的 TM 可能需要數千步來處理一個輸入，而現代計算機可以在微秒內完成。

這提醒我們：TM 是關於**可計算性**的模型，而不是**效率**的模型。效率問題屬於計算複雜度理論的範疇。
