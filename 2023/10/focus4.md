# 偏差-變異權衡

## 誤差的三個來源

機器學習的預測誤差可以分解為三個來源：**偏差**（Bias）、**變異**（Variance）和**不可約雜訊**（Irreducible Noise）。

```
期望誤差 = 偏差² + 變異 + 雜訊
```

## 數學推導

考慮回歸問題：y = f(x) + ε，其中 E[ε] = 0，Var[ε] = σ²。

設 ĥ(x) 是學習演算法在訓練集 S 上的預測。則在固定點 x 處的期望平方誤差：

```
E[(y - ĥ)²] = E[(f + ε - ĥ)²]
             = E[(f - ĥ)²] + σ²
             = E[ĥ] - f)² + E[(ĥ - E[ĥ])²] + σ²
             = Bias² + Variance + σ²
```

### 偏差（Bias）

Bias = E[ĥ(x)] - f(x)

偏差衡量模型對真實函數的**系統性偏離**。高 Bias 意味著模型太簡單，無法捕捉真實模式——這就是**欠擬合**（Underfitting）。

**範例**：用線性模型擬合二次函數，無論有多少資料，線性模型都無法準確逼近曲線。

### 變異（Variance）

Variance = E[(ĥ(x) - E[ĥ(x)])²]

變異衡量模型對訓練資料的**敏感度**。高 Variance 意味著模型在不同訓練集上的預測差異很大——這就是**過擬合**（Overfitting）。

**範例**：高階多項式擬合少量資料點，稍微改變訓練資料就會導致巨大的預測變化。

### 不可約雜訊

σ² 是資料本身包含的雜訊，無論模型多好都無法消除。這是預測誤差的下界。

## 權衡曲線

偏差和變異之間存在根本性的權衡：

```
模型複雜度增加 → 偏差下降，變異上升
模型複雜度降低 → 偏差上升，變異下降
```

**最佳複雜度**位於偏差下降和變異上升的交叉點：

```
總誤差 = 偏差² + 變異 + σ²
最佳點 = argmin(偏差² + 變異)
```

## 偏差-變異分解的實際計算

在本期的程式實作中，我們透過蒙特卡羅方法直接計算偏差和變異：

```python
bv = bias_variance_decomposition(degree=3)
# bias_sq: 0.0035, variance: 0.0074
```

步驟：
1. 定義真實函數 f(x) = sin(x)
2. 生成 2000 個不同的訓練集，每個加上高斯雜訊
3. 對每個訓練集擬合三次多項式
4. 計算所有預測的平均（估計 E[ĥ]）
5. 偏差² = mean((avg_prediction - f(x))²)
6. 變異 = mean(var(all_predictions))

### 不同 degree 的比較

| 次數 | 偏差² | 變異 | 總和 |
|------|-------|------|------|
| 1（欠擬合） | 高 | 低 | 高 |
| 3（最佳） | 0.0035 | 0.0074 | 0.0108 |
| 15（過擬合） | 低 | 高 | 高 |

一次多項式偏差高但變異低，十五次多項式偏差低但變異高，三次多項式找到最佳平衡。

## 偏差-變異的直觀理解

**偏差**：射箭時瞄準點偏離靶心——每次都偏同一方向
**變異**：射箭時手在抖——每次落點都不一樣

一個好的模型需要同時控制偏差和變異：

- **高偏差低變異**：箭全部集中在一個錯誤位置（線性回歸）
- **低偏差高變異**：箭分散在靶心周圍（高階多項式）
- **低偏差低變異**：箭全部集中在靶心（理想模型）

## 控制偏差-變異的方法

| 問題 | 方法 | 效果 |
|------|------|------|
| 高偏差 | 增加特徵、使用更複雜模型 | 增加模型容量 |
| 高變異 | 正則化、減少特徵、增加資料 | 限制模型複雜度 |
| 兩者 | 交叉驗證、集成方法 | 尋找最佳平衡 |

## 小結

偏差-變異權衡是學習理論中最直觀也最重要的概念：

| 概念 | 定義 | 控制的關鍵 |
|------|------|-----------|
| 偏差² | E[(E[ĥ] - f)²] | 模型容量 |
| 變異 | E[(ĥ - E[ĥ])²] | 正則化強度 |
| 雜訊 | σ² | 資料品質 |
