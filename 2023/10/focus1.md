# 學習問題的形式化

## 什麼是「學習」？

在回答「機器如何學習」之前，我們首先需要給出「學習」的精確定義。Tom Mitchell 的定義最為經典：

> 一個電腦程式從經驗 E 中學習任務 T，並用表現度量 P 來評估，如果它在 T 上的表現（根據 P）隨著經驗 E 的增加而改善。

這個定義雖然直觀，但在理論分析中還不夠精確。統計學習理論提供了一個更數學化的框架。

## 三要素

學習問題的三個核心要素：

### 1. 輸入空間與輸出空間

設輸入空間為 X，輸出空間為 Y。分類問題中 Y = {+1, -1}，回歸問題中 Y = ℝ。一個學習問題由 X 和 Y 上的聯合分布 P 描述。

### 2. 假說空間

假說空間 H 是從 X 到 Y 的所有可能函數的集合。學習演算法的目標是從 H 中選擇一個函數 h 來近似真實映射。

### 3. 損失函數

損失函數 L: Y × Y → ℝ 度量預測值與真實值的差異：

- 0-1 損失：L(y, ŷ) = 1{y ≠ ŷ}
- 平方損失：L(y, ŷ) = (y - ŷ)²
- 對數損失：L(y, ŷ) = -log p(y|ŷ)

## 風險與經驗風險

**真實風險**（True Risk）是在整個資料分布上的期望損失：

```
R(h) = E_{(x,y)~P}[L(y, h(x))]
```

這是最理想但無法直接計算的量——因為我們不知道真實分布 P。

**經驗風險**（Empirical Risk）是在訓練樣本上的平均損失：

```
R̂(h) = (1/n) Σ_i L(y_i, h(x_i))
```

這是我們可以計算的量。學習理論的核心問題是：**經驗風險何時接近真實風險？**

## ERM 原則

經驗風險最小化（ERM）是最基本的學習原則：

```
ĥ = argmin_{h∈H} R̂(h)
```

即選擇在訓練資料上表現最好的假說。

### ERM 的陷阱

ERM 不一定成功。考慮一個極端情況：假說空間 H 包含所有可能的函數。那麼總存在一個函數完美擬合訓練資料（經驗風險為 0），但在新資料上表現極差。

這就是過擬合（Overfitting）的本質——**如果 H 過於複雜，ERM 會記住資料而非學習模式**。

## 學習的一致性

一個學習演算法被稱為**一致**（Consistent）的，如果當樣本數 n → ∞ 時，其風險收斂到最小可能風險：

```
R(ĥ_n) → inf_{h∈H} R(h)  (in probability)
```

一致性是學習演算法有效性的最低要求。

### 可學習性

一個問題是**可學習**（Learnable）的，如果存在一個一致且收斂速率可控制的演算法。這裡「可控制」意味著我們可以（以高機率）保證：

```
R(ĥ_n) ≤ inf_{h∈H} R(h) + ε
```

其中 ε 隨 n 增加以已知速率趨近於 0。

## 形式化的學習問題

綜合以上討論，一個形式化的學習問題由以下九元組定義：

```
L = (X, Y, P, H, L, S, A, n, ε)
```

其中 S = {(x_i,y_i)}_{i=1..n} 是訓練樣本，A 是學習演算法。我們的目標是證明：

```
P[R(A(S)) ≤ inf R(h) + ε] ≥ 1 - δ
```

這正是下一期 PAC 學習框架的出發點。

## 小結

學習問題的形式化為整個機器學習理論奠定了基礎。理解「什麼是學習」是理解所有後續理論的前提。

| 概念 | 定義 | 備註 |
|------|------|------|
| 真實風險 | R(h) = E[L(y, h(x))] | 無法計算 |
| 經驗風險 | R̂(h) = (1/n) Σ L(y_i, h(x_i)) | 可計算 |
| ERM | ĥ = argmin R̂(h) | 最簡單原則 |
| 一致性 | R(ĥ_n) → inf R(h) | 最低要求 |
