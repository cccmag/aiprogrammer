# os_demo.py：作業系統核心機制綜合展示

## 概述

[os_demo.py](_code/os_demo.py) 是一個用 Python 實作的作業系統原理綜合展示程式。它模擬了作業系統的四個核心機制：

1. **行程狀態轉換** ── 狀態機模型
2. **CPU 排程演算法** ── FCFS、SJF、RR
3. **頁面置換演算法** ── FIFO、LRU、OPT
4. **死結偵測** ── 銀行家演算法

## 展示內容

### 1. 行程狀態轉換

```
New ──[建立完成]──> Ready ──[排程器分派]──> Running
Running ──[I/O 請求]──> Waiting
Waiting ──[I/O 完成]──> Ready
Running ──[時間片到期]──> Ready
Running ──[執行完畢]──> Terminated
```

### 2. CPU 排程演算法

以四個行程（P1-P4，服務時間 5/3/8/2）演示三種排程策略：

- **FCFS**：按照到達順序執行，平均等待時間由先抵達決定的短行程可能被長行程阻塞
- **SJF**：每次選擇最短的行程執行，平均等待時間最短，但可能導致長行程饑餓
- **RR**：每個行程獲得固定的時間片（2 單位），輪流執行，互動性最好

### 3. 頁面置換演算法

使用經典的參照序列 `[7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2, 1, 2, 0, 1, 7]`，在 3 個框架下比較三種演算法：

- **FIFO**：最先載入的頁面被置換，簡單但可能置換常用頁面
- **LRU**：最久未使用的頁面被置換，利用局部性原理
- **OPT**：未來最晚使用的頁面被置換，理論最優但無法實際實作

### 4. 死結偵測（銀行家演算法）

以 5 個行程、3 類資源的場景，展示如何判斷系統是否處於安全狀態：

```
行程: P0, P1, P2, P3, P4
資源: A (10), B (5), C (7)
可用: [3, 3, 2]
```

## 核心程式碼

```python
def bankers(processes, alloc, max_need, avail):
    n = len(processes)
    m = len(avail)
    need = [[max_need[i][j] - alloc[i][j] for j in range(m)] for i in range(n)]
    work = avail[:]
    finish = [False] * n
    seq = []
    while len(seq) < n:
        found = False
        for i in range(n):
            if not finish[i] and all(need[i][j] <= work[j] for j in range(m)):
                for j in range(m):
                    work[j] += alloc[i][j]
                finish[i] = True
                seq.append(processes[i])
                found = True
        if not found:
            return False, seq
    return True, seq
```

銀行家演算法的核心是尋找一個安全序列：每個行程的需求都能被當前可用資源滿足，當行程完成後釋放資源，最終所有行程都能順利完成。

## 關鍵洞察

### 排程演算法的權衡

- **FCFS**：公平但效率低，短行程被長行程阻塞
- **SJF**：平均等待時間最小，但可能導致飢餓
- **RR**：互動性最好，但上下文切換開銷大

### 頁面置換的局部性

LRU 利用時間局部性（近期使用的頁面很可能近期再次使用），在實際系統中表現接近 OPT。FIFO 的缺點是可能置換頻繁使用的頁面，導致 Belady 異常。

### 死結的預防比偵測更重要

銀行家演算法雖然可以在 O(n²m) 時間內偵測死結，但實際上大多數作業系統採用死結預防策略（如破壞循環等待條件），而非死結偵測與恢復。
