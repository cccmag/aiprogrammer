# 分頁與分段

## 記憶體管理的兩大方案

分頁（paging）和分段（segmentation）是兩種不同的記憶體管理方案。它們解決的核心問題相同——將程式的邏輯位址轉換為實體位址——但採用的策略完全不同。

## 分頁（Paging）

### 基本概念

- **頁面（Page）**：邏輯位址空間被劃分為固定大小的區塊（通常 4KB）
- **框架（Frame）**：實體記憶體被劃分為相同大小的區塊
- **頁表（Page Table）**：記錄邏輯頁面到實體框架的映射

```
邏輯位址 (32 位元)：
┌────────────┬────────────┐
│   頁號 (20) │  頁內偏移 (12) │
└────────────┴────────────┘

頁表：
頁號 0 → 框架號 5
頁號 1 → 框架號 8
頁號 2 → 框架號 3
頁號 3 → 框架號 12
```

### 分頁的優點

1. **無外部碎片**：所有記憶體區塊大小相同，不存在無法利用的小碎片
2. **簡化分配**：只需找到足夠的空閒框架
3. **支援虛擬記憶體**：頁面可以換入換出
4. **共享**：多個行程共享同一個實體頁面（如 shared library）

### 分頁的缺點

1. **內部碎片**：最後一頁可能沒有完全填滿（平均浪費半頁）
2. **頁表開銷**：32 位元系統需要約 4MB 頁表（每個行程），64 位元更驚人
3. **額外記憶體存取**：每次記憶體存取需要兩次存取（頁表 + 實際資料）——除非 TLB 命中

### 多層頁表

為了解決頁表過大的問題，現代系統使用多層頁表：

```
x86-64 4 層頁表：
PML4 (9 bits) → PDPT (9 bits) → PD (9 bits) → PT (9 bits) → 偏移 (12 bits)

總共 48 位元位址空間 (256 TB)
```

只有實際使用的頁面才會分配頁表條目——64 位元進程即使位址空間巨大，頁表也不會爆炸。

### 頁表結構

```
頁表項 (PTE)：
┌────┬───┬───┬───┬───┬────────┐
│框架│ G │ S │ A │ D │ 保護位 │
│號  │    │   │   │   │        │
└────┴───┴───┴───┴───┴────────┘

- 框架號：實體框架的起始位址
- G：全域頁面（不從 TLB 中清除）
- S：頁面大小（4KB 或 2MB/1GB）
- A：存取位（Accessed，由硬體設置）
- D：骯髒位（Dirty，頁面被寫入過）
- 保護位：讀/寫/執行權限
```

## 分段（Segmentation）

### 基本概念

分段將程式的位址空間劃分為邏輯上的「段」——程式碼段、資料段、堆疊段、堆積段等。每個段有獨立的基址和長度。

```
邏輯位址 (分段)：
┌────────────┬────────────┐
│   段號     │  段內偏移   │
└────────────┴────────────┘

段表：
段號 0 (程式碼): 基址=0x400000, 長度=0x5000, 保護=R+X
段號 1 (資料):   基址=0x800000, 長度=0x3000, 保護=R+W
段號 2 (堆疊):   基址=0x700000, 長度=0x1000, 保護=R+W
```

### 分段的優點

1. **符合程式邏輯**：不同類型的資料自然分離
2. **獨立保護**：程式碼段可設為唯讀執行，防止修改
3. **共享方便**：可以共享整個段（如動態函式庫的程式碼段）
4. **動態增長**：堆疊和堆積段可以獨立增長

### 分段的缺點

1. **外部碎片**：不同大小的段在載入和卸載後會產生碎片
2. **段大小限制**：單個段不能超過實體記憶體大小
3. **複雜分配**：需要在記憶體中找到連續的空間來放置段

## 分段分頁結合

x86 架構實際上是分段和分頁的結合：

```
邏輯位址 → 分段單元 → 線性位址 → 分頁單元 → 實體位址
```

- **分段單元**：將邏輯位址（段:偏移）轉換為 32/64 位元線性位址
- **分頁單元**：將線性位址轉換為實體位址

在現代 Linux 系統中：
- 代碼段、資料段等均設定基址為 0，長度為 4GB
- 分段單元實際上被「繞過」——所有效果等於直接使用分頁
- 這種模式稱為「平坦記憶體模型」（flat memory model）

## 比較總結

| 特性 | 分頁 | 分段 |
|------|------|------|
| 區塊大小 | 固定（4KB） | 可變 |
| 碎片類型 | 內部碎片 | 外部碎片 |
| 邏輯對應 | 無（不考慮程式結構） | 按邏輯區塊分割 |
| 保護粒度 | 頁面層級 | 段層級 |
| 共享粒度 | 頁面層級 | 段層級 |
| 虛擬記憶體 | 天然支援 | 不易實作 |
