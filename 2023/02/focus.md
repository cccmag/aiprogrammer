# 本期焦點

## 作業系統原理：從行程管理到檔案系統

### 引言

作業系統是電腦系統中最基礎也最重要的軟體層。它管理硬體資源、提供應用程式執行環境、並在複雜的並行場景中確保系統的正確性與效率。

本期雜誌將完整覆蓋作業系統的核心主題：

- **行程與執行緒管理**：如何建立作業系統對執行單元的抽象
- **CPU 排程**：在多工環境中如何公平且高效地分配處理器
- **同步與死結**：並行程式設計的核心挑戰
- **記憶體管理**：從實體記憶體到虛擬記憶體的抽象化過程
- **檔案系統與 I/O**：如何持久化儲存資料

這七個主題構成了一個完整的作業系統知識體系。

## 濃縮回顧

### 行程的生命週期

行程從建立到終止會經歷多個狀態：New → Ready → Running → Waiting → Ready → Running → Terminated。這個狀態轉換模型是所有作業系統課程的核心內容。

### 排程演算法的演進

從簡單的 FCFS 到考慮優先權的多層回饋佇列，CPU 排程演算法在公平性與效率之間不斷權衡。現代作業系統通常使用 O(1) 或 CFS 排程器，能在數千個行程間做出即時決策。

### 虛擬記憶體的威力

虛擬記憶體讓每個行程擁有獨立的位址空間，並允許程式使用超過實體記憶體的容量。這背後的關鍵機制是分頁（paging）和頁面置換演算法。

### 同步的藝術

在並行程式設計中，競爭條件是最常見的錯誤來源。互斥鎖、號誌和管程是解決同步問題的三種經典工具，每一種都有適用的場景。
