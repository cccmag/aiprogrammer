# 文章集錦

## 作業系統原理專輯

### 程式相關（10 篇）

#### 1. [系統呼叫與中斷](article1.md)

系統呼叫（system call）是用戶態程式請求核心服務的介面，中斷（interrupt）是硬體通知 CPU 事件發生的機制。本文深入探討系統呼叫的實現原理——從用戶態到核心態的切換、中斷向量表、中斷優先權，以及 Linux 和 Windows 的系統呼叫差異。

#### 2. [行程 vs 執行緒](article2.md)

行程與執行緒是作業系統中最基礎的兩個概念。行程是資源分配的最小單位，執行緒是 CPU 排程的最小單位。本文從建立開銷、上下文切換、通訊方式、隔離性等角度比較兩者，並探討協程與纖程的定位。

#### 3. [排程演算法比較：FCFS、SJF、RR](article3.md)

三種經典排程演算法各有優缺：FCFS 簡單但護理效應嚴重；SJF 平均等待時間最優但預估服務時間困難；RR 互動性好但上下文切換開銷大。本文包含完整的 Python 實作和實驗數據。

#### 4. [競爭條件與互斥鎖](article4.md)

競爭條件是並行程式設計中最常見的錯誤。本文從一個簡單的計數器範例出發，展示競爭條件的產生機制，並逐步引入互斥鎖、原子操作、自旋鎖和 RCU 等解決方案。

#### 5. [號誌與管程](article5.md)

Dijkstra 的號誌和 Hoare 的管程是並行程式設計的兩大理論基石。本文比較兩者，展示生產者-消費者和讀寫者問題的兩種解法，並探討現代語言中的同步原語選擇。

#### 6. [死結偵測與避免](article6.md)

死結是並行系統中兩個以上行程互相等待對方釋放資源的狀態。本文詳細介紹死結的四個必要條件、銀行家演算法的 Python 實作、死結偵測的資源分配圖方法，以及實際系統中的死結預防策略。

#### 7. [分頁與分段](article7.md)

分頁和分段是兩種不同的記憶體管理方案。分頁避免外部碎片但可能產生內部碎片；分段符合程式邏輯但無法避免外部碎片。本文比較兩者的優缺，並探討現代系統（x86-64）如何結合使用兩者。

#### 8. [頁面置換演算法](article8.md)

當虛擬記憶體中的頁面需要被換出時，頁面置換演算法決定哪些頁面被犧牲。本文比較 FIFO、LRU、OPT、Clock 和 LFU 等演算法，分析 Belady 異常的成因，並提供 Python 模擬器比較其效能。

#### 9. [檔案系統實作](article9.md)

從磁碟區塊到檔案抽象是一段漫長的旅程。本文剖析 inode 結構、目錄實現、區塊分配策略、日誌機制，並比較 ext4、NTFS、Btrfs 和 ZFS 的實作差異。

#### 10. [I/O 與設備驅動](article10.md)

I/O 是作業系統與外部設備溝通的橋樑。本文探討 I/O 控制方式（程式控制、中斷驅動、DMA）、設備驅動程式架構、Linux 字元設備和區塊設備的區別、以及現代儲存設備（NVMe）的 I/O 模型。
