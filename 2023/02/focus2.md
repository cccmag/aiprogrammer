# 行程管理與狀態轉換（1960s-2023）

## 什麼是行程？

行程（process）是電腦程式的執行實體。它不僅包含程式碼，還包含執行上下文——程式計數器、暫存器內容、記憶體位址空間、開啟的檔案等。

行程是作業系統進行資源分配的基本單位。

## 行程控制區塊（PCB）

作業系統為每個行程維護一個行程控制區塊（Process Control Block, PCB）——這是作業系統中最重要的資料結構之一。PCB 包含：

- **行程 ID**（PID）：唯一識別碼
- **狀態**：New、Ready、Running、Waiting、Terminated
- **程式計數器**：下一條指令的位址
- **CPU 暫存器**：通用暫存器、堆疊指標、狀態暫存器
- **排程資訊**：優先權、已用 CPU 時間、等待時間
- **記憶體資訊**：頁表、段表、界限暫存器
- **I/O 狀態**：已開啟的檔案描述符、I/O 請求佇列
- **帳號資訊**：CPU 使用統計、行程擁有者

在 Linux 核心中，PCB 對應於 `task_struct` 結構，這是一個包含數百個欄位的複雜資料結構。

## 上下文切換

當 CPU 從一個行程切換到另一個行程時，必須執行上下文切換（context switch）：

1. 保存當前行程的狀態到它的 PCB
2. 載入新行程的 PCB 內容
3. 更新記憶體管理單元（MMU）和頁表
4. 恢復新行程的暫存器內容
5. 更新程式計數器，開始執行新行程

上下文切換是純粹的開銷——它不做任何有用的工作。現代系統的上下文切換時間約為 1-100 微秒。如果切換頻率過高，系統效能會顯著下降。

## 行程的建立與終止

### 建立方式

- **系統初始化**：系統啟動時建立的守護行程
- **系統呼叫 fork()**：Unix/Linux 複製當前行程
- **系統呼叫 CreateProcess()**：Windows 建立新行程
- **使用者請求**：雙擊圖示或命令列輸入
- **批次處理**：批次腳本啟動新行程

### 終止原因

- **正常結束**：main 函式返回或 exit() 呼叫
- **錯誤終止**：除零錯誤、段錯誤等
- **外部殺死**：使用 kill 命令或任務管理器
- **父行程終止**：某些系統中父行程終止時子行程也終止

## 行程層次結構

Unix/Linux 中的行程形成樹狀結構。第一個行程是 init（PID=1），由它衍生出所有其他行程。使用 `pstree` 命令可以查看這個樹狀結構：

```
systemd─┬─NetworkManager
        ├─sshd───sshd───bash───pstree
        ├─cron
        └─nginx───nginx
```
