# 記憶體管理與虛擬記憶體（1960s-2023）

## 記憶體管理的目標

- **位址轉換**：將程式產生的邏輯位址映射到實體位址
- **保護**：防止一個行程存取另一個行程的記憶體
- **共享**：允許行程共享記憶體區段（如共享函式庫）
- **虛擬化**：讓程式可以使用的記憶體超過實體記憶體總量

## 連續記憶體分配

最早的記憶體管理方式——每個行程佔用連續的實體記憶體區域。

```
作業系統 | 行程 A | 行程 B | 閒置空間 | 行程 C | ...
```

**外部碎片**：隨著行程的載入與卸載，閒置空間被分割成許多小片段，即使總空間足夠，也無法分配給一個新行程（因為不連續）。

## 分頁（Paging）

分頁將實體記憶體劃分為固定大小的「框架」（frame，通常是 4KB），程式的邏輯位址空間也被劃分為相同大小的「頁面」（page）。

```
邏輯位址：頁號 (p) | 頁內偏移 (d)
實體位址：框架號 (f) | 頁內偏移 (d)
```

每個行程維護一個頁表（page table），記錄每個邏輯頁面對應的實體框架號。

### 兩層頁表

現代 64 位元系統的位址空間巨大（2⁶⁴ bytes），若使用單層頁表會消耗過多記憶體。解決方案是多層頁表：

```
邏輯位址：L1 索引 | L2 索引 | 偏移
```

x86-64 使用 4 層頁表（PML4 → PDPT → PD → PT → 偏移），每層 9 位元，偏移 12 位元（4KB 頁面）。

## 分段（Segmentation）

分段將程式的位址空間劃分為邏輯段：程式碼段、資料段、堆疊段等。每個段有獨立的基址和長度。

```
邏輯位址：段號 | 段內偏移
```

分段的好處是可以獨立保護和共享不同的段。缺點是會產生外部碎片。

## 虛擬記憶體

虛擬記憶體的核心思想：將程式需要的頁面載入到實體記憶體中，不需要的頁面保留在磁碟上。

- **分頁儲存**：頁面可以在實體記憶體和交換空間之間移動
- **按需分頁**：僅在頁面被存取時才載入到記憶體中
- **頁面置換**：當記憶體不足時，選擇一個頁面寫回磁碟

## 記憶體管理單元（MMU）

MMU 是硬體元件，負責即時進行位址轉換：

```
CPU → 邏輯位址 → MMU → 實體位址 → 記憶體
                   │
              TLB（快取近期轉換）
```

## TLB 快取

TLB（Translation Lookaside Buffer）是 MMU 內部的快取，儲存最近使用的頁表項。TLB 命中可以避免慢速的頁表走訪：

- TLB 命中：1 個時脈週期
- TLB 未命中：100-1000 個時脈週期（需走訪頁表）

## 頁面大小選擇

| 頁面大小 | 優點 | 缺點 |
|---------|------|------|
| 4KB | 頁內碎片少 | 頁表大、TLB 覆蓋範圍小 |
| 2MB（大頁面） | TLB 覆蓋範圍大 | 頁內碎片較多 |
| 1GB | 極大 TLB 覆蓋 | 適用於特定場景（如資料庫） |
