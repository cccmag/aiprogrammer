# 計算機歷史與馮紐曼架構

## 從算盤到 ENIAC

### 計算工具的演化

人類對計算工具的需求由來已久。從古代的算盤、計算尺，到 17 世紀巴斯卡（Blaise Pascal）發明的機械式加法器，再到 19 世紀巴貝奇（Charles Babbage）設計的分析引擎——每一步都為現代計算機的誕生奠定了基礎。

**關鍵里程碑**：

| 年份 | 事件 | 意義 |
|------|------|------|
| 1642 | 巴斯卡加法器 | 第一台機械計算機 |
| 1822 | 巴貝奇差分機 | 可程式化計算的雛形 |
| 1936 | 圖靈機 | 計算理論的數學基礎 |
| 1943 | Colossus | 第一台電子計算機 |
| 1946 | ENIAC | 第一台通用電子計算機 |
| 1945 | 馮紐曼架構 | 儲存程式概念的誕生 |

### ENIAC：電子數值積分計算機

ENIAC 於 1946 年在賓夕法尼亞大學完成，是史上第一台通用電子計算機。它使用了 17,468 個真空管、7,200 個二極體，重達 30 噸，佔地 167 平方公尺。ENIAC 每秒可執行約 5,000 次加法——比當時的機電式計算機快 1,000 倍。

但 ENIAC 有一個重大侷限：程式設計需要透過插拔電纜和設定開關來完成，每次更換任務都需要數小時甚至數天的重新配置。

## 馮紐曼架構的核心概念

1945 年，約翰·馮紐曼（John von Neumann）在「EDVAC 報告草案」中提出了儲存程式概念（Stored-Program Concept），徹底改變了計算機的設計。

### 五大組成單元

```
┌──────────┐     ┌──────────┐
│  輸入     │────→│  記憶體   │
│  (Input)  │     │ (Memory)  │
└──────────┘     └────┬─────┘
                      │
┌──────────┐     ┌────▼─────┐     ┌──────────┐
│  輸出     │←────│ 控制單元  │←────│  算術邏輯  │
│ (Output)  │     │ (Control) │     │ 單元(ALU)│
└──────────┘     └──────────┘     └──────────┘
```

1. **記憶體**（Memory）：儲存指令和資料
2. **算術邏輯單元**（ALU）：執行算術和邏輯運算
3. **控制單元**（Control Unit）：解碼指令並控制資料流
4. **輸入**（Input）：接收外部資料
5. **輸出**（Output）：輸出計算結果

### 儲存程式概念的革命性影響

儲存程式概念的核心是：**將指令視為資料**。指令和資料共用同一個記憶體空間，都可以被 CPU 讀取和修改。

這帶來了三個革命性影響：

1. **可程式化性**：更換程式不再需要重新佈線，只需載入新的指令序列
2. **自我修改程式**：程式可以在執行時修改自身指令（雖然現代程式設計不鼓勵這麼做）
3. **編譯器與組合語言的基礎**：高階語言可以編譯為機器指令，儲存在記憶體中執行

### 指令執行週期

馮紐曼架構的執行流程稱為「提取-解碼-執行」週期（Fetch-Decode-Execute Cycle）：

```
1. 提取（Fetch）：從記憶體中取出下一條指令的位址
2. 解碼（Decode）：控制單元解析指令的類型與操作數
3. 執行（Execute）：ALU 執行運算或存取記憶體
4. 寫回（Write-back）：將結果存回暫存器或記憶體
```

## 馮紐曼瓶頸

馮紐曼架構的最大限制是「馮紐曼瓶頸」（Von Neumann Bottleneck）：由於指令和資料共用同一條匯流排，CPU 和記憶體之間的資料傳輸速度限制了整體效能。這個問題在後續的哈佛架構（Harvard Architecture）中得到部分緩解——哈佛架構使用分離的指令和資料匯流排。

然而，馮紐曼架構的靈活性使其成為通用計算機的主流設計。即使是現代的處理器，雖然內部加入了快取、管線、分支預測等技術，其本質上仍然是馮紐曼架構的延伸。

## 小結

從 ENIAC 的真空管到現代 CPU 的數十億個電晶體，計算機架構的核心設計——馮紐曼架構——在近 80 年來幾乎沒有改變。它的簡潔性與通用性使其成為計算機科學史上最重要的概念之一。
