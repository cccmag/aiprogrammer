# 時間複雜度類別 P 與 NP

## 從可計算到有效可解

可計算性理論告訴我們哪些問題可以用電腦解決。但一個問題可以被解決，不等於它可以在合理的時間內被解決。複雜度理論（Complexity Theory）研究的就是「有效可計算性」——哪些問題可以在合理的時間和空間內解決。

時間複雜度類別是根據解決問題所需的計算時間來分類的。其中最重要的兩個類別是 P 和 NP。

## P 類別：有效可解

P（Polynomial time，多項式時間）是所有可以在多項式時間內被確定性圖靈機解決的決策問題的集合。

形式化定義：
> P = { L | 存在一個確定性圖靈機 M 和多項式 p(n)，使得 M 在 O(p(n)) 步內判定 L }

用白話說：P 類別的問題是「有效可解」的——當輸入規模增大時，解決問題所需的時間以多項式速度增長。

常見的 P 類別問題：
- 排序：O(n log n)
- 最短路徑（Dijkstra）：O(V²)
- 最小生成樹（Kruskal）：O(E log V)
- 線性規劃（Khachiyan）：O(n⁴L)
- 最大匹配：O(VE)

## NP 類別：有效可驗證

NP（Nondeterministic Polynomial time，非確定性多項式時間）是所有可以在多項式時間內被**非確定性圖靈機**解決的決策問題的集合。

但更直觀的理解是：NP 是那些「解答可以在多項式時間內被驗證」的問題。

形式化定義：
> NP = { L | 存在一個多項式時間驗證器 V，使得對於任意 x ∈ L，存在一個多項式長度的憑證 c，V(x, c) = 接受 }

用白話說：如果有人給你一個 NP 問題的解答，你可以快速檢查這個解答是否正確。

經典的 NP 問題：
- **SAT**：給定一個布林公式，是否存在一組變數賦值使其為真？
- **頂點覆蓋**：給定一個圖和 k，是否存在 k 個頂點覆蓋所有邊？
- **旅行推銷員（決策版本）**：是否存在一條長度 ≤ k 的路徑經過所有頂點？
- **圖著色**：給定一個圖和 k，能否用 k 種顏色為頂點著色使相鄰頂點顏色不同？
- **背包問題**：給定物品重量和價值，是否存在總價值 ≥ V 且總重量 ≤ W 的子集？

## P vs NP 問題

P vs NP 是電腦科學中最重要的開放問題：

> P = NP？還是 P ≠ NP？

如果 P = NP，那麼所有可以快速驗證解答的問題都可以快速解決。這將意味著：
- 數學定理的證明可以自動化
- 最佳排程、路線規劃等問題可以完美解決
- 現有密碼系統（如 RSA）將被攻破

如果 P ≠ NP（大多數研究人員相信這個），那麼某些 NP 問題本質上無法在多項式時間內解決。

## NP 的兩種定義

| 定義方式 | 說明 |
|---------|------|
| 非確定性圖靈機 | NP = 非確定性圖靈機在多項式時間內可解的問題 |
| 驗證器 | NP = 存在多項式時間驗證器的問題 |

這兩種定義是等價的。非確定性圖靈機的「猜測」能力對應於驗證器中的「憑證」。

## 為什麼 P ≠ NP 是主流猜想

多數複雜度理論家相信 P ≠ NP，理由包括：

1. **豐富的 NP-Complete 問題**：成千上萬的自然問題被證明是 NP-Complete，經過數十年努力仍未找到多項式演算法
2. **多項式時間驗證 vs 多項式時間求解**：兩者的直覺差異巨大
3. **密碼學的存在**：如果 P = NP，大多數現有密碼系統將不復存在
4. **理性樂觀**：但這些只是直覺，不是證明

## 開放問題的重要性

P vs NP 問題是 Clay 數學研究所的千禧年大獎問題之一，懸賞一百萬美元。但更重要的是：

- 它關係到我們對「計算本質」的理解
- 它影響著密碼學、AI、最佳化等眾多領域
- 它的解答將帶來全新的數學工具和洞見
