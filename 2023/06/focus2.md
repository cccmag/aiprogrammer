# Turing Machine 與 Church-Turing 論題

## Turing Machine 的形式定義

Alan Turing 在 1936 年的論文中提出了一個抽象的計算模型，現在被稱為 Turing Machine（圖靈機）。這個模型雖然簡單，卻可以描述任何可能的計算過程。

Turing Machine 由以下部分組成：

- **紙帶（tape）**：一條向兩端無限延伸的帶子，被分割成一個個格子
- **讀寫頭（head）**：可以在紙帶上移動，讀取和寫入符號
- **狀態寄存器（state）**：儲存當前狀態，來自有限個狀態的集合
- **轉移函式（transition function）**：定義機器在每個（狀態, 讀取符號）組合下的行為

形式化定義：一個圖靈機是一個七元組 (Q, Σ, Γ, δ, q₀, q_accept, q_reject)

- Q：有限狀態集合
- Σ：輸入字母表（不含空白符號）
- Γ：紙帶字母表（包含 Σ 與空白符號）
- δ：Q × Γ → Q × Γ × {L, R} 的轉移函式
- q₀：起始狀態
- q_accept：接受狀態
- q_reject：拒絕狀態

## 圖靈機範例：二元遞增

考慮一個對二元數字進行加一操作的圖靈機：

```
狀態 q0：向右掃描至字串末端
  δ(q0, 0) = (q0, 0, R)
  δ(q0, 1) = (q0, 1, R)
  δ(q0, _) = (q1, _, L)

狀態 q1：向左進行進位操作
  δ(q1, 0) = (q_accept, 1, R)  -- 遇到 0 改為 1，結束
  δ(q1, 1) = (q1, 0, L)        -- 遇到 1 改為 0，繼續進位
  δ(q1, _) = (q_accept, 1, R)  -- 遇到空白表示溢位，寫 1 結束
```

輸入 "1011" 的執行過程：
```
步驟 0: [1] 0  1  1  _  ...  狀態 q0
步驟 1:  1 [0] 1  1  _  ...  狀態 q0
步驟 2:  1  0 [1] 1  _  ...  狀態 q0
步驟 3:  1  0  1 [1] _  ...  狀態 q0
步驟 4:  1  0  1  1 [_] ...  狀態 q0 → q1
步驟 5:  1  0  1 [1] _  ...  狀態 q1 → 寫 0，向左
步驟 6:  1  0 [1] 0  _  ...  狀態 q1 → 寫 0，向左
步驟 7:  1 [0] 0  0  _  ...  狀態 q1 → 寫 1，接受！
```

結果 "1000" 確實是 "1011"（十進位 11）加一的結果（十進位 12）。

## 通用 Turing Machine

Turing 的天才不僅在於定義了圖靈機，更在於他證明了可以建構一個「通用圖靈機」（Universal Turing Machine, UTM）。UTM 可以接受任意圖靈機的描述作為輸入，然後模擬該圖靈機的執行。

通用圖靈機的概念直接對應到今天的通用電腦——一台可以執行任何程式的機器。UTM 證明了我們不需要為每種計算任務設計專門的硬體，一台通用機器就夠了。

## Church-Turing 論題

Church-Turing 論題（Church-Turing Thesis）是計算理論的基石：

> 任何在直覺上可計算的函式，都可以被某個圖靈機（或 λ 演算）計算。

這個論題被稱為「論題」而非「定理」，因為它涉及「直覺可計算性」這個非形式化的概念，無法被嚴格證明。然而，以下事實強烈支援這個論題的正確性：

1. **等價性**：λ 演算、圖靈機、Gödel 的一般遞迴函式、Kleene 的 μ-遞迴函式都被證明是彼此等價的
2. **穩固性**：所有合理的計算模型（RAM 機器、細胞自動機、量子圖靈機）都沒有超越圖靈機的能力
3. **驗證性**：數十年的計算實踐中從未發現反例

## 論題的當代意義

Church-Turing 論題不僅是理論上的概念，它對現代電腦科學有實際的指導意義：

- 告訴我們哪些問題在原則上可以用電腦解決（可計算的）
- 告訴我們哪些問題在原則上無法用電腦解決（不可判定的）
- 為複雜度理論提供了基礎——因為圖靈機是討論計算複雜度的標準模型
