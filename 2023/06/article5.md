# 歸約技巧入門

## 什麼是歸約？

歸約（Reduction）是計算理論中用來比較問題難度的核心工具。直觀地說，歸約就是「用問題 B 的解法來解決問題 A」。

如果你可以將問題 A 的任何實例轉換為問題 B 的實例，那麼能夠解決 B 的能力就轉移成了能夠解決 A 的能力。我們說「A 可以歸約到 B」，記作 A ≤ B。

## 歸約的兩種主要用途

### 證明可解性（正向）

如果 A ≤ B，且 B 是可判定的，那麼 A 也是可判定的。

```
A 的實例 →（轉換）→ B 的實例 →（B 的解法）→ 答案
```

### 證明不可解性（反向）

如果 A ≤ B，且 A 是不可判定的，那麼 B 也是不可判定的。

```
假設 B 可判定 → 用歸約判定 A → 矛盾
```

## 經典歸約範例

### 範例 1：停機問題 → 是否列印

考慮問題：給定一個程式 P 和輸入 I，P(I) 是否會列印 "Hello"？

從停機問題歸約：

```python
def reduce_halting_to_print(P, I):
    # 構造 Q：先執行 P(I)，然後列印 "Hello"
    def Q():
        simulate(P, I)
        print("Hello")
    return Q
```

如果 P(I) 會停機，則 Q 會列印 "Hello"。
如果 P(I) 不會停機，則 Q 也不會列印 "Hello"。

因此，如果我們能判斷「程式是否會列印 Hello」，我們就能判斷停機問題——矛盾。所以「是否列印 Hello」不可判定。

### 範例 2：3-SAT → 頂點覆蓋

3-SAT 是已知的 NP-Complete 問題，我們可以用它來證明頂點覆蓋也是 NP-Complete。

給定一個 3-CNF 公式 φ，我們構造一個圖 G 和一個整數 k：

- 每個變數 x 對應兩個頂點（x 和 ¬x）用邊連接（變數元件）
- 每個子句對應三個頂點用邊連接成三角形（子句元件）
- 將子句中的文字與對應的變數頂點用邊連接
- k = 變數數量 + 2 × 子句數量

φ 可滿足 ⇔ G 存在大小為 k 的頂點覆蓋。

### 範例 3：SAT → 3-SAT

任意 SAT 實例都可以轉換為等價的 3-SAT 實例：

對於每個子句 C = (l₁ ∨ l₂ ∨ ... ∨ lₖ)：
- 若 k = 1：替換為 (l₁ ∨ l₁ ∨ l₁)
- 若 k = 2：替換為 (l₁ ∨ l₂ ∨ l₁)
- 若 k = 3：保持不變
- 若 k > 3：引入新變數，將子句拆分為多個 3-子句

```
(l₁ ∨ l₂ ∨ ... ∨ lₖ)
→ (l₁ ∨ l₂ ∨ y₁) ∧ (¬y₁ ∨ l₃ ∨ y₂) ∧ ... ∧ (¬y_{k-3} ∨ l_{k-1} ∨ lₖ)
```

## 歸約的類型

| 歸約類型 | 限制 | 用途 |
|---------|------|------|
| 多項式時間歸約 | 轉換在 O(n^k) 內完成 | 證明 NP-Completeness |
| 對數空間歸約 | 轉換在 O(log n) 空間內完成 | 證明 P-Completeness |
| Turing 歸約 | 可多次呼叫預言機 | 一般可計算性證明 |
| 映射歸約 | 直接轉換，單次呼叫 | 經典不可判定性證明 |

## 歸約的竅門

1. **從已知的問題開始**：SAT、停機問題是很好的起點
2. **找到對應關係**：問題中的哪些元素對應於已知問題的哪些元素
3. **構造橋樑**：設計轉換函式
4. **雙向驗證**：證明轉換的正確性（如果且唯若）
5. **確保效率**：轉換本身必須是有效的（多項式時間或可計算的）

## 歸約的可視化

```
問題 A ──── 轉換函式 f ────→ 問題 B
  │                              │
  │ 如果 x ∈ A                   │ f(x) ∈ B
  │ 那麼 f(x) ∈ B                │ 那麼 x ∈ A
  │                              │
  └──────────────────────────────┘
        問題 A ≤ 問題 B
```
