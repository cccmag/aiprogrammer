# 編譯器概述與架構

## 什麼是編譯器？

編譯器是一種將高階程式語言翻譯成低階目標語言的程式。這個過程看似簡單，實則涉及多個複雜的階段。一個典型的編譯器由三個主要部分組成：前端（Front End）、中端（Middle End）和後端（Back End）。

```
原始碼 → [前端] → 中間表示 → [中端] → 中間表示 → [後端] → 目標碼
```

## 編譯器的階段

### 前端（分析階段）

前端負責理解原始碼，包含以下階段：

1. **詞法分析（Lexical Analysis）**：將原始碼字元流轉換為 Token 序列。例如 `x = 3 + 5` 轉換為 `IDENT(x) ASSIGN NUMBER(3) PLUS NUMBER(5)`。

2. **語法分析（Syntax Analysis）**：根據文法規則將 Token 序列建構成抽象語法樹（AST）。例如表達式 `3 + 4 * 2` 會被建構成正確運算子優先級的樹狀結構。

3. **語意分析（Semantic Analysis）**：檢查程式的語意正確性，包括型別檢查、變數宣告檢查等。這一步會建立符號表來追蹤所有識別字的資訊。

### 中端（最佳化階段）

中端對中間表示（IR）進行平台無關的最佳化：

- **常數折疊**：將 `2 + 3` 在編譯期計算為 `5`
- **死碼消除**：移除永遠不會被執行到的程式碼
- **迴圈不變式外提**：將迴圈內不變的計算移到迴圈外部
- **內聯展開**：將函式呼叫替換為函式體本身

### 後端（綜合階段）

後端將最佳化後的 IR 轉換為目標機器碼：

- **指令選擇**：將 IR 指令映射到目標機器的指令集
- **暫存器分配**：將虛擬暫存器分配給真實的實體暫存器
- **指令排程**：重排指令順序以充分利用管線化

## 編譯器 vs 直譯器

編譯器和直譯器都是將高階語言轉換為可執行形式，但方式不同：

| | 編譯器 | 直譯器 |
|---|---|---|
| 執行方式 | 先翻譯後執行 | 邊翻譯邊執行 |
| 啟動速度 | 需要先編譯，啟動慢 | 直接執行，啟動快 |
| 執行速度 | 機器碼直接執行，速度快 | 需要即時翻譯，速度慢 |
| 錯誤回報 | 一次回報所有錯誤 | 執行到錯誤處才回報 |

## 知名的編譯工具

- **Lex / Flex**：詞法分析器產生器
- **Yacc / Bison**：語法分析器產生器
- **LLVM**：現代化的編譯器基礎設施
- **ANTLR**：強大的解析器產生器，支援多種語言
- **GCC**：GNU Compiler Collection，支援多種語言和平台

## 實作範例

本期提供的迷你編譯器使用 Python 實作，展示了最簡化的編譯流程：

```python
# 原始碼
x := 3 + 4 * 2;

# 經過詞法分析、語法分析、AST 建構後
# 產生的目標碼
LOADI R0 3
LOADI R1 4
LOADI R2 2
MUL   R3 R1 R2
ADD   R4 R0 R3
STORE x R4
```
