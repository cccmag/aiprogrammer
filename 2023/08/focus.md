# 本期焦點

## 編譯器理論：從原始碼到執行檔的奇幻旅程

### 引言

編譯器（Compiler）是計算機科學中最重要也最影響深遠的軟體系統之一。每當你撰寫高階語言程式，編譯器就負責將人類可讀的原始碼轉化為機器可執行的指令。這個過程涉及數學、形式語言理論、資料結構和演算法的精妙結合。

本期雜誌將以七個主題深入探索編譯器的核心技術，配合一個完整的迷你編譯器實作，帶領讀者從實作中理解編譯原理。

我們將從編譯器的整體架構開始，逐步深入每個階段的理論與實踐。無論你是編譯器初學者還是有經驗的開發者，都能從中找到啟發。

## 濃縮回顧

### 編譯器的本質

編譯器本質上是一個翻譯器，它將一種語言（原始語言）轉換為另一種語言（目標語言）。這個過程可以抽象為：

```
原始碼 → 前端（分析）→ 中間表示 → 中端（最佳化）→ 中間表示 → 後端（生成）→ 目標碼
```

### 編譯器的歷史

- **1950s**：Grace Hopper 開發第一個編譯器 A-0 System
- **1957**：FORTRAN 編譯器誕生，第一個最佳化編譯器
- **1960**：COBOL、ALGOL 編譯器
- **1970s**：C 編譯器、Pascal 編譯器、Lex 與 Yacc 工具
- **1980s**：GCC、C++ 編譯器
- **1990s**：Java JIT 編譯、LLVM 項目啟動
- **2000s-2020s**：LLVM 生態成熟、Rustc、WebAssembly

### 為什麼要學編譯器？

1. **理解底層**：知道高階語言的程式碼最終如何執行
2. **語言設計**：掌握設計 DSL（領域特定語言）的能力
3. **工具開發**：Linter、Formatter、靜態分析工具都用到編譯技術
4. **最佳化思維**：理解何時以及如何最佳化程式碼

### 實作導向的學習

理論與實作相輔相成。本期提供的[迷你編譯器](focus_code.md)用不到 200 行 Python 實現了完整的編譯流程：從詞法分析、語法分析、AST 建構到程式碼生成。建議讀者一邊閱讀理論文章，一邊對照實際程式碼，動手修改和擴充。

## 結論與展望

編譯器理論雖然是一個成熟的領域，但新技術的出現不斷賦予它新的生命力。WebAssembly、JIT 編譯、GPU 編譯器、資料庫查詢編譯等新興應用，讓編譯技術在 AI 時代展現出新的價值。

無論技術如何演進，編譯器的三大核心問題始終不變：
1. **如何理解程式？**（分析）
2. **如何表示程式？**（中間表示）
3. **如何生成高效程式？**（綜合）
