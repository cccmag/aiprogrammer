# 未來展望：GitHub 與社交程式設計

## 2007 年的程式托管

2007 年，程式托管服務正在快速發展。

### 主要平台

```
2007 年程式托管平台：
───────────────────────
SourceForge    最早的開源專案托管
Google Code    2006 年上線，支援 SVN
GitHub         2008 年正式上線
Launchpad      Canonical 的平台
CodePlex       微軟的開源托管
```

## GitHub 的誕生

### GitHub 的起源

2007 年，Tom Preston-Werner 和朋友們在開發 GitHub。GitHub 的想法是「社交程式設計」——將 Git 的版本控制與社交網路的功能結合。

### GitHub 的創新

```
GitHub 的創新：
─────────────────
1. Git 仓库托管
2. Pull Request 機制
3. 社群功能（follow、watch、star）
4. 組織（Organizations）
5. 問題追蹤（Issue Tracker）
6. Wiki
7. Gist（程式碼片段分享）
```

## 社交程式設計的概念

### 社交網路的影響

```bash
# 社交程式設計的元素
# 1. 關注其他開發者
follow chris

# 2. 關注專案
watch rails/rails

# 3. Fork 別人的專案
fork octocat/Spoon-Knife

# 4. Pull Request 貢獻
pull-request --repo rails/rails --branch fix-bug
```

### Fork 和 Pull Request

```bash
# Fork 的價值
# 1. 實驗性開發
# 2. 貢獻開源專案
# 3. 獨立開發版本

# 典型流程
# 1. 在 GitHub 上 fork
# 2. 克隆到本地
git clone https://github.com/yourname/repo.git

# 3. 建立分支
git checkout -b fix-bug

# 4. 開發和提交
git add .
git commit -m "Fix the bug"

# 5. 推送到你的 fork
git push origin fix-bug

# 6. 在 GitHub 發起 Pull Request
```

## 開放原始碼的繁榮

### GitHub 催生的變化

```
GitHub 對開源的影響：
─────────────────────
1. 降低貢獻門檻（不需要是 committer）
2. 透明度提高（任何人都能看到程式碼）
3. 協作規模擴大
4. 催生新的授權模式
```

### 2007 年的預測

2007 年，人們預測：
- Git 將成為主流
- 程式托管將「社交化」
- Fork 和 Pull Request 將改變開源貢獻方式

這些預測在 2008 年 GitHub 上線後全部應驗。

## 未來趨勢

### 2007 年的技術趨勢

```
未來幾年的發展：
─────────────────
2008      GitHub 上線
2009      Git 成為最多使用的 DVCS
2010      Pull Request 成為標準貢獻方式
2011      社交程式設計流行
2012      GitHub 成為最大程式碼托管
2013      企業開始採用 GitHub
```

## 結語

2007 年是程式托管的轉捩點。Git 的成熟和社交程式設計的興起，將徹底改變軟體開發的協作方式。

GitHub 不只是一個代码仓库——它是一個開發者的社交網路。這個概念在 2007 年還很新，但它的潛力是巨大的。
