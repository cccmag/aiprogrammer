# Git 分散式版本控制興起

## 前言

2007 年，Git 版本控制系統開始獲得關注，這得益於 Linux 核心開發團隊的採用。

## Git 的優勢

```bash
# Git 基本命令
git init                    # 初始化
git add file.txt            # 添加到暫存區
git commit -m "message"    # 提交
git push origin main        # 推送到遠端
git pull                    # 拉取代碼
git branch feature          # 創建分支
git merge feature           # 合併分支
```

### 對比 Subversion

| 特性 | Git | SVN |
|------|-----|-----|
| 類型 | 分散式 | 集中式 |
| 分支 | 快速、簡單 | 較慢 |
| 離線支援 | 完全支援 | 有限 |
| 效能 | 更快 | 較慢 |

## GitHub 的崛起

2007 年，GitHub 正在興起（2008 年正式上線），為 Git 提供了絕佳的托管平台。

## 結論

Git 的分散式設計彻底改变了版本控制的思维，为日後開源專案的協作模式奠定基礎。
