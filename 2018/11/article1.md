# 詞袋模型到詞嵌入

## 傳統文本表示

### One-Hot 編碼
每個詞被表示為一個長度為詞彙表大小的向量，只有該詞位置為 1：
```
Apple = [1, 0, 0, 0, ...]
Banana = [0, 1, 0, 0, ...]
```

問題：
- 維度極高（詞彙表可達數十萬）
- 完全稀疏
- 無法表達語義關係

### TF-IDF
詞語頻率 - 逆文件頻率，考慮詞在文檔中的重要性：
```
TF(t) = 詞 t 在文檔中出現次數 / 文檔總詞數
IDF(t) = log(總文檔數 / 包含詞 t 的文檔數)
TF-IDF = TF × IDF
```

仍屬於稀疏表示，無法捕捉深層語義。

## 詞嵌入的誕生

詞嵌入（Word Embedding）將詞映射到低維稠密向量空間：
- 維度通常為 50、100、200 或 300
- 向量值為連續實數
- 相似詞在空間中接近

## 詞嵌入的學習方式

### 監督式學習
在特定任務（如情感分類）上訓練，同時學習分類器與詞嵌入。

### 無監督式學習
從大規模無標注語料中學習，語言模型本身提供監督信號：
```
P(w_1, w_2, ..., w_n) = Π P(w_i | w_1, ..., w_{i-1})
```

## 詞嵌入的性質

訓練良好的詞嵌入具有以下性質：
1. **語義相似性**：相似詞（如「狗」與「犬」）向量接近
2. **語法規律性**：語法關係（如複數）的位移相似
3. **類比推理**：「國王 - 男人 + 女人 ≈ 女王」

## 詞嵌入的局限性

靜態詞嵌入每個詞只有一個向量，無法處理：
- **一詞多義**：「bank」的不同含義
- **上下文依賴**：同一詞在不同語境的語義差異

這促使了動態語境表示（ELMo、BERT）的發展。
