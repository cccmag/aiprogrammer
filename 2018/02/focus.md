# 本期焦點：版本控制與 Git

## 概述

版本控制是軟體開發的核心技能，Git 是目前最流行的分散式版本控制系統。本期將介紹 Git 的基礎概念、常用指令，以及 GitHub 協作流程。

## 主題文章

1. **Git 基礎概念** - 理解 Git 的核心思想
2. **Git 常用指令** - 日常開發必備指令
3. **分支管理** - Git 分支的藝術
4. **GitHub 協作流程** - 團隊開發必備知識
5. **Pull Request 實務** - 程式碼審查與協作
6. **Git 進階技巧** - stash、rebase、reset
7. **版本控制最佳實踐** - 團隊合作規範

## 為什麼需要版本控制

### 版本控制的好處

1. **追蹤變更** - 清楚知道誰在什麼時候改了什么
2. **回復版本** - 輕鬆回到之前的版本
3. **分支開發** - 多人並行開發不同功能
4. **備份機制** - 代码有完整的歷史記錄
5. **團隊協作** - 有效管理多人同時編輯

### Git vs 其他系統

| 特性 | Git | SVN | CVS |
|------|-----|-----|-----|
| 類型 | 分散式 | 集中式 | 集中式 |
| 速度 | 快 | 較慢 | 較慢 |
| 離線工作 | 可 | 不可 | 不可 |
| 分支 | 輕量快速 | 較慢 | 困難 |

## 基本概念

### 工作區域

```
Working Directory → Staging Area → Git Directory
     (工作目錄)        (暫存區)       (本地倉庫)
```

### Git 的三個狀態

1. **Modified** - 檔案已修改但未暫存
2. **Staged** - 檔案已修改並暫存，等待提交
3. **Committed** - 檔案已提交到本地倉庫
