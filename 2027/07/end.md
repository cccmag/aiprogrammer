# 結語

## 從管理模型到管理 AI 系統

AI 原生開發維運不僅是工具鏈的升級，更是思維模式的轉變。當 AI 系統從單一模型演進為提示詞、RAG 管線、Agent 行為的複雜組合時，我們需要全新的維運哲學。

### 關鍵學習

1. **可觀測性是基礎** — 沒有可觀測性，就無法除錯；沒有除錯能力，AI 系統就不該上線。從開發階段嵌入 OpenTelemetry 追蹤，比事後補救有效十倍。

2. **提示詞就是程式碼** — 2027 年的標準實踐是：提示詞必須經過版本控制、審查、測試、A/B 驗證才能部署。Prompt Registry 是每個 AI 團隊必備的基礎設施。

3. **評估驅動迭代** — LLM-as-Judge 雖然不完美，但結合人工抽樣與品質閘道，已能建立可靠的評估迴圈。沒有評估，就無法知道變更是進步還是退步。

4. **自動化是唯一出路** — 從漂移偵測到自動回滾，從 CI/CD 管線到基礎設施即程式碼——手動管理 AI 系統在大規模下不可行。

### 下一步

- 在本期 [Mini MLOps 平台](focus_code.md) 的基礎上串接真實 LLM API
- 使用 LangSmith 或 MLflow 4.0 建立你團隊的 LLMOps 基礎設施
- 導入 OpenTelemetry 追蹤你的 RAG 管線與 Agent 行為
- 為你的提示詞建立版本控制與 A/B 測試流程

### 下期預告

2027 年 8 月號（202708）將探討 **多代理系統與 AI 協作 — 從單一 Agent 到 Agent 生態系**，涵蓋 Agent 角色設計、任務分解、通訊協議、多 Agent 安全治理。
