# AI 安全威脅全景

## 從提示詞注入到供應鏈攻擊（2022-2026）

### 前言

2022 年底 ChatGPT 問世後，AI 安全威脅從學術討論迅速變成真實世界的風險。OWASP 在 2023 年發布了 LLM 應用的十大安全風險，標誌著 AI 安全正式成為資安領域的核心議題。

### OWASP Top 10 for LLM 演變

**2023 年第一版**列出以下關鍵威脅：

| 排名 | 威脅 | 說明 |
|------|------|------|
| LLM01 | 提示詞注入 | 透過惡意輸入繞過系統提示 |
| LLM02 | 輸出處理不當 | 模型產出未過濾的有害內容 |
| LLM03 | 訓練資料中毒 | 惡意資料污染模型行為 |
| LLM04 | 模型阻斷服務 | 大量請求導致系統崩潰 |
| LLM05 | 供應鏈漏洞 | 第三方元件中的安全缺陷 |

**2025 年第二版**新增了「過度代理權限」和「向量與 Embedding 攻擊」兩項，反映 AI Agent 和 RAG 架構的普及。

### 真實攻擊案例

**2023 年**：研究人員發現透過 base64 編碼可以繞過 ChatGPT 的內容過濾器。例如將「如何製作炸彈」編碼後餵給模型，仍能獲得詳細步驟。

**2024 年**：某知名 AI 程式助手遭供應鏈攻擊——攻擊者上傳包含後門的惡意套件，當模型推薦此套件時，開發者無意中引入漏洞。

**2025 年**：多個企業的客服 AI Agent 遭間接提示詞注入——攻擊者在公開網頁上嵌入隱形文字，當 AI 爬取該頁面時，這些文字覆蓋了系統提示，導致資料外洩。

**2026 年**：模型提取攻擊成為主流。攻擊者透過數十萬次 API 查詢，從封閉模型中重建出精度超過 90% 的影子模型，威脅商業機密保護。

### 攻擊面分類

```
輸入層         模型層          輸出層
提示詞注入  →  模型對齊漏洞  →  有害內容
資料污染    →  模型提取      →  PII 外洩
供應鏈攻擊  →  後門植入      →  版權侵權
```

### AI Agent 帶來的全新風險

2025 年之後，具備工具使用能力的 AI Agent 大幅擴大了攻擊面。一個被注入的 Agent 可以：
- 讀取使用者的電子郵件
- 執行系統命令
- 存取內部 API
- 修改資料庫內容

### 小結

AI 安全威脅從 2022 年至今經歷了爆炸性成長。下一個前沿是**多模態攻擊**（圖片、音訊、影片中的隱蔽注入）和**自主 Agent 安全**。沒有單一解法，需要從輸入過濾、模型安全、輸出驗證到監控告警的縱深防禦。
