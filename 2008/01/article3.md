# 開放手機聯盟的成立

## 前言

2007 年 11 月 5 日，Google 宣布成立了「開放手機聯盟」（Open Handset Alliance，OHA），這是手機產業史上最大的聯盟之一。本文介紹這個聯盟的成立背景、成員和意義。

## 成立的背景

### 智慧型手機的需求

2007 年，智慧型手機市場正在快速成長：

- **Palm OS**：逐漸衰落
- **Windows Mobile**：功能有限
- **Symbian**：市場領導但過於複雜
- **iPhone**：剛發布，封閉生態

在這樣的背景下，Google 決定創建一個完全開放的手機平台。

### Google 的願景

Google 的目標是：
- 降低手機應用程式的開發門檻
- 創造一個開放的競爭環境
- 將網路服務整合到手機

## 聯盟的目標

### 主要任務

1. **建立開放標準**：創建手機軟體的開放標準
2. **分享智慧財產權**：成員間共享必要專利
3. **推動創新**：降低進入障礙
4. **確保互通性**：讓不同廠商的裝置可以互相溝通

### 商業模式

```
                ┌──────────────┐
                │   Android    │
                │    平台      │
                └──────┬───────┘
                       │
         ┌─────────────┼─────────────┐
         │             │             │
         ▼             ▼             ▼
   ┌──────────┐  ┌──────────┐  ┌──────────┐
   │  硬體    │  │   電信   │  │  軟體    │
   │  製造商  │  │   業者   │  │  開發者  │
   └──────────┘  └──────────┘  └──────────┘
         │             │             │
         └─────────────┼─────────────┘
                       │
                       ▼
              ┌────────────────┐
              │   使用者       │
              │ (最終受益者)   │
              └────────────────┘
```

Google 的收入模式：
- 透過 Android 推動 Google 服務（搜尋、地圖、郵件）
- 廣告收入
- Play Store 分潤（後來）

## 對產業的影響

### 打破封閉生態

傳統手機平台的問題：
- ** Nokia Symbian**：需簽署嚴格的授權協議
- **Microsoft Windows Mobile**：昂貴的開發工具
- **iPhone OS**：完全封閉，只有 App Store

Android 的出現改變了這一切：
- **免費授權**：任何人都能使用
- **開放原始碼**：可查看、修改原始碼
- **自由發布**：不強制使用特定商店

### 催生新興企業

OHA 的成立催生了許多新興企業：

1. **應用程式開發商**：大量小型開發團隊興起
2. **客製化 ROM**：CyanogenMod 等
3. **周邊裝置**：Android 支援的各種配件
4. **新創公司**：基於 Android 的商業模式

### 硬體夥伴的角色

硬體夥伴對 Android 的貢獻：
- 提供裝置進行測試
- 開發硬體驅動程式
- 參與 Android 相容性測試
- 客製化 UI（如 Sense、Motion Launch）

## 爭議與批評

### 開放性的質疑

OHA 也面臨一些批評：

1. **專利授權**：部分專利仍有爭議
2. **Google 服務**：強制預裝 Google Apps
3. **相容性**：各家裝置差異大
4. **更新延遲**：作業系統更新緩慢

### 聯盟內部的緊張

- 硬體規格不統一
- 客製化導致碎片化
- 電信業者干預

## 未來展望

### 持續成長

OHA 持續擴大：

- 更多中國和印度廠商加入
- 新興市場的 Android 裝置普及
- 汽車、電視、物聯網裝置

### 開放的承諾

Google 持續強調 Android 的開放性：

```
Android 開放宣言：
- 開放原始碼
- 公平授權
- 不強制預裝
- 開發者自由
```
