# 從 Hadoop 看巨量資料處理

## Hadoop 的起源

Hadoop 起源於 Apache Nutch 專案。2003 年，Google 發表 GFS 論文，2004 年發表 MapReduce 論文。Nutch 開發者 Doug Cutting 根據這些論文，實現了 Hadoop 的前身。

### 命名由來

Hadoop 的名字來源於 Doug Cutting 兒子的玩具大象。

## 核心元件

### HDFS（Hadoop Distributed File System）

- 儲存層，支援 PB 等級資料
- 高吞吐量讀取
- 自動副本容錯

### MapReduce

- 批次處理引擎
- 基於磁碟的運算
- 適合離線分析

## 周邊工具

### Hive

Facebook 開發的資料倉儲工具，提供類似 SQL 的查詢介面。

### Pig

Yahoo 開發的資料流程語言，簡化 MapReduce 程式開發。

### HBase

基於 BigTable 的 NoSQL 資料庫，支援即時讀寫。

### Sqoop

用於在 Hadoop 和關聯式資料庫之間傳輸資料。

### Flume

收集、聚合和移動巨量日誌資料的系統。

## 產業應用

### Yahoo!

Yahoo 是 Hadoop 的最大使用者之一，應用於：
- 網頁索引
- 日誌分析
- 廣告優化

### Facebook

使用 Hive 進行資料分析，每天處理 PB 等級資料。

### 電子商務

京東、阿里巴巴等電商平台使用 Hadoop 進行：
- 推薦系統
- 風險控制
- 資料分析

## 挑戰與限制

### 即時性不足

MapReduce 設計用於批次處理，延遲較高。

### 複雜度

開發和维护 Hadoop 應用需要專業知識。

### 資源利用

傳統 MapReduce 資源利用率不高。

## 未來方向

### YARN

將資源管理與應用框架分離。

### Spark

記憶體內運算，大幅提升效能。

### 雲端化

降低部署和運維複雜度。

## 結論

Hadoop 開啟了巨量資料處理的時代。其開源生態系統持續發展，為企業提供了處理大規模資料的能力。
