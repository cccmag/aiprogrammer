# 分散式檔案系統的設計原理

## 從本地檔案系統談起

傳統本地檔案系統如 ext4、XFS，是為單一機器設計的。它們依賴本地磁碟儲存資料，不支援跨機器的資料共享。

## 分散式檔案系統的必要性

當資料量超過單一機器容量時，需要分散式儲存：

- **容量擴展**：透過新增節點增加儲存空間
- **頻寬擴展**：多節點同時提供讀寫服務
- **可靠性**：資料副本提供容錯能力

## GFS 的創新設計

Google File System (2003) 開創了現代分散式檔案系統的先河。

### 核心假設

GFS 的設計基於對實際工作負載的觀察：

1. **元件故障是常態**：叢集由數千台普通伺服器組成
2. **大檔案為主**：通常 100MB 到數 GB
3. **新增為主**：很少覆蓋，多是附加
4. **大量順序存取**：隨機寫入較少

### 架構特點

```
                    Master
                 (單一主節點)
                     │
        ┌─────────────┼─────────────┐
        ↓             ↓             ↓
    ChunkServer   ChunkServer   ChunkServer
        │             │             │
        ↓             ↓             ↓
     區塊副本       區塊副本       區塊副本
```

### 元數據管理

Master 只儲存目錄結構和檔案到區塊的映射，而非實際資料：

```
元數據：
/users/data.txt → [chunk_001, chunk_002, chunk_003]
                 ↓
           chunk_001 → [cs1:8080, cs2:8080, cs3:8080]
```

### 一致性模型

GFS 放寬一致性要求，支援高效的追加操作：

- 追加是原子性的
- 讀取可能看到不一致的資料（但這對許多應用可接受）

## HDFS 的設計

Apache HDFS 是 GFS 的開源實現，保留了 GFS 的核心設計。

### 與 GFS 的差異

| 特性 | GFS | HDFS |
|------|-----|------|
| 開發者 | Google | Apache |
| 開源 | 否 | 是 |
| 語言 | C++ | Java |
| 快照 | 不確定 | 支援 |
| 許可 | 專有 | Apache |

### NameNode 記憶體

NameNode 將所有元數據保存在記憶體中。這決定了它能支援的檔案數量上限。

### 副本配置

HDFS 預設 3 副本配置：

```xml
<property>
    <name>dfs.replication</name>
    <value>3</value>
</property>
```

## Ceph 的創新

Ceph（2006 年開始開發）提出了創新的設計理念。

### CRUSH 演算法

Ceph 使用 CRUSH（Controlled Replication Under Scalable Hashing）演算法，客戶端可以直接計算資料位置，無需透過主節點。

```
傳統方式：                      CRUSH：
client → master → servers     client → CRUSH(servers)
         (瓶頸)                 (無中心)
```

### 統一儲存

Ceph 提供統一的介面：

- **RBD**：區塊儲存
- **CephFS**：檔案系統
- **RadosGW**：物件儲存

## 比較與選擇

### 比較表

| 特性 | HDFS | GlusterFS | Ceph |
|------|------|-----------|------|
| 統一儲存 | 否 | 是 | 是 |
| 元資料 | 集中 | 分佈 | 分佈 |
| 副本策略 | 固定 | 可配置 | CRUSH |
| 適用場景 | 分析 | 雲端 | 雲端+容器 |

### 選擇指南

- **Hadoop 生態**：使用 HDFS
- **雲端儲存**：Ceph 或 GlusterFS
- **對象儲存**：Ceph RADOS

## 未來發展

### 去中心化

新一代分散式檔案系統趨向去中心化，避免單點故障。

### 智慧資料放置

利用機器學習優化資料放置策略。

### 整合計算

將運算帶到資料所在位置，減少網路傳輸。

## 結論

分散式檔案系統是巨量資料處理的基礎設施。理解其設計原理，有助於選擇合適的儲存方案並最佳化應用效能。
