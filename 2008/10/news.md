# 2008 年 10 月 — Ruby on Rails 生態新聞

## Rails 2.2 正式發布

2008 年 11 月，Rails 2.2 正式發布，這是 Rails 框架史上重要的里程碑版本。新版本帶來了多執行緒支援、整合 i18n 國際化框架，以及更嚴格的 Rails 慣例。

參考：[Rails 2.2 Release Notes](https://www.google.com/search?q=Rails+2.2+release+2008)

## Ruby 1.9 開發进展

Ruby 1.9.1 正在積極開發中，預計將於 2009 年初發布。1.9 分支帶來了全新的虛擬機 YARV，理論效能提升可達 3-5 倍。

參考：[Ruby 1.9 YARV Virtual Machine](https://www.google.com/search?q=Ruby+1.9+YARV+virtual+machine+2008)

## Merb 框架崛起

Merb（Metal Embedded Ruby）作為輕量級 Rails 替代方案開始受到關注。其高性能和模組化設計吸引了許多追求效能的開發者。

參考：[Merb Framework 2008](https://www.google.com/search?q=Merb+framework+Ruby+2008)

## Git 成為 Rails 主流版本控制

Git 在 Rails 社群迅速普及。GitHub 的出現徹底改變了開源專案的協作方式，越來越多 Rails 插件遷移到 Git 版本控制。

參考：[GitHub Rails Projects 2008](https://www.google.com/search?q=GitHub+Rails+projects+2008)

## Capistrano 部署自動化

Capistrano 成為 Rails 應用部署的標準工具，支援一鍵部署到多台伺服器，大幅簡化了部署流程。

參考：[Capistrano Deployment Rails](https://www.google.com/search?q=Capistrano+Rails+deployment+2008)

## Rack 中間件興起

Rack 介面於 2008 年成為 Ruby Web 框架的標準底層，統一的介面讓不同框架和伺服器之間的相容性大幅提升。

參考：[Rack Ruby Web Interface](https://www.google.com/search?q=Rack+Ruby+web+interface+2008)

## JQuery 成為 Rails 預設 JavaScript 庫

Rails 2.x 開始支援 JQuery 作為預設的 JavaScript 庫，取代了 Prototype.js 的地位。

參考：[Rails JQuery Integration 2008](https://www.google.com/search?q=Rails+jQuery+prototype+2008)
