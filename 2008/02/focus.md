# 本期焦點

## Google Chrome 瀏覽器 — 新世紀的瀏覽器革命

### 引言

2008 年 9 月，Google 將發布 Chrome 瀏覽器，這是網頁瀏覽歷史上的一個重要時刻。Chrome 以其創新的 JavaScript 引擎、快速的分頁隔離技術、以及對網頁標準的優異支援，為網頁應用開創了新的時代。

本期雜誌將帶您深入了解 Chrome 的誕生背景、技術架構、以及對網頁開發的影響。

## 濃縮回顧

### 瀏覽器市場份額（2008 年）

```
IE 6/7：     ~70%
Firefox 2/3：~20%
Safari：     ~5%
其他：       ~5%
```

### Chrome 的創新

| 技術 | 說明 |
|------|------|
| V8 JavaScript 引擎 | 即時編譯 JavaScript，大幅提升效能 |
| 分頁隔離 | 每個分頁獨立程序，防止當機蔓延 |
| 沙盒安全 | 限制程式碼的系統存取權限 |
| Omnibox | 結合搜尋和網址輸入 |
| 應用程式分頁 | 可釘選為獨立視窗 |

### V8 引擎的效能提升

```
傳統直譯器：每次都需要解析位元組碼
V8 JIT 編譯：直接產生高效機器碼

效能提升：5-10 倍（取決於程式碼）
```

### 多程序架構的優點

```
單程序瀏覽器：
┌─────────────────────┐
│ 所有分頁 ──→ 一個程序│
│ 分頁當掉 ──→ 全部當掉│
└─────────────────────┘

多程序 Chrome：
┌──────┐ ┌──────┐ ┌──────┐
│分頁 1│ │分頁 2│ │分頁 3│  ← 各自獨立程序
└──────┘ └──────┘ └──────┘
     ↘          ↙
    ┌──────────────────┐
    │   瀏覽器主程序    │
    │  (負責 UI 和協調整)│
    └──────────────────┘
```

## 結論與展望

Chrome 的出現標誌著瀏覽器技術的新時代。V8 引擎的 JIT 編譯技術讓 JavaScript 的執行效率大幅提升，而多程序架構則解決了傳統瀏覽器的穩定性和安全性問題。
