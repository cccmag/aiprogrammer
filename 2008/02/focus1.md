# 瀏覽器大戰與 Chrome 的誕生

## 第一次瀏覽器大戰

### 網景 Navigator 的崛起

1994 年，網景通訊（Netscape Communications）推出了 Netscape Navigator，這是第一個商業化的網頁瀏覽器：

```
網景 Navigator 的創新：
├── 友善的圖形介面
├── 支援表格和框架
├── JavaScript 腳本支援
├── Cookie 機制
└── Plug-in 擴充系統
```

網景在巔峰時期佔據了超過 90% 的瀏覽器市場份額。

### Internet Explorer 的興起

1995 年，微軟在 Windows 95 中捆綁 Internet Explorer，並透過免費策略和與 Windows 的深度整合，快速搶佔市場：

| 年份 | IE 版本 | 市場份額 |
|------|---------|----------|
| 1995 | IE 1 | 少數 |
| 1996 | IE 2 | 上升中 |
| 1997 | IE 3 | 40% |
| 1998 | IE 4 | 60% |
| 1999 | IE 5 | 80% |
| 2001 | IE 6 | 90%+ |

### 網景的衰落

微軟的「擁抱、延伸、熄滅」策略讓網景無力招架：

- Windows 強制捆綁 IE
- IE 與系統深度整合
- 網景市佔率持續下滑

2001998 年，網景開放了 Navigator 的原始碼，最終促成了 Mozilla 專案的誕生。

## 第二次瀏覽器大戰

### Firefox 的崛起

2004 年，Mozilla Firefox 1.0 正式發布，開啟了第二次瀏覽器大戰：

```python
firefox_innovations = {
    "分頁瀏覽": "一個視窗多個分頁",
    "彈出式視窗封鎖": "對抗廣告視窗",
    "隱私瀏覽": "不留下紀錄的瀏覽",
    "擴充套件": "強大的擴充生態",
    "更好的標準支援": "HTML, CSS, JavaScript"

```

Firefox 的市場份額從 2004 年的 5% 逐漸上升到 2008 年的約 20%。

### 各瀏覽器比較（2008 年）

| 瀏覽器 | 優點 | 缺點 |
|--------|------|------|
| IE 6/7 | 市場佔有率高、ActiveX | 安全問題多、標準支援差 |
| Firefox | 擴充性、安全、標準 | 記憶體使用高 |
| Safari | WebKit 引擎、Mac 整合 | 平台限制 |
| Opera | 速度快、功能豐富 | 市佔率低 |

## Google 的策略

### 為何 Google 要做瀏覽器？

Google 進入瀏覽器市場有戰略考量：

```
Google 的商業模式：
┌──────────────────────────────────┐
│       搜尋 + 廣告收入            │
└──────────────────────────────────┘
           │
           ▼
瀏覽器是網頁入口 ──→ 影響搜尋使用量
           │
           ▼
    需要掌控入口
           │
           ▼
    開發自己的瀏覽器
```

### Chrome 的定位

Google Chrome 的設計目標：

1. **提升網頁應用體驗**：讓 Web App 如同桌面應用
2. **推動網頁標準**：讓網頁開發更容易
3. **增強線上廣告平台**：改善瀏覽體驗
4. **進入桌面市場**：Google Gears 等技術

## Chrome 的誕生

### 開發背景

2006 年，Google 內部開始 Chrome 的開發計畫。開發團隊來自：
- Google 工程師
- 部分 Mozilla 前成員

### 公開發布

2008 年 9 月 1 日，Google 發布了 Chrome 的 beta 版，2008 年 12 月正式版發布。

### 技術創新

Chrome 帶來了多項創新：

| 功能 | 說明 |
|------|------|
| V8 JavaScript 引擎 | JIT 編譯，大幅提升效能 |
| 分頁隔離 | 每個分頁獨立程序 |
| 沙盒安全 | 限制惡意程式影響 |
| Omnibox | 智慧網址列 |
| Incognito 模式 | 隱私瀏覽 |

## 開瀏覽器市場的變化

### Chrome 的市場表現

```
Chrome 市佔率變化：
2008 年：< 1%
2009 年：~3%
2010 年：~10%
2011 年：~20%
2012 年：~30%
```

### 對其他瀏覽器的影響

Chrome 的出現刺激了瀏覽器市場的創新：

- **Firefox** 加速開發 Firefox 4+
- **IE** 推出 IE 9/10 追趕標準
- **Safari** 持續改進 WebKit
- **Opera** 獲得更多關注

## 網頁應用新時代

### Web 2.0 與瀏覽器

2008 年是 Web 2.0 的高峰期：

```python
web_20_applications = [
    "Gmail - 完整功能的郵件用戶端",
    "Google Docs - 線上文書處理",
    "Facebook - 社交網路平台",
    "Twitter - 微博服務",
    "YouTube - 影片分享",
    "Flickr - 圖片分享"
]
```

這些應用需要更快的 JavaScript 效能和更穩定的瀏覽器。

### Chrome 的優勢

Chrome 專為 Web 2.0 應用設計：

```
Chrome 對 Web App 的支援：

1. 快速 JavaScript → 流暢的使用者介面
2. 分頁隔離 → 穩定的多人應用
3. 沙盒安全 → 安全的線上服務
4. 開發者工具 → 更容易除錯
5. V8 引擎 → 支援復雜的 Web App
```

## 未來展望

### 瀏覽器的演進方向

2008 年後的瀏覽器趨勢：

- **效能提升**：更快的 JavaScript、更短的啟動時間
- **標準化**：HTML 5、CSS 3 廣泛支援
- **安全**：更強的沙盒、更嚴格的政策
- **雲端整合**：書籤、設定的雲端同步
