# 檢索式對話模型

## 核心概念

檢索式對話系統的核心思想是：從預先構建的回覆庫中選擇最合適的回覆，而非即時生成。這種方法在問答系統、客服機器人等場景中廣泛應用。

## TF-IDF 與向量檢索

TF-IDF（詞頻-逆文件頻率）是最經典的檢索方法。系統將使用者查詢和候選回覆都表示為 TF-IDF 向量，然後計算餘弦相似度：

```
使用者查詢：q = "今天天氣如何"
候選回覆：r = "今天天氣晴朗高溫28度"
相似度 = cosine(TF-IDF(q), TF-IDF(r))
```

### BM25 演算法

BM25 是 TF-IDF 的改進版本，引入了詞頻飽和和文件長度歸一化機制，在對話檢索任務中表現優異。

## 語義匹配模型

2010 年代後期，基於 BERT 的語義匹配模型大幅提升了檢索式系統的性能。Dual Encoder 架構是主流方法：

```
查詢編碼器: q → BERT → 查詢向量
回覆編碼器: r → BERT → 回覆向量
相似度: cos(q_vector, r_vector)
```

這種方法能夠理解查詢和回覆之間的深層語義關係，超越單純的關鍵詞匹配。

## 候選回覆排序

檢索式系統通常使用兩階段架構：

1. **召回階段**：使用 BM25 或向量檢索找出 top-K 候選
2. **排序階段**：使用精細的語義模型對候選進行重排序

## 混合檢索策略

現代檢索式對話系統通常結合多種檢索方法：

- **稀疏檢索**（TF-IDF、BM25）：精確匹配關鍵詞
- **稠密檢索**（BERT、Sentence-BERT）：語義匹配
- **知識圖譜檢索**：結構化知識查詢

## 優缺點

檢索式系統的優點包括回覆品質可控、內容安全可管理、推理速度快。缺點包括無法處理知識庫未涵蓋的問題，且缺乏生成新內容的能力。
