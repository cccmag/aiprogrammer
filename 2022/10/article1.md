# 強化學習 vs 監督式學習

## 1. 引言

在機器學習領域，監督式學習（SL）和強化學習（RL）是兩種截然不同的學習範式。監督式學習從標記資料中學習輸入到輸出的映射，強化學習則透過與環境的互動來學習最佳行為策略。理解兩者的區別，是掌握 RLHF 的基礎。

## 2. 監督式學習

監督式學習的流程很直觀：

```
輸入 x → 模型 f_θ(x) → 預測 ŷ
預測 ŷ → 損失 L(ŷ, y) → 梯度 ∇θ → 參數更新
```

**關鍵特徵**：
- 每個訓練樣本有「正確答案」（標籤）
- 訓練資料是靜態的、獨立的
- 目標是擬合訓練資料的分佈
- 常見於分類、回歸、序列生成

### 監督式學習的局限性

當我們要訓練一個「有用且安全的對話模型」時，監督式學習面臨根本問題：

1. **何謂「正確」？** 對話沒有標準答案——什麼是「好的回應」是主觀判斷
2. **標籤品質**：人類撰寫示範回應的成本高，且不同人寫的不同
3. **分佈偏移**：模型在訓練資料上學到的模式，在真實使用時可能不適用

## 3. 強化學習

強化學習的流程：

```
Agent 觀測狀態 s → 採取動作 a → 環境回饋獎勵 r
獎勵 r → 策略更新 → 更好的行為
```

**關鍵特徵**：
- 沒有「正確答案」，只有「獎勵信號」
- 訓練資料來自 Agent 自己的互動
- 目標是最大化累積獎勵
- 需要處理「探索與利用」的權衡

### 強化學習的優勢

對於對話模型的訓練，RL 有天然優勢：

1. **不需要正確答案**：只需要比較「哪個回應更好」
2. **動態互動**：模型在真實使用場景中持續學習
3. **長期規劃**：考慮當前回應對後續對話的影響

## 4. 為什麼 RLHF 需要強化學習？

RLHF 選擇強化學習而非監督式學習，核心原因：

**監督式學習只能複製人類的行為，不能超越人類的行為。**

如果我們只用 SFT，模型最多只能學到人類標註者的平均水準。但透過 RLHF，模型可以：
- 發現人類自己都沒有意識到的「更好回應」
- 在人類偏好的方向上持續改進
- 超越人類標註者的品質

## 5. 兩者比較

| 特性 | 監督式學習 | 強化學習 |
|------|---------|---------|
| 反饋形式 | 標籤（正確答案） | 獎勵（好/壞） |
| 反饋粒度 | 每個樣本 | 延遲、稀疏 |
| 資料來源 | 靜態資料集 | 動態互動 |
| 探索要求 | 無 | 關鍵 |
| 泛化方式 | 擬合分佈 | 最佳化回報 |
| 適用場景 | 有標準答案的任務 | 有偏好的任務 |

## 6. 混和模式：IL + RL

在實際應用中，監督式學習和強化學習不是互斥的。RLHF 本身就是混合模式：

1. **SFT 階段**：使用監督式學習（人類示範資料）
2. **RL 階段**：使用強化學習（人類偏好獎勵）

這種混合策略結合了兩者的優點：SFT 提供穩定的初始策略，RL 進一步最佳化策略。

## 7. 結語

強化學習不是監督式學習的替代品，而是互補。在那些「有標準答案」的任務中，監督式學習仍然是更好的選擇。但在那些「只有偏好沒有標準答案」的任務中——對話、創作、判斷——強化學習是唯一可行的方法。RLHF 的出現，正是對這種需求的最優回應。
