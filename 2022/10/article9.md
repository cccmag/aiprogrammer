# ChatGPT 的 RLHF 流程

## 1. 引言

ChatGPT 於 2022 年 11 月底發布，迅速成為史上成長最快的應用程式。它的核心技術就是 RLHF——強化學習從人類反饋。本文將解析 ChatGPT 的 RLHF 訓練流程。

## 2. 從 GPT-3.5 到 ChatGPT

ChatGPT 基於 GPT-3.5 系列模型，該系列是 GPT-3 的改進版本，在程式碼和指令遵循上進行了專門的最佳化。

### 訓練管線

```
GPT-3 預訓練模型
    │
    ▼
GPT-3.5（Codex + text-davinci-002）
    │
    ▼
SFT 微調（人類示範資料）
    │
    ▼
獎勵模型訓練（人類偏好資料）
    │
    ▼
PPO 強化學習
    │
    ▼
ChatGPT
```

## 3. SFT 階段

ChatGPT 的 SFT 階段使用人類對話示範資料進行微調：

**資料來源**：
- 標註者扮演「使用者」和「AI 助手」，撰寫對話
- 涵蓋各種主題：問答、寫作、編碼、分析等
- 包含邊際案例：如何拒絕有害請求、如何承認不確定

**訓練細節**：
- 使用標準的語言模型損失（交叉熵）
- 訓練數個 epoch
- 學習率較低，防止遺忘預訓練知識

## 4. 獎勵模型訓練

ChatGPT 的獎勵模型訓練使用大規模偏好資料：

**資料收集流程**：
1. 使用 SFT 模型對給定提示生成多個回應
2. 將回應隨機配對
3. 人類標註者對每個配對選擇更好的回應
4. 收集標註者的書面理由（用於品質分析）

**模型架構**：
- 基底模型：GPT-3.5（共享大部分權重）
- 獎勵頭：線性層輸出標量
- 訓練目標：Bradley-Terry 損失

## 5. PPO 強化學習

ChatGPT 的 PPO 訓練有以下特點：

### 獎勵設計

總獎勵由三部分組成：
1. **獎勵模型分數**：人類偏好的代理指標
2. **KL 懲罰**：防止偏離 SFT 模型
3. **任務特定獎勵**：根據任務類型添加（如程式碼正確性）

### 訓練規模

- 策略模型（Actor）：數十億到上百億參數
- 獎勵模型：數十億參數
- 參考模型（KL 計算）：與 SFT 模型相同
- 值函數模型（Critic）：與策略模型架構相同

同時維護四個大規模模型，對記憶體和計算資源要求極高。

### 分佈偏移處理

ChatGPT 使用多種技術處理 PPO 中的分佈偏移：
- **KL 自適應調節**：根據實際 KL 散度動態調整 β
- **獎勵歸一化**：標準化獎勵分數，提高穩定性
- **梯度裁剪**：防止梯度爆炸

## 6. ChatGPT 的關鍵設計

### 對話格式

ChatGPT 使用特殊的對話格式：
```
system: 你是一個有用的AI助手...
user: 你好！
assistant: 你好！有什麼可以幫助你的？
user: 什麼是量子力學？
assistant: ...
```

這種格式在 RLHF 訓練中被視為「提示」的一部分。

### 安全過濾

ChatGPT 在 RLHF 之外還有多層安全機制：
- **輸入過濾**：檢測並拒絕有害提示
- **輸出過濾**：檢測並阻止有害生成
- **使用政策**：定義模型的使用邊界

## 7. RLHF 的挑戰

OpenAI 在開發 ChatGPT 時面臨的挑戰：
- **獎勵駭客**：模型學會利用獎勵模型的漏洞
- **訓練不穩定**：PPO 訓練在大規模下難以穩定
- **評估困難**：人類評估耗時且昂貴
- **持續改進**：需要持續收集人類反饋進行迭代

## 8. 結語

ChatGPT 的 RLHF 流程是 InstructGPT 方法的直接延伸，但在規模和實作細節上進行了大量工程最佳化。ChatGPT 的成功證明了 RLHF 在大規模語言模型中的有效性，也為後續的工作奠定了基礎。
