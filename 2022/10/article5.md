# 偏好資料收集策略

## 1. 引言

RLHF 的品質從根本上取決於人類偏好資料的品質。好的偏好資料能訓練出準確的獎勵模型，進而產生對齊良好的語言模型。本文探討偏好資料的收集策略——如何設計、收集和驗證高品質的人類偏好資料。

## 2. 偏好資料的格式

### 二元偏好（最常見）

```
prompt: "解釋量子糾纏"
回應 A: "量子糾纏是..."  ← 人類選擇（勝出）
回應 B: "量子糾纏是..."  ← (失敗)
```

二元偏好是最受歡迎的格式：
- 標註者容易理解
- 標註者一致率高
- 資料處理簡單

### 排名清單

```
prompt: "如何學習 Python？"
排名: C > A > D > B
```

排名提供更多的偏好資訊，每個提示產生多個比較對。但標註者疲勞度更高。

### 連續評分

```
prompt: "寫一首詩"
評分: 4/7
```

評分看似提供了更精細的資訊，但不同標註者的評分尺度不同（有人從不打滿分，有人總打高分），正規化處理複雜。

## 3. 回應配對策略

偏好資料的核心是「配對」——每個樣本包含同一個提示下的兩個回應。

### 隨機配對

最簡單的方法：從模型生成的所有回應中隨機配對。

優點：簡單、覆蓋面廣
缺點：配對難度可能差異大（明顯好壞 vs 難以區分）

### 難度感知配對

優先選擇難以區分的配對（模型得分接近的回應）：

```python
def difficulty_aware_pairing(responses, model_scores):
    pairs = []
    scores = sorted(enumerate(responses), key=lambda x: model_scores[x[0]])
    for i in range(0, len(scores), 2):
        if i+1 < len(scores):
            pairs.append((scores[i][1], scores[i+1][1]))
    return pairs
```

這種策略將標註資源集中在「邊界案例」上，提高資料效率。

### 主動學習配對

使用獎勵模型的不確定性選擇最有價值的配對：

1. 訓練初始獎勵模型
2. 對未標註配對預測偏好，計算不確定性
3. 選擇不確定性最高的配對交給人類標註
4. 更新獎勵模型，重複步驟 2

## 4. 標註品質控制

### 一致性檢查

- **內部一致**：同一配對多次出現，檢查標註者是否一致
- **外部一致**：不同標註者對同一配對的選擇是否一致
- **黃金標準**：混入已知答案的測試配對

### 標註者訓練

標註指南應包含：
- 任務說明（什麼是有用的回應？）
- 評分標準的維度（有用性、安全性、誠實性）
- 邊際案例的處理方式
- 常見錯誤示例

### 偏差緩解

常見偏差及其緩解方法：

| 偏差 | 緩解策略 |
|------|---------|
| 位置偏差 | 隨機交換配對順序（每個配對兩個方向） |
| 長度偏差 | 控制回應長度範圍；在指南中強調品質不長度 |
| 風格偏差 | 多樣化訓練資料中的回應風格 |
| 疲勞偏差 | 限制每日標註量；提供充足休息 |

## 5. 資料規模

典型的 RLHF 偏好資料規模：
- InstructGPT：約 5 萬個偏好配對
- Anthropic HH-RLHF：約 16 萬個配對
- 通常 3-10 萬個配對即可訓練出有效的獎勵模型

## 6. 結語

偏好資料的收集是 RLHF 中最昂貴但最重要的環節。好的收集策略可以大幅降低標註成本，同時提高資料品質。隨著 RLHF 的普及，偏好資料的自動化生成（如 AI 輔助標註）將成為重要的研究方向。
