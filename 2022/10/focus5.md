# PPO 與 RLHF 流程

## 完整 RLHF 管線

### 三階段流程總覽

RLHF 是一個三階段的訓練流程，從預訓練模型出發，逐步將人類偏好融入模型行為：

```
Stage 1: 監督微調（SFT）
  └─ 讓模型學會基本的指令遵循能力

Stage 2: 獎勵模型訓練
  └─ 從人類偏好中學習獎勵函數

Stage 3: PPO 強化學習
  └─ 使用獎勵模型最佳化策略
```

### Stage 1: 監督微調

SFT（Supervised Fine-Tuning）是 RLHF 的起點。目標是讓預訓練語言模型學會基本的「對話」和「指令遵循」能力：

- **資料來源**：人類撰寫的高品質示範資料（prompt → 理想回應）
- **損失函數**：標準的交叉熵損失（語言模型預訓練目標）
- **訓練方式**：使用 prompt 條件下生成理想回應
- **約束**：不追求完美的指令遵循，只需「夠好」的行為

SFT 階段約需要 1-2 萬條人類示範資料。

### Stage 2: 獎勵模型訓練

在 SFT 模型基礎上加入獎勵模型頭（線性層），使用人類偏好資料訓練：

- **資料來源**：人類對模型生成的回應進行偏好排序
- **損失函數**：Bradley-Terry 配對損失
- **資料量**：通常需要 5-10 萬個偏好配對
- **輸出**：對任何給定的回應輸出獎勵分數

### Stage 3: PPO 強化學習

PPO 階段是 RLHF 的核心，將獎勵模型整合到強化學習框架中：

1. **取樣**：從當前策略（語言模型）中生成回應
2. **評分**：使用獎勵模型對生成的回應給出獎勵
3. **優勢估計**：使用 Critic 網路計算優勢值
4. **策略更新**：使用 PPO 目標函數更新策略
5. **KL 懲罰**：確保策略不偏離 SFT 模型太遠

### KL 懲罰的作用

KL 懲罰是 RLHF 的關鍵設計，它防止策略模型在追求高獎勵時喪失語言能力：

```
總獎勵 = 獎勵模型分數 - β * KL(π_θ, π_SFT)
```

- β 控制懲罰強度
- KL 項防止模型找到「欺騙」獎勵模型的捷徑
- 維持模型的語言多樣性和自然度

### PPO 在 RLHF 中的實作細節

與標準 PPO 的關鍵差異：

| 項目 | 標準 PPO | RLHF PPO |
|------|---------|----------|
| 動作空間 | 連續或離散 | 詞元序列（離散） |
| 環境 | 模擬器 | 獎勵模型 |
| 獎勵 | 環境給出 | RM 給出 |
| 回合長度 | 固定步數 | 生成結束為止 |

### PPO 訓練的挑戰

- **獎勵分佈偏移**：隨著策略更新，生成的回應分佈改變，獎勵模型可能不準確
- **記憶體消耗**：同時維護 Actor、Critic、獎勵模型、參考模型四個模型
- **訓練穩定性**：KL 係數的調整需要仔細調參

### 訓練流程圖

```
預訓練模型
  │
  ▼
[SFT 微調] ←── 人類示範資料
  │
  ▼
[獎勵模型訓練] ←── 人類偏好資料
  │
  ▼
[PPO 強化學習] ←── 獎勵模型
  │               ←── SFT 參考模型（KL）
  │
  ▼
最終模型（對齊）
```
