# 人類反饋的收集與標註

## 偏好資料建構

### 為什麼需要人類反饋？

大型語言模型在預訓練階段只是學習文字的分佈，並不理解人類的偏好——什麼是有用的回應、什麼是安全的回應、什麼是符合倫理的回應。要讓模型「對齊」人類價值觀，就需要人類的直接反饋。

### 偏好資料的類型

RLHF 中常見的人類反饋形式：

1. **二元偏好**：給定兩個回應 A 和 B，人類標註者選擇哪個更好。最常見、最簡單的形式
2. **排名清單**：將多個回應排序，提供更多的偏好信息
3. **分數評分**：對每個回應給出連續分數（如 1-7），但不同標註者尺度不同
4. **詳細反饋**：文字形式的修改建議或評論，難以直接量化

二元偏好是最受歡迎的選擇——簡單、直觀、標註者一致性好。

### 偏好資料集的建構

以 Anthropic 的 HH-RLHF 資料集為例：

```
對話歷史 + 回應 A + 回應 B → 人類選擇更好的回應
```

建構步驟：
1. 使用現有模型生成多個回應候選
2. 將候選配對（pairwise），確保多樣性
3. 人類標註者對每個配對做出選擇
4. 收集標註者的「選擇理由」，用於品質分析

### 標註工具與平台

工業級標註需要專業工具支援：

- **自建平台**：OpenAI、Anthropic 使用自定義標註介面
- **Scale AI**：提供 RLHF 專用標註服務
- **Amazon SageMaker Ground Truth**：支援遷移學習和主動學習
- **Label Studio**：開源標註工具，支援自訂模板

### 標註者訓練與品質控制

人類標註是 RLHF 中最昂貴也最關鍵的環節：

**標註者訓練**：
- 提供詳細的標註指南（如「什麼是有用的回應」）
- 展示「好」和「壞」回應的例子
- 給出邊際案例的處理規則

**品質控制**：
- 測試題：混入已知答案的配對，篩選不合格標註者
- 一致率檢查：同一配對交給多人標註，計算一致率
- 專家覆審：隨機抽樣由資深標註者覆核

### 偏好中的偏差

人類偏好不可避免地存在偏差：

| 偏差類型 | 描述 | 緩解方法 |
|---------|------|---------|
| 位置偏差 | 更傾向選擇第一個看到的回應 | 隨機交換配對順序 |
| 長度偏差 | 更傾向選擇較長的回應 | 控制回應長度範圍 |
| 風格偏差 | 偏好像「專家」一樣的回應 | 多樣化訓練資料 |
| 疲勞偏差 | 疲勞時選擇不一致 | 控制工作時長 |

### 資料規模

RLHF 的人類標註需求：
- 獎勵模型訓練：約 5-10 萬個偏好配對
- SFT 示範資料：約 1-2 萬條人工撰寫的回應
- 標註者數量：通常數十到數百人

### 從人類反饋到獎勵模型

收集到的偏好資料並不能直接用於強化學習——我們需要一個可自動計算的獎勵函數。這就是獎勵模型的任務：從人類偏好中學習，然後對任何給定的回應輸出獎勵分數。
