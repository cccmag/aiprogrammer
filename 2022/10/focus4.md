# 獎勵模型訓練

## 從人類偏好到獎勵函數

### 為什麼需要獎勵模型？

RLHF 的關鍵挑戰：人類無法在每個步驟都提供反饋。如果每次模型生成一個詞元都要人類判斷，成本太高。解決方案是訓練一個獎勵模型（Reward Model, RM），學會模擬人類的偏好判斷。

### Bradley-Terry 模型

獎勵模型的標準框架是 Bradley-Terry 模型。給定一對回應 (y_1, y_2)，人類偏好 y_1 勝過 y_2 的機率表示為：

```
P(y_1 > y_2) = σ(r_θ(y_1) - r_θ(y_2))
```

其中 r_θ(y) 是獎勵模型對回應 y 輸出的獎勵分數，σ 是 sigmoid 函數。

### 損失函數

獎勵模型的訓練目標是最大化人類偏好資料的對數似然：

```
L(θ) = -E[ log σ(r_θ(y_w) - r_θ(y_l)) ]
```

其中 y_w 是勝出的回應（人類偏好），y_l 是失敗的回應。

這個損失函數的直觀理解：讓獎勵模型給「人類喜歡的回應」更高的分數，給「人類不喜歡的回應」更低的分數。分數差距越大，損失越小。

### 獎勵模型的架構

典型的獎勵模型架構：

1. **基底模型**：使用預訓練語言模型（如 GPT-3）作為編碼器
2. **輸出層**：移除語言模型的語言模型頭，改為線性層輸出單一標量（獎勵分數）
3. **共享權重**：獎勵模型通常共享基底模型的大部分權重

```
輸入：對話歷史 + 回應
  → 基底模型（共享預訓練權重）
  → 線性層（標量輸出）
  → 獎勵分數
```

### 訓練細節

- **初始化**：從 SFT 模型初始化獎勵模型的基底部分
- **配對方式**：每個訓練樣本是一對回應（來自同一個提示）
- **批次大小**：通常較小（如 32-64），因為每個樣本包含兩個回應
- **學習率**：較小（1e-5 到 5e-5），保持基底模型的語言理解能力
- **正則化**：使用權重衰減和 dropout 防止過擬合

### 獎勵模型的評估

獎勵模型的品質至關重要，因為它直接影響 PPO 階段的訓練效果：

- **準確率**：在測試集上預測偏好正確的比例
- **一致性**：不同提示下獎勵分數的穩定性
- **泛化能力**：對訓練分佈外的提示的表現
- **獎勵分佈**：觀察獎勵分數的分佈，避免過度集中

### Over-optimization 問題

當 PPO 階段過度最佳化獎勵模型時，模型可能學會「欺騙」獎勵模型而不是真正改進品質。這個問題被稱為「獎勵駭客」（reward hacking）。

解決方案：
1. **KL 正則化**：約束策略模型與 SFT 模型的距離
2. **集成獎勵模型**：使用多個獎勵模型的平均分數
3. **主動學習**：在訓練過程中持續收集人類反饋

### 獎勵模型的侷限

獎勵模型本質上是人類偏好的近似，而不是「真理」。它繼承了人類標註者的所有偏差和錯誤。因此，獎勵模型的輸出應該被視為「引導信號」而不是「絕對真理」。
