# 策略梯度與 PPO

## 策略最佳化演算法

### 策略梯度定理

策略梯度方法的核心思想是直接對策略參數 θ 進行梯度上升，以最大化期望獎勵 J(θ)：

```
∇J(θ) = E[ ∇log π_θ(a|s) Q^π(s, a) ]
```

這個梯度是「沿著能提高好動作機率的方向」更新策略。直觀理解：如果某個動作帶來了正向回饋，就增加它的機率；反之則降低。

**REINFORCE** 是最簡單的策略梯度演算法，使用蒙特卡羅採樣估計 Q 值：

```
∇J(θ) ≈ Σ ∇log π_θ(a_t|s_t) G_t
```

其中 G_t 是從 t 時刻開始的累積折扣獎勵。

### Actor-Critic 方法

REINFORCE 的方差很大，因為 G_t 的估計不穩定。Actor-Critic 方法引入一個值函數網路（Critic）來估計 Q 值，降低方差：

```
∇J(θ) ≈ Σ ∇log π_θ(a_t|s_t) ( r_t + γ V(s_{t+1}) - V(s_t) )
```

其中 r_t + γ V(s_{t+1}) - V(s_t) 是 TD（Temporal Difference）誤差，是優勢函數 A^π(s, a) 的估計。

### 重要性採樣

在策略梯度中，我們需要從新策略 π_θ 採樣來計算梯度。但採樣成本很高——每次更新策略後都要重新採樣。重要性採樣允許我們使用舊策略 π_old 的樣本來估計新策略的梯度：

```
J(θ) = E_{a~π_old}[ (π_θ(a|s) / π_old(a|s)) * A^π_old(s, a) ]
```

比值 r_t(θ) = π_θ(a_t|s_t) / π_old(a_t|s_t) 是重要性採樣權重，衡量新舊策略的差異。

### PPO 截斷目標

PPO（Proximal Policy Optimization）的核心創新是截斷（clipping）重要性採樣權重，防止策略更新過大導致訓練崩潰：

```
L^CLIP(θ) = E[ min( r_t(θ) A_t, clip(r_t(θ), 1-ε, 1+ε) A_t ) ]
```

- 當 A_t > 0（好動作）：權重被截斷在 1+ε，防止過度增加好動作的機率
- 當 A_t < 0（壞動作）：權重被截斷在 1-ε，防止過度降低壞動作的機率

ε 通常是 0.2，控制每次更新的大小。

### PPO 的完整目標函數

PPO 的總損失包含三部分：

1. **截斷策略損失**：L^CLIP，確保策略更新穩定
2. **值函數損失**：L^VF = (V_θ(s_t) - R_t)^2，訓練 Critic
3. **熵獎勵**：S[π_θ]，鼓勵探索，防止策略過早收斂

```
L^PPO = L^CLIP - c1 * L^VF + c2 * S[π_θ]
```

### 在 RLHF 中的應用

在 RLHF 的 PPO 階段：
- **Actor**：語言模型（策略），生成回應
- **Critic**：值函數網路，估計回應的期望獎勵
- **獎勵**：來自訓練好的獎勵模型
- **KL 懲罰**：防止語言模型偏離 SFT 模型太遠

### PPO 的優勢

| 特性 | 說明 |
|------|------|
| 穩定 | 截斷機制防止劇烈更新 |
| 樣本效率 | 重要性採樣重複使用舊樣本 |
| 簡單 | 不需要複雜的信任區域計算 |
| 通用 | 適用於離散和連續動作空間 |
