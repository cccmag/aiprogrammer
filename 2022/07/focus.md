# 本期焦點

## 網路資料與語料庫 — 爬蟲、清洗、預處理

### 引言

在機器學習領域，有一個廣為流傳的名言：「資料比模型更重要。」當我們談論大型語言模型時，這句話尤為真實。GPT-3、Bloom、PaLM 等模型的成功，不僅來自於精妙的架構設計，更來自於海量高品質的訓練資料。然而，這些資料並非憑空產生——它們來自於網路上的數十億個網頁，經過複雜的爬取、清洗、預處理流程，才最終成為模型訓練的養料。

本期雜誌將深入探討網路語料庫建構的完整流程：從網頁爬蟲的技術與倫理，到資料清洗的各種策略，再到文本預處理的標準管線，最後討論語料庫的管理與版本控制。

## 濃縮回顧

### 網路資料的價值

現代 NLP 模型依賴於海量文字資料。GPT-3 使用約 570 GB 的文本，Bloom 使用 1.6 TB 的 ROOTS 語料庫，這些資料絕大部分來自於網路爬取。研究表明，訓練資料的規模與模型效能之間存在明確的冪律關係（scaling law）。

### 爬蟲技術的核心

網路爬蟲的核心是自動化地獲取網頁內容。最常用的 Python 工具組合是 `requests` + `BeautifulSoup`，前者負責 HTTP 請求，後者負責 HTML 解析。對於動態內容，則需要使用 Selenium 或 Playwright 等瀏覽器自動化工具。

### 清洗與預處理

從原始 HTML 到乾淨的訓練文本，需要經過多層處理：

1. 去除 HTML/JavaScript/CSS 標籤
2. 編碼轉換（UTF-8 統一）
3. 正規化（全形轉半形、統一空白）
4. 敏感資訊遮蔽（Email、URL）
5. 分句與分詞

### Common Crawl 的影響

Common Crawl 是最大的開源網路語料庫，每月爬取數十億網頁。許多知名語言模型（如 GPT-3、T5、Bloom）都使用 Common Crawl 作為訓練資料的主要來源。然而，Common Crawl 的原始資料混雜大量低品質內容，需要經過多層過濾才能使用。

### 語料庫管理

隨著語料庫規模達到 TB 級別，傳統的 Git 版本控制已不再適用。DVC（Data Version Control）和 Git LFS 成為管理大規模資料集的標準工具。Hugging Face Datasets 則提供了統一的資料集存取介面。

## 結論與展望

從網頁爬取到語料庫建置，每一步都影響著最終模型的品質。在這個資料驅動的時代，掌握語料庫建構的技術，是每個 AI 工程師必備的核心能力。
