# 注意力權重計算

## 從分數到權重

注意力機制的核心是將「原始分數」轉換為「歸一化權重」。這個過程看似簡單，但其中包含多個設計選擇，每個選擇都會對模型行為產生深遠影響。

### 原始分數的生成

注意力分數（scores）衡量查詢與鍵之間的相似程度。常見的計算方式包括：

**點積相似度**：
```
score(q, k) = q · k = Σ q_i × k_i
```

這是最基本的相似度量測方法。點積的大小受向量長度影響，較長的向量傾向於產生較大的點積。

**雙線性相似度**：
```
score(q, k) = q^T W k
```

透過可學習的矩陣 W 將 q 和 k 映射到共同的相似度空間。W 可以學習到 q 和 k 之間複雜的交互模式。

**加性相似度**：
```
score(q, k) = v^T tanh(W_q q + W_k k)
```

使用神經網路來計算相似度，靈活性最高，但計算成本也最高。

## Softmax 的數學意義

### 為什麼是 Softmax？

注意力權重使用 softmax 歸一化有三個重要原因：

**機率解釋**：softmax 將分數轉換為機率分佈，所有權重總和為 1，權重值在 (0, 1) 範圍內。

**競爭機制**：如果一個位置獲得較高的權重，其他位置的權重自動降低。這模擬了生物注意力中的「選擇性」——注意力是一種有限的資源。

**可微分性**：softmax 是可微的，這使得它可以透過反向傳播進行端到端訓練。

### Softmax 的溫度參數

在實際應用中，softmax 通常帶有一個可選的溫度參數 τ：

```
α_i = exp(score_i / τ) / Σ_j exp(score_j / τ)
```

- **τ → 0**：注意力變得極度集中（接近 one-hot 選擇）
- **τ = 1**：標準 softmax
- **τ → ∞**：注意力變得均勻（所有位置等權重）

訓練時通常使用 τ = 1，但在推理時可以調整溫度來控制注意力的集中程度。

## 縮放因子的作用

### 數值穩定性的考量

在 Transformer 的 Self-Attention 中，注意力分數計算為 `Q K^T / √d_k`。縮放因子 `1/√d_k` 扮演著關鍵角色。

當 d_k 較大時（如 512 或 1024），Q 和 K 的元素通常為均值 0、方差 1 的隨機變量。此時：

- Q K^T 的方差 ≈ d_k
- 當 d_k = 512 時，標準差 ≈ 22.6
- softmax 在這種情況下產生極度集中的權重

### 梯度健康的維護

更重要的是，縮放因子有助於維護梯度的健康。標準 softmax 的梯度為：

```
∂α_i / ∂score_j = α_i (δ_{ij} - α_j)
```

當 softmax 的輸入值差異很大時，α 趨近於 one-hot 向量，梯度接近於零。縮放因子確保了輸入值保持在合理的範圍內，使 softmax 能產生「軟性」的注意力分佈。

### 實際驗證

實證研究表明，移除或改變縮放因子對模型性能有顯著影響：

```
基線 Transformer (√d_k) :  BLEU 27.3
無縮放             :  BLEU 23.5 (訓練不穩定)
√d 縮放            :  BLEU 26.8
d 縮放             :  BLEU 25.1
```

## 注意力權重的可視化

### 熱力圖解讀

注意力權重最常以熱力圖的形式呈現。典型的熱力圖中：

- 橫軸為鍵序列（被關注的內容）
- 縱軸為查詢序列（關注的主體）
- 每個儲存格的顏色深淺代表注意力權重的大小

### 常見模式

分析注意力權重可以發現多種模式：

**對角線模式**：常見於機器翻譯的編碼器-解碼器注意力。查詢位置主要集中在對應的鍵位置上。

**垂直條紋**：某些特殊 token（如句首符號、分隔符號）被大部分查詢位置關注。

**水平條紋**：某些查詢位置均勻地關注所有鍵位置，通常發生在解碼器生成停止符號時。

### 注意力的熵

注意力權重的熵可以用來量化注意力的集中程度：

```
H(α) = -Σ α_i log α_i
```

- H 接近 0：注意力高度集中（選擇性強）
- H 接近 log(n)：注意力均勻分布（選擇性弱）

過低的熵可能表示模型過度依賴少數位置；過高的熵可能表示模型沒有學到有用的注意力模式。

## 結論

注意力權重的計算雖然只有幾個簡單的數學步驟（點積、縮放、softmax），但每個步驟都有其精密的設計考量。理解這些細節不僅有助於調試模型，也能幫助設計新的注意力變體。
