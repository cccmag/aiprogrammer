# 全域 vs 區域注意力

## 兩種注意力範圍

注意力機制在設計時需要回答一個基本問題：模型應該關注輸入的所有位置，還是只關注一個子集？這個問題的答案將注意力機制分為兩大類：全域（Global）注意力和區域（Local）注意力。

## 全域注意力

### 工作原理

全域注意力在每個解碼步驟計算編碼器所有位置的注意力權重。這是最早在 Seq2Seq 模型中提出的注意力形式。

```
全域注意力示意圖（n=6）：

解碼器 t₁：關注位置 [1, 2, 3, 4, 5, 6]
解碼器 t₂：關注位置 [1, 2, 3, 4, 5, 6]
解碼器 t₃：關注位置 [1, 2, 3, 4, 5, 6]
```

### 優點

- **無資訊損失**：所有位置都有機會被注意到
- **長距離依賴**：即使相距很遠，也可以直接互動
- **簡單實現**：不需要預測注意力範圍

### 缺點

- **計算成本高**：O(n²) 的複雜度
- **雜訊引入**：不相關的位置也會被考慮
- **記憶體佔用大**：需要儲存所有位置的注意力分數

## 區域注意力

### 動機與原理

區域注意力（Luong 等人，2015）的動機是：在一個序列中，真正相關的位置通常只集中在一個局部範圍內。例如，在翻譯中，目標語言的一個詞通常只對應源語言中相鄰的 3-5 個詞。

區域注意力只計算編碼器輸出的一個子集上的注意力：

```
區域注意力示意圖（窗口大小 w=3）：

解碼器 t₁：關注位置 [1, 2, 3]
解碼器 t₂：關注位置 [2, 3, 4]
解碼器 t₃：關注位置 [3, 4, 5]
```

### Local-m 方法

Local-m 方法假設對齊點是單調的（與解碼器時間步大致對應）。注意力窗口以當前解碼步對應的對齊位置為中心：

```
窗口中心 p_t = t × (S / T)
其中 S 是源序列長度，T 是目標序列長度
```

權重計算時，窗口內的位置的權重還乘以一個高斯分數：

```
α_t(s) = α_t(s) × exp(-(s - p_t)² / (2σ²))
```

### Local-p 方法

Local-p 方法不假設對齊是單調的。它讓模型預測對齊位置 p_t 和標準差 σ：

```
p_t = S × sigmoid(v_p^T tanh(W_p h_t))
σ = S / 2  (固定或可學習)
```

位置預測是透過一個可微的小型網路實現的，因此可以端到端訓練。

## 比較分析

### 計算效率

| 方法 | 時間複雜度 | 記憶體複雜度 |
|------|-----------|-------------|
| 全域注意力 | O(n²) | O(n²) |
| 區域注意力 (m) | O(n·w) | O(w) |
| 區域注意力 (p) | O(n·w) | O(w) |

其中 w = 視窗大小（通常 w ≪ n）。當 n = 1000, w = 20 時，區域注意力比全域注意力快約 50 倍。

### 品質對比

Luong 的實驗結果顯示：
- 全域注意力品質略優於區域注意力
- 但區域注意力（特別是在大型資料集上）的品質差距很小
- Local-p 通常勝過 Local-m

### 適用場景

- **全域注意力**適合：中等長度序列（< 512 token）、需要精確對齊的任務
- **區域注意力**適合：超長序列（> 1000 token）、即時應用（低延遲需求）、資源受限的設備

## 在現代 Transformer 中的應用

雖然現代 Transformer 在編碼器中使用全域 Self-Attention，但區域注意力的思想在高效變體中得到了繼承：

- **滑動窗口注意力**（Longformer）：每個位置只關注周圍 w 個鄰居
- **壓縮注意力**（Compressive Transformer）：對歷史資訊進行摘要後再關注
- **全域+局部混合**（BigBird）：關鍵位置使用全域注意力，其他位置使用局部注意力

## 結論

全域與區域注意力的選擇是一個經典的效率-品質權衡。理解不同注意力範圍的優劣，有助於針對具體應用選擇最合適的注意力機制。在資源充足且序列長度適中時，全域注意力通常是首選；在長序列或資源受限的場景中，區域注意力是不可或缺的工具。
