# 回顧與結語

## 2024 年 5 月號結語

### 本月主題回顧

五月是行動開發領域特別熱鬧的月份。本期雜誌聚焦手機 App 開發，特別是 React Native 這個充滿活力的跨平台框架。

#### 程式技術亮點

**React Native 0.75 新架構**正式成為預設值，這標誌著 React Native 歷經數年研發的新架構終於達到穩定。Fabric Renderer 和 TurboModules 帶來的是質的飛躍：不再依賴 Bridge 進行序列化通訊，JavaScript 與原生端可以通過 JSI 直接呼叫，效能瓶頸被徹底打破。

**Expo SDK 52** 的發布讓開發者體驗達到新高度。EAS Build 整合 Apple Silicon Native 建置、雲端模擬器測試、OTA 更新等服務，讓「撰寫程式碼即可發布」不再是夢想。

**TypeScript 5.5** 和 **Kotlin 2.0** 的相繼發布，為 React Native 生態提供了更強大的基礎語言支援。型別安全、編譯速度提升，間接幫助 React Native 開發者寫出更可靠的程式碼。

#### AI 與行動開發的融合

**Core ML 6** 和 **ML Kit 即時翻譯** 的推出，展示了 AI 正在更深層次地融入行動開發。React Native 開發者可以透過原生模組輕鬆整合這些能力，為 App 帶來智慧化功能。

從邊緣 AI 推論到即時翻譯，行動裝置正在變成強大的 AI 運算平台。React Native 的角色，就是讓開發者能用最有效率的方式接觸這些能力。

### 主題延伸：跨平台的意義

跨平台開發的本質，不是「一次編寫，到處運行」這個技術口號，而是「一次學習，到處應用」這個思維模式。

React Native 的成功說明了幾件事：
- **抽象層的價值**：將 UI 元件抽象為宣告式 Component，是跨平台的關鍵
- **生態的力量**：npm 上百萬個套件，讓開發者不需要重複造輪子
- **社群驅動**：Facebook/Meta 開源後，社群的貢獻遠超過任何單一公司能達到的

### 未來展望

**React Native 的下一步**：新架構的穩定只是開始。我們期待看到更多原生 API 的 TurboModule 化，以及更好的 Windows、macOS、tvOS 支援。

**AI 原生 App**：隨著邊緣 AI 能力增強，我們將看到更多「AI-first」的行動應用。React Native 需要提供更好的 AI 整合開箱體驗。

**跨平台融合**：React Native 和 Flutter 的競爭將推動兩者都變得更好。最終受益的是開發者和使用者。

**Web + Native 界線模糊**：隨著 React Native for Web 和 Expo 的發展，Web 和 App 的界線將越來越模糊。
