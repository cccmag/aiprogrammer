# Focus 5：資料流水線架構設計

## 流水線的重要性

資料流水線是現代資料工程的命脈。從原始資料到可用資料資產，需要經歷提取、轉換、載入（ETL）、驗證、監控等多個環節。良好的流水線設計具備可重現性、可監控性、容錯能力，以及足够的彈性應對需求變化。

## 工具選擇

建立資料流水線有多種工具。 воздуш потоки是 Airbnb 開源的元資料管理系統，支援流水線編排和資料目錄。Apache Airflow 是成熟的任務排程系統，強調可擴展性和豐富的運算子生態。 Prefect 以 Python 優先的設計提供更現代的體驗。DVC 則專注於資料版本控制，與 Git 深度整合。

## 批次處理 vs 串流處理

批次處理適合定時執行的作業，如每日報表生成。串流處理則處理即時資料，如異常偵測。2021 年的趨勢是 Lambda 架構——結合批次層和速度層，用批處理結果保證準確性，用串流處理提供低延遲。Apache Kafka 是串流事實標準，Apache Flink 提供强大的串流處理能力。

## 資料品質監控

流水線的監控不僅是任務成功與否，還包括資料品質。Great Expectations 是 Python 的資料驗證庫，定義資料期望並自動驗證。結合告警系統，可在資料問題影響下游之前及時發現。Deequ 是 AWS 開源的資料品質工具，擅長大規模資料集的驗證。

## 設計模式

常見模式包括：鏡射模式（鏡像源資料庫）、事件採樣模式（響應特定事件）、久化檢查點模式（定期保存狀態）、及混合模式（結合多種策略）。選擇模式時需考慮資料特性、延遲需求和運算成本。
