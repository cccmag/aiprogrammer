# 為什麼要寫測試？測試驅動開發入門

## 測試的本質

測試是一種確認程式行為的行為。當我們執行一段程式並檢查結果是否符合預期時，其實就是在做測試。自動化測試將這個過程系統化、可重複化，讓電腦代替人工檢查。

## 手動測試的問題

假設你修改了一個函式，手動測試後發現「看起來正常」。但幾週後，這個改動悄悄破壞了某個邊緣功能，而你不知道。傳統手動測試無法捕捉這類回歸問題。

## 自動化測試的優勢

自動化測試可以：
- 快速執行，隨時反饋
- 可重複，消除人為錯誤
- 文件化預期行為
- 支援重構信心

## 測試驅動開發（TDD）

TDD 的核心循環是「紅-綠-重構」：

1. 紅：寫一個會失敗的測試
2. 綠：寫最少量程式碼讓測試通過
3. 重構：改善程式碼，保持測試通過

```python
# 第一步：寫測試（會失敗）
def test_add():
    assert add(1, 2) == 3

# 第二步：寫實現
def add(a, b):
    return a + b
```

## 第一個 pytest 測試

```python
# test_math.py
import pytest

def test_addition():
    assert 2 + 2 == 4

def test_string():
    assert "hello".upper() == "HELLO"

def test_list():
    nums = [1, 2, 3]
    nums.append(4)
    assert len(nums) == 4
```

執行 `pytest test_math.py`。

## 測試的組織原則

- 每個測試聚焦一個行為
- 測試名稱描述預期行為
- AAA 模式：Arrange（準備）、Act（執行）、Assert（斷言）
- 測試之間相互獨立

## 克服寫測試的心理障礙

很多開發者不喜歡寫測試，覺得是負擔。建議：
- 從最簡單的函式開始
- 先為 bug 寫測試確保修復
- 逐漸建立測試覆蓋率

測試是投資，不是負擔。初期投入的時間會在未來獲得回報。
