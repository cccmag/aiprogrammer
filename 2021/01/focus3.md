# Scaling Laws 規模法則

## 參數量、資料量與效能的關係

### Kaplan 等人的研究（2020）

OpenAI 的 Kaplan 等人發表了系統性的規模法則研究，發現：

```
效能 ∝ 參數量^0.073 × 資料量^0.095 × 計算量^0.076
```

關鍵發現：
- **模型大小**的影響最大
- **資料量**次之
- **計算量**的邊際效益遞減

### 神經語言模型的規模法則

```
         效能
          ↑
          │     ████  模型越大
          │   ██        效能越高
          │ ██
          │█
          └────────────────→ 計算量
```

### 計算資源的最佳配置

Kaplan 建議的計算資源分配：

| 資源類型 | 建議比例 |
|----------|----------|
| 模型參數 | 60-70% |
| 訓練資料 | 20-30% |
| 其他開銷 | 10% |

### 規模法則的意涵

1. **越大越好**：在一定範圍內，增加規模幾乎總是有效的
2. **計算效率**：更大的模型可以用更少的資料達到相同效能
3. **湧現能力**：當規模超過某個閾值時，會突然出現新能力

### 湧現能力（Emergent Abilities）

大規模語言模型表現出「湧現能力」：

- 300M 參數：無明顯進步
- 10B 參數：開始出現簡單推理
- 175B 參數：展現復雜推理和Few-shot能力

```
能力閾值：
10B ──┬── 簡單任務
       │
175B ─┼── Few-shot、複雜推理、程式碼生成
       │
1T+ ──┴── （未來可能出現更多能力）
```

### 批評與限制

- 規模法則可能只適用於特定任務
- 環境影響和效率問題常被忽略
- 「湧現」可能只是測量不連續的假象
