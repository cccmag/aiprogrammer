# 開源行動作業系統的興起

## 開源行動 OS 概況

### 主要專案

```
開源行動作業系統（2010 年）：
───────────────────────────
Android：       完全開源（Linux 核心）
Openmoko：      完全開源（早期嘗試）
WebOS：         部分開源（後來開源）
Maemo/MeeGo：   基於 Debian（ Nokia )
Symbian：       基金會形式開源
```

## Android 開源

### 開源程度

```
Android 開源現況（2010 年）：
───────────────────────────
開源部分：     Linux 核心、框架、函式庫
未開源：       Google Mobile Services (GMS)
差異：        AOSP vs 商業版本
```

### 原始碼結構

```
Android 開源專案（AOSP）：
───────────────────────────
frameworks/      應用框架
packages/        應用程式
system/          系統層
kernel/          Linux 核心（各廠商不同）
external/        外部專案
```

## Openmoko

### 專案介紹

```
Openmoko 介紹：
───────────────────────────
成立：        2006 年
目標：        完全自由的智慧手機
軟體：        基於 Openmoko Linux 发行版
硬體：        GTA01、GTA02
失敗原因：     硬體延遲、軟體不成熟
```

### 技術特點

```
Openmoko 技術：
───────────────────────────
OS：           客製化 Linux 发行版
UI：           基于 GTK+ 的窗口管理器
開發語言：     Python、C
開放程度：     完全開源
```

## WebOS

### Palm 的嘗試

```
WebOS 歷史：
───────────────────────────
2009 年：      Palm Pre 發布，WebOS 首次亮相
2010 年：      HP 收購 Palm
2011 年：      WebOS 開源（部分）
2013 年：      HP 將 WebOS 捐給 LG
```

### 技術架構

```
WebOS 架構：
───────────────────────────
核心：         Linux + HTML5 + JavaScript
中介軟體：     Node.js
UI：           Enyo framework
設計理念：     卡片式多工、網路為中心
```

## Maemo/MeeGo

### Nokia 的選擇

```
Maemo/MeeGo 歷史：
───────────────────────────
Maemo：        Nokia N900 使用
MeeGo：        Maemo + Moblin 合併
Intel：        積極參與
合作：         Nokia + Intel
```

### 與 Android 的比較

```
MeeGo vs Android：
───────────────────────────
目標市場：     MeeGo 更注重平板和車載
開發語言：     Both C++/Qt vs Java
生態系：       Android 領先
開放程度：     兩者都開源
```

## Symbian 基金會

### 開放策略

```
Symbian 基金會（2008-2010）：
───────────────────────────
2008 年：      Nokia 主導成立基金會
2009 年：      原始碼開源
2010 年：      基金會會員變動
2011 年：      Nokia 結束對 Symbian 的支持
```

### 開源影響

```
Symbian 開源的影響：
───────────────────────────
延遲：         開源反而延緩了開發
決定：         Nokia 最終放棄 Symbian
教訓：         開源不等於成功
```

## 開發者觀點

### 選擇考量

```
選擇開源 OS 的考量：
───────────────────────────
硬體支援：     驅動程式是否完整
文件：         開發文件是否充足
社群：         是否有活躍的開發者社群
未來：         專案的長期發展
```

### 參與方式

```
貢獻開源專案：
───────────────────────────
貢獻程式碼：   修复 bug、添加功能
翻譯：         多語言支持
文件：         撰寫和改進文件
測試：         測試回報問題
推廣：         推廣使用
```

---

## 結論

2010 年見證了多個開源行動作業系統的嘗試。雖然 Android 最終勝出，但其他專案如 WebOS、MeeGo 都提供了寶貴的經驗教訓。

開源模式在行動領域的影響深遠：它降低了進入門檻，促進了創新，但也顯示生態系和時機同樣重要。
