# 傳統視覺與深度視覺的比較

## 前言

在深度學習兴起之前，傳統電腦視覺方法主導了數十年。本文比較兩種方法的優劣。

---

## 一、傳統視覺方法

### 代表性特徵

| 方法 | 描述 |
|------|------|
| SIFT | 尺度不變特徵轉換 |
| HOG | 方向梯度直方圖 |
| Haar | Haar 小波特徵 |
| LBP | 局部二值模式 |
| SURF | 加速穩健特徵 |

### 經典流程

```
輸入圖像
  ↓
興趣點檢測
  ↓
特徵描述子提取
  ↓
特徵匹配
  ↓
幾何驗證
```

---

## 二、深度視覺方法

### 代表性架構

| 模型 | 年份 | 特點 |
|------|------|------|
| AlexNet | 2012 | 深度學習突破 |
| VGGNet | 2014 | 深層結構 |
| ResNet | 2015 | 殘差連接 |
| EfficientNet | 2019 | 高效率 |

### 深度學習流程

```
輸入圖像
  ↓
CNN 特徵提取（端到端）
  ↓
分類/檢測/分割
```

---

## 三、比較分析

### 1. 特徵學習

| 方面 | 傳統方法 | 深度學習 |
|------|---------|---------|
| 特徵設計 | 手動 | 自動學習 |
| 可解釋性 | 高 | 低 |
| 領域知識 | 需要 | 不需要 |

### 2. 效能

| 任務 | 傳統方法 | 深度學習 |
|------|---------|---------|
| 圖像分類 | 70-80% | 95%+ |
| 物體檢測 | 30-40% mAP | 80%+ mAP |
| 語義分割 | 50-60% | 85%+ |

### 3. 資料需求

| 方面 | 傳統方法 | 深度學習 |
|------|---------|---------|
| 訓練資料 | 少量（數百） | 大量（數萬） |
| 計算資源 | CPU 可行 | GPU 必要 |
| 推理速度 | 快 | 較慢 |

### 4. 泛化能力

| 方面 | 傳統方法 | 深度學習 |
|------|---------|---------|
| 跨領域 | 較好 | 需要微調 |
| 光照變化 | 敏感 | 較魯棒 |
| 遮擋 | 敏感 | 較魯棒 |

---

## 四、SIFT vs CNN

### SIFT 原理

```python
# 尺度空間極值檢測
# 興趣點定位
# 方向分配
# 特徵描述子（128維）
```

### CNN 原理

```python
# 多層卷積特徵提取
# 層次化表示
# 端到端分類
```

### 對比

| 方面 | SIFT | CNN |
|------|------|-----|
| 維度 | 128 | 數千 |
| 不變性 | 手動設計 | 學習 |
| 計算 | 複雜 | 矩陣運算 |
| 準確率（匹配） | 中等 | 高 |

---

## 五、何時使用哪種方法？

### 選擇傳統方法

- 訓練資料有限
- 需要可解釋性
- 計算資源有限
- 領域特定任務

### 選擇深度學習

- 大規模資料
- 需要最高精度
- 計算資源充足
- 通用任務
