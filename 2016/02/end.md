# 結語

本期介紹了 Docker 容器化技術的進階應用，從 Docker 1.10 的新功能、網路模型、映象管理到容器編排，涵蓋了建立、生產級容器化部署所需的關鍵知識。

## 關鍵 takeaways

1. **容器不是迷你虛擬機**：容器共享主機核心，設計理念是隔離應用而非硬體。理解這個根本差異，才能用好容器。

2. **網路模型靈活多變**：Docker 提供 bridge、host、overlay、macvlan 等多種網路驅動，適用於不同的應用場景。選擇正確的網路模式對效能與安全性都有影響。

3. **映象最佳化是持續的工作**：從多階段建置到 .dockerignore，每個環節都可能影響最終映象的大小與安全性。

4. **Swarm 讓編排不再困難**：Docker 1.12 整合的 Swarm Mode 提供了原生的容器編排能力，不需要學習複雜的工具就能開始使用。

5. **安全是基本要求**：從映象掃描到資源限制，從非 root 執行到網路隔離，安全性應該是每個容器化部署的基本要素，而非後續再加的功能。
