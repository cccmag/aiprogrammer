# 1. 雲端運算基本概念

## 雲端運算的起源

雲端運算的概念可追溯到 1960 年代，John McCarthy 就提出「運算力將來可成為公共設施」。然而真正落地需要克服虛擬化技術、網路頻寬、資料中心規模經濟等多項挑戰。2006 年 Amazon 推出 EC2，被視為現代雲端服務的起點。此後 Google、Microsoft、IBM 相繼投入，雲端產業快速成長。

## 雲端運算的五大特性

根據 NIST（美國國家標準與技術研究院）的定義，雲端運算具備以下特性：

**隨選自助服務（On-demand Self-service）**：使用者可隨時透過網頁或 API 要求配置資源，無需供應商人工介入。這大幅縮短了從需求到上線的時間。

**廣泛網路存取（Broad Network Access）**：透過標準網路協定的 API 存取資源，使用者可從筆電、手機、平板等各種裝置管理雲端環境。

**資源池化（Resource Pooling）**：供應商將實體伺服器、儲存、網路設備虛擬化後形成共享資源池，透過多租戶機制隔離不同客戶的資源。

**快速彈性（Rapid Elasticity）**：資源可快速橫向擴展或縮減，無論是應對突發流量或節省閒置成本，都能在短時間內完成調整。

**服務可計量（Measured Service）**：所有資源使用都有計量和監控，依實際使用量計費讓成本更加透明可預測。

## 公有雲、私有雲、混合雲

**公有雲**由雲端供應商提供基礎設施，複数客戶共享同一組實體資源。優勢是成本低、彈性高、免維護；劣勢是資料存放位置與合規要求需要留意。

**私有雲**是企业或組織自建或委託第三方管理的專屬雲端環境。優勢是完全控制、安全性高、適合高度監管的產業；劣勢是前期投資大、需要專門維運團隊。

**混合雲**結合公有雲與私有雲的優點，常見模式是將敏感資料放在私有雲，一般工作負載放在公有雲，透過 VPN 或專線連接兩者。

## 雲端服務的價值

對新創公司而言，雲端大幅降低了創業門檻。過去需要數十萬甚至數百萬的初始硬體投資，現在每個月幾百元就能開始營運。對大型企業而言，雲端提供了快速擴展的能力，能更靈活地因應市場變化。對開發者而言，雲端讓實驗新想法變得更容易，失敗成本更低。
