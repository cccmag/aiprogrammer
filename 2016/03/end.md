# 結語

本期介紹了資訊安全的核心概念與常見漏洞防護。從 OWASP Top 10、密碼學基礎、常見漏洞類型到安全開發生命週期，希望讀者能建立系統性的資安知識。

## 關鍵 takeaways

1. **安全是每個人的責任**：不僅是安全團隊，每位開發者都應該具備基本的安全意識。許多安全漏洞源於開發階段的疏漏，而非系統管理或網路配置。

2. **防禦深度是關鍵**：不要依賴單一安全措施。輸入驗證、輸出編碼、權限控制、日誌監控，這些防線需要多層次設置。

3. **了解攻擊才能防護**：理解 SQL 注入、XSS、CSRF 的攻擊原理，才能設計有效的防護機制。建議讀者動手演練，在安全的環境下嘗試這些攻擊手法。

4. **加密是保護的最後防線**：即使其他防線都被突破，良好的加密實踐（如密碼雜湊、TLS）可以保護敏感資料不被解讀。

5. **持續關注安全議題**：資安領域發展迅速，新的漏洞與攻擊手法層出不窮。建議定期關注 OWASP、CVE、資安新聞，保持對威脅態勢的了解。
