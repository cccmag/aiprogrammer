# 2. OWASP Top 10 風險

## OWASP 簡介

OWASP（Open Web Application Security Project）是一個開放的資安社群，致力於提升軟體安全性。他們發布的「OWASP Top 10」是最被廣泛引用的 Web 應用程式安全風險文件。2013 年版本涵蓋了當時最常見且最危險的 Web 安全問題。

## A1 - 注入（Injection）

當不受信任的資料被傳送到直譯器執行時，就會發生注入攻擊。最常見的是 SQL 注入。

**風險**：攻擊者可以竊取、篡改或刪除資料庫中的資料，甚至執行系統命令。

**防護**：
- 使用參數化查詢（Prepared Statements）
- 輸入驗證
- 最小權限原則

## A2 - 身份驗證失效（Broken Authentication）

應用程式的身份驗證功能存在缺陷，允許攻擊者竊取密碼、Token 或利用漏洞繞過驗證。

**風險**：攻擊者可以假冒任何使用者身份，包括管理員。

**防護**：
- 實作適當的 session 管理
- 使用多因素認證
- 強制複雜密碼策略
- 限制登入失敗次數

## A3 - 敏感資料暴露（Sensitive Data Exposure）

應用程式未能正確保護敏感資料，如信用卡號、身份證號、密碼等。

**風險**：資料在傳輸或儲存過程中被竊取。

**防護**：
- 加密靜止資料與傳輸中資料
- 使用現代加密演算法（如 AES-256、TLS 1.2）
- 不要儲存不必要的敏感資料

## A4 - XML 外部實體（XXE - XML External Entities）

應用程式解析 XML 的方式有缺陷，允許引用外部實體。

**風險**：可讀取伺服器上的任意檔案、執行遠端程式碼、進行 DDoS 攻擊。

**防護**：
- 使用簡單的資料格式（如 JSON）
- 禁用 XML 外部實體解析
- 輸入驗證

## A5 - 存取控制失效（Broken Access Control）

使用者可以超出其權限執行操作，例如存取其他使用者的資料或修改權限。

**風險**：未授權的資料存取與操作。

**防護**：
- 預設拒絕存取
- 每次請求都驗證權限
- 記錄存取失敗

## A6 - 安全設定錯誤（Security Misconfiguration）

不安全的預設設定、過時的軟體、錯誤的權限設定等。

**風險**：完整的系統入侵。

**防護**：
- 自動化安全設定檢查
- 及時更新所有軟體
- 最小化功能原則

## A7 - 跨站腳本（XSS - Cross-Site Scripting）

應用程式未經適當驗證就將使用者輸入包含在輸出中。

**風險**：攻擊者可以在受害者瀏覽器中執行腳本，竊取 session、破壞頁面。

**防護**：
- 輸出編碼
- 內容安全策略（CSP）
- HTTPOnly Cookie

## A8 - 不安全的反序列化（Insecure Deserialization）

應用程式反序列化不受信任的資料。

**風險**：遠端程式碼執行、注入攻擊、權限提升。

**防護**：
- 不反序列化來自不可信來源的資料
- 完整性檢查
- 隔離執行反序列化程式碼

## A9 - 使用有漏洞的元件（Using Components with Known Vulnerabilities）

使用了含有已知漏洞的函式庫或框架。

**風險**：任何已知漏洞都可能被利用。

**防護**：
- 及時更新元件
- 使用元件安全掃描工具
- 只從官方來源取得元件

## A10 - 不足的日誌與監控（Insufficient Logging & Monitoring）

缺乏適當的日誌記錄與入侵偵測，導致攻擊無法被發現。

**風險**：攻擊可以長期潛伏而不被察覺。

**防護**：
- 記錄所有重要事件
- 建立警報與響應機制
- 定期審查日誌
