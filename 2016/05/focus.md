## 程式語言的層次

程式語言可以分為多個層次：

- **高階語言**：Python、Java、JavaScript、C#
- **低階語言**：C、Go、Rust
- **組合語言**：特定於硬體的指令
- **機器碼**：CPU 直接執行的二進位

從高階語言到機器碼，需要經過編譯或解釋的過程。

## 編譯 vs 解釋

### 編譯型語言

原始碼在執行前被完全翻譯成機器碼：

```
原始碼 → [編譯器] → 目的碼 → [執行] → 結果
```

優點：執行速度快、預先發現錯誤
缺點：平臺相依、編譯時間長

代表：C、C++、Rust、Go

### 解釋型語言

原始碼在執行時逐行翻譯：

```
原始碼 → [直譯器] → 翻譯 + 執行（逐行）
```

優點：跨平臺、開發效率高
缺點：執行速度慢

代表：Python、Ruby、JavaScript

### 虛擬機與位元組碼

折衷方案：編譯成平臺無關的位元組碼，再由虛擬機執行：

```
原始碼 → [編譯器] → 位元組碼 → [VM] → 結果
```

優點：跨平臺、兼具安全性
缺點：執行速度較純編譯慢

代表：Java（JVM）、C#（.NET CLR）

## 我們從哪裡來，要往哪裡去

從 1950 年代的 FORTRAN 和 Lisp，到 1970 年代的 C 和 Pascal，到 1990 年代的 Java 和 Python，到 2010 年代的 Rust 和 Swift——程式語言持續演化。每種語言都體現了不同的設計哲學和技術選擇。
