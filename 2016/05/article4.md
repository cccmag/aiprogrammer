# V8 引擎的 JIT 編譯

## V8 簡介

V8 是 Google 的開源 JavaScript 引擎，為 Chrome 和 Node.js 提供動力。V8 直接將 JavaScript 編譯為機器碼，無需位元組碼。

## V8 的歷史

- 2008：Chrome 發布，V8 首次亮相
- 2010：V8 引入 Crankshaft 優化編譯器
- 2016：Ignition 解釋器整合，減少記憶體使用

## V8 架構

```
JavaScript 原始碼
       ↓
   Parser
       ↓
   AST（抽象語法樹）
       ↓
   Ignition（直譯器）
   + TurboFan（優化編譯器）
       ↓
    機器碼
```

## Ignition 解釋器

2016 年 V8 開始整合 Ignition 解釋器：

```javascript
// Ignition 將 JS 編譯為位元組碼
function add(a, b) {
    return a + b;
}
```

位元組碼示例：
```
[generated bytecode for function: add]
Parameter count 3
Frame size 0
   0 : Ldar a0        ; 將 a0 載入累加器
   2 : Add a1, [0]    ; 加上 a1
   5 : Return          ; 返回
```

## TurboFan 優化編譯器

TurboFan 對熱點程式碼進行高度優化：

### 優化觸發條件

```javascript
// 熱點檢測
function add(a, b) {
    return a + b;
}

// 多次呼叫觸發 JIT
for (let i = 0; i < 100000; i++) {
    add(1, 2);
}
```

### 內聯

```javascript
// 內聯前
function square(x) {
    return x * x;
}
function calc(x) {
    return square(x) + square(x);
}

// 內聯後
function calc(x) {
    return x * x + x * x;  // 直接展開
}
```

### Deoptimization

當假設失敗時回退：

```javascript
function add(a, b) {
    return a + b;
}

add(1, 2);      // 數字版本
add("a", "b");  // 觸發 Deoptimization，回退到直譯
```

## V8 效能技巧

### 穩定類型

```javascript
// 好的做法：類型穩定
function sum(arr) {
    let total = 0;
    for (let i = 0; i < arr.length; i++) {
        total += arr[i];  // 保證 arr 是數字陣列
    }
    return total;
}
```

### 避免動態屬性

```javascript
// 避免動態增加屬性
function Point(x, y) {
    this.x = x;  // 固定屬性
    this.y = y;
}
```

### 使用 TypedArray

```javascript
// TypedArray 效能更好
const arr = new Float64Array(1000000);
```
