# 本期焦點：函數式程式設計

## 什麼是函數式程式設計？

函數式程式設計（Functional Programming，簡稱 FP）是一種將計算視為數學函式評估的程式設計範式。其核心思想源自 Lambda Calculus——由 Alonzo Church 在 1930 年代發明的形式系統。與命令式程式設計不同，函數式程式設計強調：

- **不可變性（Immutability）**：資料一旦創建便不可更改
- **純函式（Pure Functions）**：相同輸入永遠產生相同輸出，無副作用
- **高階函式（Higher-Order Functions）**：函式可作為參數和返回值
- **運算式導向（Expression-Oriented）**：以運算式而非語句構建程式

## 為什麼要學習函數式程式設計？

### 提升程式碼品質

純函式易於測試和除錯，因為它們沒有副作用，不依賴全域狀態。當問題被分解為純函式時，推理程為為變得簡單。

### 並行與分散式運算

不可變資料結構天然適合並行處理。由於資料不會被意外修改，不需要鎖機制即可安全地在多執行緒環境中使用。

### 抽象層次提升

高階函式允許我們將常見模式抽象化。`map`、`filter`、`reduce` 等函式可以將複雜的迴圈邏輯簡化為清晰的宣告式表達。

## 函數式語言的復興

近年來，隨著多核心 CPU 和分散式系統的普及，函數式程式設計從學術界走向主流產業。從金融科技公司使用 Haskell 建構交易系統，到 Twitter 採用 Scala 處理海量資料，函數式語言在生產環境中展現了卓越的表現。

更重要的是，函數式概念正在滲透到主流語言中。Java 8 引入 Stream API 和 Lambda 表達式，C++11 支援 std::function 和 Lambda，Python 的 functools 模組日益豐富，JavaScript 的陣列方法（map、filter、reduce）成為標準慣例。

## 我們從哪裡來，要往哪裡去

1958 年，John McCarthy 發明了 Lisp，這是第一個函數式語言。1978 年，John Backus 發表經典論文《Can Programming Be Liberated from the von Neumann Style?》，系統闡述函數式程式設計的優勢。

2016 年的今天，我們站在巨人的肩膀上。經過半個世紀的發展，函數式程式設計從理論走向實踐，從學術走向產業。讓我們一起探索這段精彩的旅程。
