# Linux 與開源精神

## 自由軟體運動的起源

1983 年，Richard Stallman 在 MIT 人工智慧實驗室發起了 GNU 計畫。他的目標是建立一個完全自由的 Unix 相容作業系統。Stallman 認為軟體應該自由分享，使用者應該擁有執行、複製、散佈、研究和修改軟體的自由。

1985 年，Stallman 創立了自由軟體基金會 (FSF) 並撰寫了 GNU 通用公共授權條款 (GPL)。GPL 的核心思想是「Copyleft」——利用版權法來保障軟體的自由。如果你散佈 GPL 授權的軟體，你必須同時提供原始碼。

## Linux 核心的誕生

1991 年，芬蘭赫爾辛基大學的學生 Linus Torvalds 在 comp.os.minix 新聞群組中發布了一個訊息：「我正在開發一個 (免費的) 作業系統——這只是一個業餘愛好，不會像 GNU 那樣龐大和專業。」這就是 Linux 核心的起點。

```
Linux 發展歷程：
1991: Linux 0.01 發布 (約 10,000 行程式碼)
1992: 採用 GPLv2 授權
1994: Linux 1.0 發布
1996: Linux 2.0 支援 SMP
2003: Linux 2.6 企業級功能
2015: Linux 4.0 無停機更新
2025: Linux 6.x 持續進化
2026: Linux 7.0 正式發布
```

## 開源授權的種類

### 主要開源授權

- **GPL**: 強 Copyleft，修改後的衍生作品必須同樣使用 GPL
- **LGPL**: 弱 Copyleft，允許動態連結的專有軟體使用
- **MIT/BSD**: 寬鬆授權，允許閉源的商業使用
- **Apache 2.0**: 類似 MIT，但包含專利授權條款

### 如何選擇授權

```
想讓所有人使用 (包含閉源) → MIT / Apache 2.0
想確保衍生作品也開源 → GPL
想給函式庫使用 → LGPL
不想承擔法律責任 → 任何一種都包含免責條款
```

## 社群文化與貢獻

Linux 和開源社群的獨特之處在於其治理模式。Linus Torvalds 作為「仁慈的獨裁者」(BDFL) 擁有核心決策權，但日常開發由數千名來自世界各地的貢獻者協作完成。

### 貢獻方式

- 提交程式碼 (Pull Request)
- 回報錯誤 (Bug Report)
- 撰寫文件 (Documentation)
- 翻譯與本地化
- 社群支援與回答問題

## 為什麼開發者應該關心開源

1. **學習機會**: 閱讀頂尖開發者的程式碼是學習的最佳方式
2. **作品集**: 開源貢獻是履歷表上的亮點
3. **社群網絡**: 與全球開發者建立連結
4. **影響力**: 你的程式碼可能被數百萬人使用
5. **職涯發展**: 許多公司重視開源貢獻經驗
