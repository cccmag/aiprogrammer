# 本期焦點

## Linux 命令列與開發環境：掌握開發者的核心工具

### 引言

命令列介面 (CLI) 是每一位軟體開發者的基本功。無論你使用的是 Windows、macOS 還是 Linux，命令列工具都是最高效、最強大的開發環境。在圖形化介面日益普及的今天，為什麼我們還要學習命令列？

答案很簡單：**自動化**。命令列工具可以串接、組合、腳本化——這是圖形化介面永遠無法比擬的能力。一個熟練的命令列使用者可以在幾秒鐘內完成圖形化介面需要滑鼠點擊數十次才能完成的工作。

本期雜誌將帶領讀者深入探索 Linux 命令列與開發環境，從開源精神的本源到現代開發環境的建置。

## 濃縮回顧

### 命令列的哲學

Unix 命令列工具遵循「做一件事、做好一件事」的哲學。每一個工具都很簡單，但透過管線 (pipe) 組合起來，它們可以完成極其複雜的任務。

```
who | awk '{print $1}' | sort | uniq -c | sort -rn
```

這一行命令可以列出系統中所有登入使用者的活躍程度——這就是 Unix 哲學的威力。

### 三大核心能力

一個開發者需要掌握三項命令列核心能力：

1. **導航與操作**：在檔案系統中移動、建立和刪除檔案、管理目錄結構
2. **文字處理**：搜尋、取代、分析文字資料
3. **自動化腳本**：將重複的工作寫成腳本一次執行

### 開發環境的演進

從 1970 年代的 ed 行編輯器到 VS Code 的遠端開發容器，開發環境已經走過了漫長的演進之路。但不變的是命令列的永恆地位——即使是最現代化的 IDE，底層仍然依賴命令列工具來完成編譯、測試和部署。
