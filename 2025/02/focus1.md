# 資料結構為什麼重要？

## 程式 = 資料結構 + 演算法

瑞士電腦科學家 Niklaus Wirth 曾寫過一本經典著作《Algorithms + Data Structures = Programs》，這句話道出了程式設計的核心本質。資料結構是程式的骨架，演算法則是程式的靈魂，兩者相輔相成，缺一不可。

## 為什麼要先學資料結構？

許多初學者急著寫出能動的程式，卻忽略了資料結構的重要性。事實上，選擇正確的資料結構，往往能讓程式碼更簡潔、執行更高效。舉例來說，若你需要「先進後出」的資料存取方式，堆疊（Stack）就是最佳選擇；若需要「先進先出」，則該使用佇列（Queue）。

## 資料結構影響效能

不同資料結構在同樣操作上的時間複雜度可能天差地別。陣列存取是 O(1)，但插入刪除是 O(n)；鏈結串列存取是 O(n)，但插入刪除是 O(1)。了解這些特性，才能為問題選擇最合適的工具。

## 面試與競賽的必備知識

國內外科技公司的技術面試，幾乎都會考資料結構與演算法。從 Google、Meta 到台灣的聯發科、台積電，面試官都希望看見你對這些基礎概念的掌握程度。
